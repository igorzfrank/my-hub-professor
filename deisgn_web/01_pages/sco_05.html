1 🎯 O que é o Figma?
🎯 Definição
Figma é uma ferramenta de design de interface baseada em navegador (web) que permite criar, prototipar e colaborar em tempo real. É a ferramenta mais usada no mercado para UI/UX Design, funcionando em qualquer sistema operacional (Windows, Mac, Linux) sem instalação de software pesado.
💡 Analogia simples
O Figma é como um Google Docs do design — várias pessoas podem trabalhar no mesmo arquivo ao mesmo tempo, vendo as alterações em tempo real, comentando e co-criando sem precisar baixar ou enviar arquivos.

🖼️Frames — As telas do seu projeto
🎯 O que são Frames?
Frames são contêineres que representam telas ou seções do seu design. Eles funcionam como "pranchetas" onde você organiza os elementos visuais. Cada frame pode ter tamanhos predefinidos (iPhone, Desktop, Tablet) ou personalizados.

3 🧩 COMPONENTES — Reutilização Inteligente
🎯 O que são Componentes?
Componentes são elementos reutilizáveis que funcionam como um "molde mestre". Quando você cria um componente (ex: um botão), todas as cópias (instâncias) se atualizam automaticamente se o principal for alterado. É o conceito mais poderoso do Figma.
💡 Analogia
Componente principal é como o molde de um carimbo — cada vez que você "carimba" (copia), gera uma instância idêntica. Se você mudar o molde, todos os carimbos já feitos se atualizam automaticamente.
📂Hierarquia dos Componentes
⭐ Componente Principal
O "molde mestre". Fica no arquivo e define o padrão para todas as cópias.
📋 Instância
Cópia do componente principal. Pode ter pequenas variações (cor, texto).
🎨 Variantes
Versões diferentes do mesmo componente (ex: botão primary, secondary, disabled).
📚 Biblioteca
Coleção de componentes compartilhada entre diferentes arquivos da equipe.
✅ Exemplo prático: Crie um componente "Botão" com cor azul. Use 50 instâncias em 20 telas. Depois, mude a cor do principal para verde. Todas as 50 instâncias se atualizam automaticamente.
4 ⚡ AUTO LAYOUT — Layouts Flexíveis e Responsivos
🎯 O que é Auto Layout?
Auto Layout é uma propriedade que você aplica a frames ou componentes para que eles se ajustem automaticamente ao conteúdo interno. Funciona como o Flexbox do CSS — os elementos internos se redistribuem quando você adiciona, remove ou redimensiona itens.
💡 Analogia
É como um elástico inteligente — se você adiciona mais cartões a uma lista, o contêiner cresce automaticamente. Se você remove, ele encolhe. Os espaçamentos entre os itens se mantêm consistentes.
⚙️Propriedades do Auto Layout
↕️ Direção
Vertical (empilhar) ou Horizontal (lado a lado).
📏 Espaçamento (Gap)
Distância fixa entre os itens internos do layout.
📐 Padding
Espaço interno entre a borda do contêiner e o conteúdo.
📦 Alinhamento
Como os itens se alinham dentro do contêiner (topo, centro, base).
🔄 Resizing
Hug contents (abraçar conteúdo) ou Fill container (preencher espaço).
🎨 Distribuição
Como o espaço extra é distribuído entre os itens.
✅ Dica prática: Use Auto Layout em botões (texto + padding ajustam automaticamente), listas de cards (adicionar/remover itens sem quebrar o layout) e formulários (campos se ajustam ao tamanho do frame).
5 🎨 Estilos — Cores, Textos e Efeitos
🎨 Color Styles
O que são: Paletas de cores salvas que podem ser reutilizadas em todo o projeto.

Vantagem: Mudar uma cor no estilo atualiza automaticamente todos os elementos que a usam.

Exemplo: Criar estilos "Primary/500", "Secondary/300", "Neutral/100" e aplicar em botões, fundos e textos.
🔤 Text Styles
O que são: Conjuntos salvos de tipografia (família, tamanho, peso, altura de linha).

Vantagem: Consistência tipográfica em todo o design system.

Exemplo: Criar estilos "H1/Desktop/48px", "Body/16px", "Caption/12px" e aplicar em todos os textos.
✨Outros estilos reutilizáveis
📐 Grid Styles
Grades reutilizáveis para alinhamento consistente.
🖼️ Effect Styles
Sombras, blur, gradientes salvos para reutilização.
🔤 Font Variables
Fontes com múltiplos pesos organizadas em famílias.
6 🔗 PROTOTIPAGEM — Dando Vida ao Design
🎯 O que é Prototipagem no Figma?
Prototipagem é o recurso que permite conectar frames e criar interações clicáveis que simulam a navegação real do produto. É como criar um "vídeo interativo" onde o cliente ou usuário pode clicar e navegar entre telas como se fosse o app final.
💡 Analogia
É como criar um PowerPoint com hiperlinks, mas muito mais fluido. Você liga um botão da Tela A ao destino Tela B, define uma animação de transição e pronto — ao clicar, o protótipo navega exatamente como o app real.
⚙️Principais recursos de prototipagem
🔗 Conexões (Links)
Conecte um elemento a outro frame. Gatilhos: clique, hover, pressionar, arrastar.
🎬 Animações
Transições: dissolve, slide, push, move, smart animate.
🔄 Smart Animate
Anima automaticamente mudanças entre frames com mesmos nomes de camadas.
📱 Preview
Teste no navegador ou no app Figma Mirror (celular) em tempo real.
🎯 Overlays
Sobrepor modais, tooltips e dropdowns sobre a tela atual.
📤 Compartilhamento
Gere um link público para qualquer pessoa testar o protótipo.
✅ Dica: Use Smart Animate para transições suaves — ele interpola automaticamente posição, tamanho, opacidade e rotação entre dois frames com elementos de mesmo nome de camada.
7 🤝 Colaboração, Variáveis e Recursos Avançados
👥 Colaboração em Tempo Real
• Múltiplas pessoas editando ao mesmo tempo
• Cursores coloridos com nome de cada pessoa
• Comentários diretamente nos elementos
• Histórico de versões (30 dias grátis)
• Modo observação (View only)
• Compartilhamento por link público ou restrito
🔢 Variáveis
• Armazenam valores reutilizáveis (cores, números, textos)
• Modo light/dark com um clique
• Mudança global de cores, espaçamentos, bordas
• Diferentes coleções para diferentes marcas
• Suporte a números para paddings e tamanhos
• Integração com tokens de design
🛠️Outros recursos essenciais
🔌 Plugins
Ícones, imagens, gráficos, Lorem ipsum, acessibilidade.
📤 Exportação
PNG, SVG, JPG, PDF — controle de escala e qualidade.
💻 Dev Mode
Modo para desenvolvedores extraírem CSS, medidas e assets.

9 ✅ Boas Práticas e Fluxo de Trabalho
✅ Fazer
✅ Nomear camadas e frames com clareza
✅ Usar componentes para elementos repetidos
✅ Criar Auto Layout em listas e botões
✅ Manter biblioteca de estilos (cores, textos)
✅ Prototipar o fluxo principal
✅ Comentários para feedback da equipe
✅ Organizar páginas por funcionalidade
✅ Usar grids e colunas para alinhamento
❌ Evitar
❌ Camadas com nomes genéricos ("Rectangle 237")
❌ Copiar e colar em vez de usar componentes
❌ Ajustar manualmente cada item da lista
❌ Cores e fontes não padronizadas
❌ Protótipo complexo demais desnecessário
❌ Ignorar feedback dos desenvolvedores
❌ Misturar telas de diferentes fluxos
❌ Esquecer de testar no celular (Figma Mirror)
