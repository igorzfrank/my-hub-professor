1 👁️ HIERARQUIA VISUAL — Guiando o Olhar
🎯 O que é?
Organização que guia o olhar do usuário pela interface, mostrando o que é mais importante, secundário e a ordem de leitura.
💡 Analogia
Como um maestro — ele decide quais instrumentos tocam mais alto e quais mais baixo. Se tudo grita, nada é ouvido.
🎻8 Princípios da Hierarquia Visual
📏
1. Tamanho
Maior = mais importante. Use escala proporcional.
🎨
2. Cor e Contraste
Cores vibrantes para CTAs. Alto contraste texto-fundo.
🧲
3. Proximidade
Elementos próximos = relacionados. Agrupe labels com campos.
⬜
4. Espaço em Branco
Mais espaço ao redor = mais atenção. Menos é mais.
🔤
5. Tipografia
Bold = títulos. Regular = corpo. Itálico = ênfase.
📐
6. Alinhamento
Cria ordem de leitura previsível. Use grid.
📦
7. Textura e Estilo
Sombras criam profundidade. Destaque cards e botões.
✨
8. Movimento
Animações com moderação. Nunca distraia do conteúdo.
🧠 Regra de ouro: Se tudo grita, nada é ouvido. A hierarquia funciona por contraste — o importante se destaca porque o secundário é discreto.
2 📐 GRIDS APLICADOS — A Estrutura Invisível
🎯 Por que usar?
Espinha dorsal do layout profissional. Garante consistência, alinhamento e ritmo visual. Sem grid, os elementos "flutuam".
💡 Analogia
Como o esqueleto de um prédio — invisível para o usuário final, mas essencial para o designer construir.
📏Anatomia do Grid
📐
Colunas
Desktop: 12 | Tablet: 8 | Mobile: 4
⬜
Gutters
Espaço entre colunas: 16px a 32px
📏
Margens
Desktop: 80-120px | Mobile: 16-24px
📱
Breakpoints
Mobile < 768px | Desktop > 1024px
/* Grid no Figma */
Desktop (1440px): 12 colunas | Margin: 120px | Gutter: 24px
Tablet (768px): 8 colunas | Margin: 40px | Gutter: 16px
Mobile (375px): 4 colunas | Margin: 16px | Gutter: 16px

/* Como aplicar: Frame → Layout Grid → + → Grid */
💡 Mobile-First: Comece pelo mobile (375px) e expanda para desktop. É mais fácil adicionar do que remover elementos.
3 🖼️ ALTA FIDELIDADE — Do Rabisco ao Protótipo
🎯 O que é?
Representação visual mais próxima do produto final. Cores reais, tipografia definitiva, imagens finais e componentes interativos. Pronto para desenvolvimento.

Botões, cards e inputs reutilizáveis

⚠️ Erros Comuns
❌ Cores inconsistentes
❌ Espaçamentos "no olho"
❌ Textos placeholder
❌ Imagens baixa resolução
❌ Ignorar estados (hover, active)
❌ Não testar responsividade
4 🏗️ CONSTRUÇÃO DE TELAS — Passo a Passo
📋 5 Passos para uma Landing Page
1 Configure Frame + Grid (12 col, margin 120px)
2 Estruture Zonas: Header, Hero, Body, Footer
3 Preencha com Componentes (Navbar, Cards, CTAs)
4 Aplique Hierarquia Visual (tamanhos, cores)
5 Revise e Prototipe (conecte botões, teste fluxo)
HEADER → Logo + Menu + CTA
HERO → Título + Subtítulo + CTA + Imagem
SEÇÕES → Diferenciais, Produtos, Depoimentos
FOOTER → Links, Contato, Redes Sociais
✨ Acabamento Profissional
✅ 8px grid para espaçamentos
✅ Alinhamento pixel-perfect
✅ Contraste WCAG AA
✅ Estados de interação documentados
