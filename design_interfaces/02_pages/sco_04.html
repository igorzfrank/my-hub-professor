1 ⚡ AUTO LAYOUT — Layouts Flexíveis
🎯 O que é?
Propriedade do Figma que torna elementos dinamicamente responsivos. Como Flexbox do CSS — itens se redistribuem automaticamente ao adicionar, remover ou redimensionar.
💡 Analogia
Como um elástico inteligente — adicionou mais itens? O contêiner cresce. Removeu? Ele encolhe. Os espaçamentos se mantêm consistentes automaticamente.

2 ✅ FINALIZAÇÃO — Checklist Profissional
🎯 O que é finalizar?
Garantir que tudo está pronto para entrega: componentes nomeados, camadas organizadas, cores consistentes, prototipagem funcional. É o acabamento profissional.
💡 Pergunta do desenvolvedor
"Se eu recebesse este arquivo para codar, saberia o que fazer sem perguntar nada?" Se a resposta for não, ainda há trabalho.
✅Checklist de Finalização

🧩
Componentes
Não grupos soltos · Variantes configuradas
📐

Alinhado · Auto Layout aplicado · Escala 8px
📝
Conteúdo
Textos reais · Imagens finais · Ícones oficiais
🔗

Organização
Camadas nomeadas · Páginas organizadas
3 📂 ORGANIZAÇÃO — Fluxo Profissional
🎯 Por que organizar?
Arquivo desorganizado = oficina com ferramentas espalhadas. Organização aumenta produtividade, facilita colaboração e simplifica manutenção.
💡 Dica de ouro
Reserve os últimos 30 minutos de cada sessão para organizar: renomear, limpar estilos, revisar componentes.
📄 Páginas do Arquivo
├── 🎨 Style Guide
├── 🧩 Components
├── 📱 Mobile
├── 💻 Desktop
├── 🔗 Prototype
└── 🗑️ Archive
✅ Frames: Funcionalidade/Variação
Home/v1 · Checkout/Step2
✅ Componentes: Tipo/Variação/Estado
Button/Primary/Default
✅ Camadas: nomes descritivos
Nunca "Rectangle 237"
⌨️ Atalhos Essenciais
Shift + A → Auto Layout
Ctrl + Alt + K → Componente
Ctrl + G → Agrupar em Frame
Ctrl + R → Renomear
🔌 Plugins Úteis
Rename It → renomear em lote
Similayer → selecionar similares
Clean Document → limpar estilos
Design Lint → verificar consistência
4 🧪 EXEMPLO — Projeto Real Passo a Passo
1 Configure o arquivo — Crie páginas: Style Guide, Components, Mobile, Desktop, Prototype. Defina grids.
2 Crie componentes base — Button, Input, Card, Navbar, Footer. Use Auto Layout em todos.
3 Monte telas (Mobile First) — Comece pelo mobile (375px). Arraste componentes do Assets. Conteúdo real.
4 Adapte para Desktop — Duplique mobile, mude para 1440px, ajuste para 12 colunas com Auto Layout.
5 Prototipe e Revise — Conecte fluxos, revise cores, tipografia, alinhamentos. Aplique checklist.

5 ✅ BOAS PRÁTICAS — Hábitos Profissionais
✅ O que fazer
✅ Auto Layout em todos componentes
✅ Nomeie camadas assim que criá-las
✅ Style Guide em página separada
✅ Escala de 8px para espaçamentos
✅ Commit semântico (versões nomeadas)
✅ Revise o checklist antes de entregar
✅ Arquive versões antigas no Archive
❌ O que evitar
❌ "Rectangle 237" — nomes genéricos
❌ Ajustes manuais sem Auto Layout
❌ Cores e fontes fora do Style Guide
❌ Deixar "para depois" a organização
❌ Componentes sem variantes
❌ Ignorar o grid
❌ Misturar rascunhos com telas finais
🧠 Lembre-se: Organização não é perda de tempo — é investimento. 30 minutos organizando hoje economizam horas de retrabalho amanhã. Um arquivo bem estruturado é sua assinatura profissional.
