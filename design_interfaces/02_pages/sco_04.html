1 ⚡ AUTO LAYOUT — Layouts Flexíveis e Automatizados
🎯 O que é Auto Layout?
Auto Layout é uma propriedade do Figma que torna frames e componentes dinamicamente responsivos. Funciona como o Flexbox do CSS — os elementos internos se redistribuem automaticamente quando você adiciona, remove ou redimensiona itens. É o recurso que separa designers iniciantes de profissionais.
💡 Analogia simples
Auto Layout é como um elástico inteligente — se você adiciona mais cartões a uma lista, o contêiner cresce automaticamente. Se você remove, ele encolhe. Os espaçamentos entre os itens se mantêm consistentes sem que você precise ajustar manualmente.
⚙️Propriedades do Auto Layout
↕️ Direção (Direction)
Vertical: empilha itens (listas, menus). Horizontal: lado a lado (botões, breadcrumbs). Wrap: quebra para nova linha.
📏 Espaçamento (Gap)
Distância entre os itens internos. Pode ser igual para todos ou personalizado (horizontal e vertical diferentes).
📐 Padding
Espaço interno entre a borda do contêiner e o conteúdo. Pode ser igual nos 4 lados ou personalizado.
📦 Alinhamento (Alignment)
Como os itens se alinham no eixo principal (topo, centro, base) e no eixo cruzado (esquerda, centro, direita).
🔄 Resizing
Hug Contents: abraça o conteúdo. Fill Container: preenche o espaço disponível. Fixed: tamanho fixo.
🎨 Distribuição
Como o espaço extra é distribuído entre os itens quando há Fill Container (packed, space between).
🧪Onde usar Auto Layout
🔘 Botões
Texto + padding ajustam automaticamente
📋 Listas de Cards
Adicionar/remover itens sem quebrar o layout
📝 Formulários
Campos se ajustam ao tamanho do frame
/* Como aplicar Auto Layout no Figma */
1. Selecione o frame ou grupo de elementos
2. Pressione Shift + A (atalho) ou clique em "+" no painel Auto Layout
3. Ajuste direção, gap, padding e alinhamento no painel direito

/* Exemplo: Navbar com Auto Layout */
Navbar → Direction: Horizontal | Gap: 24px | Padding: 16px 120px
Menu Links → Direction: Horizontal | Gap: 32px | Align: Center
2 ✅ Finalização de Telas — O Checklist do Profissional
🎯 O que significa "finalizar" uma tela?
Finalizar uma tela vai além de apenas desenhar os elementos. Significa garantir que tudo está pronto para entrega: componentes nomeados, camadas organizadas, cores consistentes, textos reais, prototipagem funcional e documentação adequada. É o acabamento profissional que separa um projeto amador de um portfolio de respeito.
✅Checklist de Finalização de Telas
🎨 Cores e Estilos
✅ Todas as cores vêm do Style Guide (não há cores "soltas")
✅ Textos usam Text Styles (não há font-size manual)
✅ Contraste de cores validado (WCAG AA)
🧩 Componentes
✅ Elementos repetidos são componentes (não grupos soltos)
✅ Variantes configuradas (primary, secondary, disabled)
✅ Instâncias usadas (não cópias desconectadas)
📐 Layout e Grid
✅ Todos elementos alinhados ao grid
✅ Auto Layout aplicado em listas e containers
✅ Espaçamentos seguem a escala do projeto (ex: 8px)
📝 Conteúdo
✅ Textos reais (não Lorem Ipsum genérico)
✅ Imagens finais (não placeholders cinza)
✅ Ícones da biblioteca oficial do projeto
🔗 Prototipagem
✅ Todos botões e links têm conexões
✅ Animações configuradas (Smart Animate)
✅ Fluxos de erro e sucesso contemplados
📂 Organização
✅ Camadas nomeadas (não "Rectangle 237")
✅ Páginas organizadas por funcionalidade
✅ Componentes na página Assets/Components
💡 Regra de ouro da finalização: Antes de entregar, faça a "pergunta do desenvolvedor": "Se eu recebesse este arquivo para codar, eu saberia exatamente o que fazer sem precisar perguntar nada?" Se a resposta for não, ainda há trabalho a fazer.
3 📂 Organização do Posto de Trabalho — Fluxo Profissional
🎯 Por que organizar o posto de trabalho?
Um arquivo Figma desorganizado é como uma oficina com ferramentas espalhadas — você perde tempo procurando, comete erros e entrega com menos qualidade. A organização do posto de trabalho aumenta a produtividade, facilita a colaboração em equipe e torna a manutenção do projeto muito mais simples.
📂Estrutura de Páginas Recomendada
📄 Páginas do Arquivo Figma
├── 🎨 Style Guide (cores, tipografia, ícones)
├── 🧩 Components (botões, cards, inputs, modais)
├── 📱 Mobile (todas as telas mobile)
├── 💻 Desktop (todas as telas desktop)
├── 🔗 Prototype (fluxos navegáveis organizados)
└── 🗑️ Archive (versões antigas, rascunhos)
📐Convenções de Nomenclatura
✅ Nomes de Frames
Home/v1 · Checkout/Step2 · Modal/Confirm
Formato: Funcionalidade/Variação
✅ Nomes de Componentes
Button/Primary/Default · Card/Product
Formato: Tipo/Variação/Estado
✅ Nomes de Camadas
Header · Hero Image · CTA Button
Nomes descritivos, nunca "Rectangle 237"
✅ Ordenação de Camadas
Da mais externa (container) para a mais interna (texto). Use Auto Layout sempre que possível.
🛠️Ferramentas e Atalhos para Produtividade
⌨️ Atalhos Essenciais
Shift + A → Auto Layout
Ctrl + Alt + K → Criar Componente
Ctrl + G → Agrupar em Frame
Ctrl + R → Renomear camada
Ctrl + / → Buscar comandos
🔌 Plugins Úteis
• Rename It → Renomear em lote
• Similayer → Selecionar camadas similares
• Clean Document → Limpar estilos não usados
• Design Lint → Verificar consistência
4 🧪 Exemplo Prático — Organização de um Projeto Real
Veja como seria a estrutura de um projeto profissional de landing page no Figma, do início ao fim:
1Configure o arquivo base
Crie as páginas: Style Guide, Components, Mobile, Desktop, Prototype. Defina grids para cada breakpoint.
2Crie os componentes base
Button, Input, Card, Navbar, Footer como componentes principais. Adicione variantes. Use Auto Layout em todos.
3Monte as telas (Mobile First)
Comece pelo mobile (375px). Use componentes arrastando do Assets. Ajuste textos e imagens com conteúdo real.
4Adapte para Desktop
Duplique as telas mobile, mude para frame Desktop (1440px), ajuste layout para 12 colunas. Componentes se adaptam com Auto Layout.
5Prototipe e Revise
Conecte os fluxos na página Prototype. Revise cores, tipografia, alinhamentos. Verifique o checklist de finalização.
💡 Dica de ouro: Reserve os últimos 30 minutos de cada sessão de design apenas para organizar: renomear camadas, limpar estilos não usados, revisar componentes e garantir que tudo está no lugar. Isso evita o acúmulo de bagunça e torna a entrega final muito mais rápida.
5 ✅ Boas Práticas e Hábitos de um Designer Profissional
✅ O que fazer
✅ Use Auto Layout em todos os componentes
✅ Nomeie camadas assim que criá-las
✅ Mantenha Style Guide e Components em páginas separadas
✅ Use a escala de 8px para todos os espaçamentos
✅ Faça commit semântico (versões nomeadas)
✅ Revise o checklist de finalização antes de entregar
✅ Peça feedback de um desenvolvedor sobre o arquivo
✅ Arquive versões antigas na página Archive
❌ O que evitar
❌ Camadas com nomes genéricos ("Rectangle 237")
❌ Ajustar posições manualmente sem Auto Layout
❌ Cores e fontes fora do Style Guide
❌ Deixar "para depois" a organização
❌ Criar componentes sem variantes (muitos soltos)
❌ Ignorar o grid (elementos "flutuando")
❌ Entregar sem revisar o protótipo
❌ Misturar rascunhos com telas finais na mesma página
