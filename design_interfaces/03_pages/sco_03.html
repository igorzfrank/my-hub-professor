1 🔗 Conexões de Artboards — A Espinha Dorsal do Protótipo
🎯 O que são Conexões de Artboards?
Conexões de artboards são as ligações lógicas que você estabelece entre as telas do seu protótipo. Cada conexão define de onde o usuário parte (gatilho), para onde ele vai (destino) e como essa transição acontece (animação). É isso que transforma um conjunto de telas estáticas em uma experiência navegável.
💡 Analogia simples
Conectar artboards é como construir estradas entre cidades. Cada cidade é uma tela (artboard). As estradas são as conexões. E o tipo de estrada (asfalto, terra, pedágio) é a transição — define a experiência da viagem. Sem estradas, as cidades são ilhas isoladas.
⚙️Anatomia de uma Conexão no Figma
🎯 Gatilho (Trigger)
O que o usuário faz para ativar a navegação. Ex: On Click, While Hovering, After Delay, Key Press, Drag.
🎬 Ação (Action)
O que acontece quando o gatilho é ativado. Ex: Navigate to, Open overlay, Swap overlay, Back, Close.
📱 Destino (Destination)
Para qual frame o usuário será levado. Pode ser outra tela, um overlay ou voltar à tela anterior.
✨ Animação (Animation)
Como a transição acontece visualmente. Ex: Instant, Dissolve, Smart Animate, Slide, Push, Move.
📂Tipos de Ações de Navegação
Navigate To
Vai para outra tela. A mais comum. Substitui a tela atual.
Open Overlay
Sobrepoe um frame sobre a tela atual. Para modais, tooltips, dropdowns.
Swap Overlay
Troca um overlay por outro. Para navegação dentro de modais.
Back
Volta para a tela anterior. Mantém o histórico de navegação.
Close Overlay
Fecha o overlay e retorna à tela anterior.
Scroll To
Rola para um ponto específico da mesma tela.
💡 Como conectar no Figma: Selecione o elemento → clique na bolinha azul (⚪) que aparece ao lado → arraste até o frame de destino → no painel direito, configure gatilho, ação e animação. A seta azul indica a conexão estabelecida.
2 🎬 Transições Lógicas — A Alma da Navegação Fluida
🎯 O que são Transições Lógicas?
Transições lógicas são animações que fazem sentido para o contexto da navegação. Elas não são apenas "efeitos bonitos" — elas comunicam relações espaciais entre as telas. Por exemplo: um modal sobe de baixo (vem de uma ação), um detalhe desliza da direita (avanço na hierarquia), um menu aparece com fade (sobreposição temporária).
💡 Analogia simples
Transições são como as portas entre cômodos de uma casa. Uma porta de correr (slide) faz sentido para um armário. Uma porta que abre para dentro (push) faz sentido para um quarto. Uma porta sanfonada faz sentido para uma despensa. Cada tipo de porta comunica o tipo de espaço que está atrás dela.
🎬Tipos de Transição e Quando Usar
⚡ Instant— Sem animação
Mudança imediata. Use para: troca de abas, filtros, ordenação de lista.
🌫️ Dissolve— Fade entre telas
Uma tela desaparece enquanto a outra aparece. Use para: transições entre páginas não relacionadas, onboarding.
➡️ Slide— Desliza em uma direção
A tela desliza de fora para dentro. Use para: avançar/retroceder em fluxos sequenciais, carrosséis.
📤 Push— Empurra a tela atual
A nova tela empurra a atual para fora. Use para: hierarquia de navegação (ex: Home → Produto → Detalhe).
📦 Move— Move da posição original
O elemento se move de sua posição atual para o destino. Use para: expandir cards, abrir detalhes.
✨ Smart Animate— Transição inteligente
Anima automaticamente diferenças entre frames com mesmo nome de camada. Use para: microinterações, toggle, expansão de cards.
💡 Regra de ouro: A transição deve reforçar a relação entre as telas. Se o usuário está "indo mais fundo" na hierarquia (Home → Categoria → Produto), use Push da direita. Se está "voltando", use Push da esquerda. Se é uma ação temporária (modal), use Slide de baixo. A direção comunica a relação espacial.
3 🧭 Simulação de Rotas do Usuário — Mapeando Jornadas Completas
🎯 O que é uma Rota do Usuário?
Uma rota do usuário é o caminho completo que uma pessoa percorre para concluir uma tarefa no seu produto. No protótipo, simular rotas significa conectar múltiplas telas em sequência lógica, incluindo não apenas o "caminho feliz" (happy path), mas também desvios, erros e caminhos alternativos.
🗺️Tipos de Rotas que Você Deve Simular
😊 Happy Path (Caminho Feliz)
O fluxo ideal, onde tudo dá certo. Ex: Login → Home → Produto → Carrinho → Checkout → Confirmação. É o fluxo principal que você apresenta ao cliente.
🔄 Alternate Path (Caminho Alternativo)
Variações do fluxo principal. Ex: Login com Google em vez de e-mail, pagamento com PIX em vez de cartão.
⚠️ Error Path (Caminho de Erro)
O que acontece quando algo falha. Ex: senha incorreta, cartão recusado, produto indisponível, conexão perdida.
🔙 Edge Cases (Casos Limite)
Situações extremas. Ex: carrinho vazio, lista de resultados zerada, primeiro acesso sem dados, limite máximo atingido.
📋Exemplo Prático — Rota Completa de Compra
FLUXO PRINCIPAL — Happy Path
Home → Categoria → Produto → Carrinho → Checkout → Confirmação

FLUXO ALTERNATIVO — Login Social
Home → Login → [Google] → Home (logado)

FLUXO DE ERRO — Cartão Recusado
Checkout → Pagamento → ❌ Recusado → Tentar novamente → ✅ Aprovado → Confirmação

EDGE CASE — Carrinho Vazio
Carrinho → "Seu carrinho está vazio" → [Continuar Comprando] → Home
💡 Dica para o Figma: Organize os fluxos em páginas separadas ou use seções com títulos no canvas. Ex: crie uma página "Happy Path", outra "Error Paths", outra "Edge Cases". Isso mantém o arquivo navegável e profissional.
4 🧪 Exemplo Prático — Montando um Protótipo Navegável Completo
📱 App de Delivery — Conexões e Fluxos
Veja como seria a estrutura de conexões de um app de delivery, com as transições lógicas para cada tipo de navegação:
🏠 Home → Categoria (Push Right)
O usuário avança na hierarquia. A nova tela vem da direita, empurrando a anterior. Gatilho: On Click no card da categoria. Animação: Push 300ms.
📋 Produto → Carrinho (Slide Up Overlay)
O carrinho é um overlay que sobe de baixo — ação temporária. Gatilho: On Click em "Adicionar". Animação: Move 300ms from bottom.
💳 Checkout → Pagamento (Smart Animate)
Seleção de método de pagamento com animação suave entre opções. Gatilho: On Click na opção. Animação: Smart Animate 200ms.
✅ Confirmação → Home (Dissolve + Back)
Pedido confirmado, volta ao início com fade. Gatilho: After Delay 2s. Animação: Dissolve 500ms.
❌ Erro no Pagamento → Modal de Erro (Open Overlay)
Modal centralizado com fundo escuro. Gatilho: On Click em "Pagar" (quando cartão recusado). Animação: Dissolve 200ms.
💡 Regra de ouro do protótipo: "Se o usuário pode fazer, deve estar no protótipo." Cada botão deve ter uma conexão. Cada estado de erro deve ser mapeado. Cada caminho alternativo deve ser navegável. Um protótipo incompleto gera perguntas desnecessárias do desenvolvedor e do cliente.
5 ✅ Checklist e Boas Práticas
✅ O que verificar nas conexões
✅ Todo botão clicável tem uma conexão
✅ Modais têm entrada (Open Overlay) e saída (Close Overlay)
✅ A direção das transições faz sentido lógico
✅ Telas de erro e sucesso estão conectadas
✅ Há um caminho de volta para a Home de qualquer tela
✅ O tempo de animação não é muito longo (200-400ms)
✅ Edge cases estão mapeados (vazio, erro, limite)
❌ Erros comuns em protótipos
❌ Botões sem conexão (dead ends — becos sem saída)
❌ Transições que não fazem sentido (ex: modal vindo de cima)
❌ Apenas o happy path — sem tratamento de erro
❌ Animações muito longas (mais de 500ms)
❌ Falta de consistência: mesma ação, transições diferentes
❌ Esquecer de conectar o botão "Voltar"
❌ Overlays sem opção de fechar
🧠 Perguntas para validar seu protótipo
1. Se eu clicar em cada botão, todos levam a algum lugar?
2. As animações reforçam ou confundem a relação entre as telas?
3. Se eu cometer um erro (ex: senha errada), o protótipo mostra o que acontece?
4. Consigo voltar à tela inicial de qualquer ponto do fluxo?
5. As transições estão consistentes (mesmo tipo para mesma hierarquia)?
