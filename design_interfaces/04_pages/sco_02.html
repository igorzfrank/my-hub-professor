1 📱 MOBILE FIRST — Comece pelo Menor, Escale com Inteligência
🎯 O que é Mobile First no Figma?
Mobile First é uma filosofia de design onde você começa projetando para a tela menor (smartphone 375px) e, depois de finalizado, expande o layout para tablets e desktops. No Figma, isso significa criar primeiro o frame mobile com todo o conteúdo essencial e, em seguida, duplicar e adaptar para tamanhos maiores.
💡 Analogia simples
Mobile First no Figma é como esculpir uma estátua. Você começa com um bloco pequeno (frame mobile) contendo apenas a forma essencial. Depois, duplica e adiciona detalhes (tablet). Por fim, polimento e acabamento (desktop). O contrário — começar com um frame gigante e tentar "cortar pedaços" para caber no mobile — é muito mais trabalhoso.
⭐Por que Mobile First no Figma?
📈 Tráfego Mobile
Mais de 60% do tráfego web vem de dispositivos móveis. O mobile não é o futuro — é o presente.
🎯 Foco no Essencial
A tela pequena força você a priorizar. Se não cabe no mobile, talvez não seja tão importante assim.
🧠 Escalabilidade
Adicionar elementos é mais fácil que removê-los. Expandir para desktop é natural; comprimir para mobile é doloroso.
⚡ Produtividade
Com Auto Layout e Constraints, o Figma adapta automaticamente muitos elementos ao redimensionar frames.
📱 Fluxo Mobile First (Recomendado)
1. Crie o frame Mobile (375px)
2. Projete tudo no mobile
3. Duplique para Tablet (768px)
4. Ajuste o layout
5. Duplique para Desktop (1440px)
6. Refine o layout final
🖥️ Fluxo Desktop First (Evitar)
1. Crie o frame Desktop (1440px)
2. Projete tudo no desktop
3. Tente comprimir para Tablet
4. Sofra para caber no Mobile
5. Retrabalho e frustração
2 📏 BREAKPOINTS — Os Pontos de Quebra no Figma
🎯 O que são Breakpoints no Figma?
No Figma, Breakpoints são os tamanhos de frame específicos para os quais você projeta. Em vez de criar telas para cada dispositivo do mercado, você trabalha com 3 a 4 tamanhos estratégicos que cobrem a grande maioria dos usuários. O Figma facilita isso com frames pré-definidos e grids configuráveis.
📐Breakpoints Estratégicos no Figma
📱 Mobile
375 × 812px
4 colunas · Margin 16px
Gutter 16px
📋 Tablet
768 × 1024px

Gutter 24px
💻 Desktop
1440 × 900px
12 colunas · Margin 120px
Gutter 24px
🖥️ Wide

12 colunas · Margin 160px
Gutter 32px
⚙️Como configurar Grids no Figma para cada Breakpoint
Passo a passo no Figma
1. Selecione o Frame → 2. Painel direito → Layout Grid → 3. Clique no + → 4. Escolha Grid → 5. Defina colunas, margin e gutter → 6. Ajuste a cor e opacidade da grade para visualização
💡 Regra de ouro: Defina breakpoints por necessidade do conteúdo, não por dispositivos específicos. Redimensione o frame no Figma e observe onde o layout "quebra". Ajuste ali. Isso garante que seu design funcione em qualquer tela, presente ou futura.
3 🔧 Técnicas de Adaptação de Layouts no Figma
🎯 Como adaptar layouts sem perder a essência?
Adaptar um layout para diferentes telas no Figma não é apenas redimensionar frames. É preciso repensar a organização dos elementos, usar Auto Layout e Constraints para automatizar a adaptação e, em alguns casos, criar variações específicas de componentes para cada breakpoint.
1Auto Layout — A Ferramenta Principal de Responsividade
O Auto Layout faz com que frames e componentes se ajustem automaticamente ao conteúdo. Configure direção, gap, padding e alinhamento para que os elementos se reorganizem sozinhos quando o frame for redimensionado.
Wrap
Itens quebram para nova linha automaticamente
Fill Container
Elementos preenchem o espaço disponível
Hug Contents
Frame abraça o tamanho do conteúdo
Space Between
Distribui espaço igualmente entre itens
2Constraints — Fixar Elementos nas Bordas
As Constraints definem como um elemento se comporta quando o frame pai é redimensionado. Use para fixar botões nas laterais, centralizar títulos ou fazer barras acompanharem a largura da tela.
Left & Right
Estica horizontalmente
Center
Mantém centralizado
Right
Fixo na direita
Top & Bottom
Estica verticalmente
Scale
Redimensiona proporcionalmente
Fix
Posição fixa, não move

4 🧪 Exemplo Prático — Adaptação de uma Landing Page
📱 Mobile (375px) → 💻 Desktop (1440px)
🧭 Header / Navbar
Mobile: Logo + Hamburger menu (menu escondido em overlay). Tablet: Logo + 3 links visíveis + CTA. Desktop: Logo + 5 links + CTA + ícones. Técnica: Ocultar/Mostrar + Auto Layout com Space Between.
🦸 Hero Section
Mobile: Título + texto + CTA empilhados verticalmente. Imagem abaixo. Desktop: Texto à esquerda (6 colunas), imagem à direita (6 colunas). Técnica: Auto Layout (direção Vertical → Horizontal).
🛍️ Grid de Produtos
Mobile: 1 coluna (cards empilhados). Tablet: 2 colunas. Desktop: 3-4 colunas. Técnica: Auto Layout com Wrap + definir largura mínima/máxima dos cards.
📋 Footer
Mobile: Links empilhados, 1 coluna. Tablet: 2 colunas de links. Desktop: 4 colunas lado a lado. Técnica: Auto Layout com Wrap nos grupos de links.
💡 Fluxo de trabalho no Figma: 1. Crie o frame Mobile (375px) → 2. Projete todos os componentes com Auto Layout → 3. Duplique o frame e mude para Tablet (768px) → 4. Ajuste direções de Auto Layout e visibilidade de camadas → 5. Duplique para Desktop (1440px) → 6. Refine alinhamentos e espaçamentos → 7. Teste redimensionando os frames para verificar a adaptação.
5 ✅ Boas Práticas e Checklist de Responsividade no Figma
✅ Checklist de Responsividade
✅ O layout funciona no frame de 375px (mobile mínimo)
✅ Todos os botões têm touch target ≥ 44×44px
✅ Textos são legíveis (mín. 14px no mobile)
✅ Auto Layout está aplicado em todos os containers
✅ Constraints estão configuradas nos elementos internos
✅ Menu mobile (hamburger) tem overlay funcional
✅ Grid está configurado para cada breakpoint
✅ Testado redimensionando frames no Figma
❌ Erros Comuns
❌ Texto muito pequeno no mobile (< 14px)
❌ Botões muito próximos (toque acidental)
❌ Não usar Auto Layout (ajustes manuais dolorosos)
❌ Esquecer de configurar Constraints
❌ Usar pixels fixos em vez de Fill Container
❌ Não testar redimensionando o frame
❌ Criar um frame para cada dispositivo (muitos frames)
❌ Ignorar orientação landscape no mobile
🧠 Regras de Ouro da Responsividade no Figma
1. Comece sempre pelo mobile (frame 375px).
2. Use Auto Layout em absolutamente todos os containers e componentes.
3. Configure Constraints para elementos que devem grudar nas bordas ou centralizar.
4. Defina poucos breakpoints estratégicos (3-4 no máximo).
5. Pense em "componentes adaptáveis" com variantes para cada tamanho de tela.
6. Teste redimensionando os frames — não confie apenas no visual estático.
💡 Dica final: Use o plugin "Responsive" no Figma para visualizar rapidamente como seu layout se comporta em diferentes tamanhos de tela sem precisar criar frames manualmente. Ele simula a experiência de redimensionamento do navegador diretamente no Figma.
