1 🧭 ESTRUTURA DE NAVEGAÇÃO
🎯 O que é?
Estrutura de Navegação é a organização hierárquica e lógica de telas, conteúdos e funcionalidades de um produto digital. Ela define como o usuário se move entre as páginas e onde encontra o que precisa.
💡 Analogia simples
É como a planta baixa de um shopping – mostra onde ficam as lojas, escadas, elevadores e banheiros. Uma boa navegação evita que o usuário se "perca" no seu produto.
⭐Por que é fundamental?
✅ Reduz frustração e abandono
✅ Aumenta eficiência do usuário
✅ Melhora taxa de conversão
✅ Facilita aprendizado do sistema
✅ Impacta diretamente o SEO (para sites)
2 📂 Tipos de Estrutura de Navegação
🌳1. Hierárquica (ou em árvore)
A mais comum. Conteúdo organizado em níveis (pai → filho).
Home
├── Produtos
│ ├── Eletrônicos
│ └── Roupas
├── Sobre
└── Contato
🌐2. Global (ou principal)
Presente em todas as telas. Links principais do sistema.
Exemplo: Menu fixo no topo do site.
📑3. Local (ou secundária)
Navegação dentro de uma seção específica.
Exemplo: Submenu de "Eletrônicos" mostrando Celulares, Notebooks, TVs.
🔗4. Contextual
Links dentro do conteúdo que levam a conteúdo relacionado.
Exemplo: "Você também pode gostar" ou links no texto do blog.
🔍5. Facetada (ou filtros)
Permite combinar múltiplos critérios para refinar busca.
Exemplo: Amazon – filtrar por preço, marca, tamanho, cor.
➡️6. Sequencial (ou passo a passo)
Guia o usuário por etapas lineares.
Exemplo: Checkout de compra (Carrinho → Endereço → Pagamento → Confirmação).
3 🧩 Elementos Comuns de Navegação
📋 Menu (navbar)
Acesso às principais seções
🍞 Breadcrumb
Home > Produtos > Tênis
📑 Sidebar
Menu lateral para navegação secundária
📌 Footer
Links úteis no rodapé
🍔 Hamburger menu
Menu oculto (comum em mobile)
📑 Tabs
Alternar entre conteúdos relacionados
🔘 Botões de ação
Próximo/voltar, salvar, cancelar
🔗 Links internos
Conectam conteúdos relacionados
4 📱 Exemplo Prático e Princípios
🏦 Exemplo — App de Banco
🏦 Home (Global)
├── 📊 Saldo (Local)
│ ├── Extrato
│ └── Cartões
├── 💸 Transferências (Sequencial)
│ Passo 1: Conta
│ Passo 2: Valor
│ Passo 3: Confirmação
├── 📱 Pagamentos (Contextual)
│ "Contas próximas ao vencimento"
└── ⚙️ Configurações (Global)
✅ Princípios de uma boa navegação
1. Previsibilidade — Links fazem o esperado
2. Consistência — Mesmo padrão em todas as telas
3. Simplicidade — Menos opções é melhor (Lei de Hick)
4. Visibilidade — Opções importantes sempre à vista
5. Feedback — Indicar onde o usuário está
6. Acessibilidade — Funcionar com teclado, leitores de tela
❌ Erros comuns (anti-padrões)
📋 Menu com 15 itens — Sobrecarrega
🔗 Links criativos demais — "Saiba mais" não diz o que é
🙈 Esconder navegação — Ícone vago para "menu"
⏬ Profundidade excessiva — 5+ cliques para encontrar algo
📍 Sem indicação de localização — Usuário não sabe onde está
5 🗺️ SITEMAP — O Mapa do Site
🎯 O que é Sitemap?
Sitemap (ou mapa do site) é uma representação visual ou documental da estrutura hierárquica de um site ou aplicativo. Ele mostra todas as páginas, seções e como elas se relacionam entre si, como um organograma.
💡 Analogia
Se a Estrutura de Navegação é a "planta baixa do shopping", o Sitemap é o mapa que você vê no totem de informações – mostrando todas as lojas, escadas e saídas de uma só vez.
🏪Exemplo de Sitemap — Loja Virtual
Home
├── Produtos
│ ├── Masculino (Filtros: tamanho, cor, preço)
│ ├── Feminino
│ └── Infantil
├── Promoções (Local: cupons, frete grátis)
├── Minha Conta
│ ├── Meus Pedidos (Sequencial: Acompanhar)
│ ├── Endereços
│ └── Formas de Pagamento
├── Ajuda
│ ├── Trocas e Devoluções
│ └── FAQ
└── Carrinho (Sequencial: Checkout 3 etapas)
🧠 Relação com outros conceitos: A Estrutura de Navegação é PARTE da Arquitetura da Informação. A UX usa a navegação como MEIO para boa experiência. A UI dá suporte VISUAL à navegação.
6 🚶 FLUXO DO USUÁRIO — User Flow
🎯 O que é Fluxo do Usuário?
Fluxo do Usuário (User Flow) é a representação visual do caminho completo que um usuário percorre dentro de um produto digital para realizar uma tarefa específica, incluindo cada passo, decisão, interação e emoção ao longo do processo.
💡 Analogia
Se o Sitemap é o mapa do metrô (todas as estações), o Fluxo do Usuário é a viagem REAL que uma pessoa faz – ela pode pegar o trem errado, descer na estação errada, voltar, pedir informação, ou até desistir.
✅ O que DEVE incluir
• Ponto de entrada
• Todas as ações (cliques, scroll)
• Decisões do usuário
• Caminhos alternativos
• Pontos de erro
• Pontos de sucesso
• Pontos de abandono
• Tempo estimado
❌ O que NÃO deve incluir
• Detalhes de UI (cores, fontes)
• Código de programação
• Todas as páginas do site
• Elementos que não afetam o usuário
• Especificações técnicas
• Lógica de backend
📂Tipos de Fluxo de Usuário
Task flow
Uma tarefa específica
Happy path
Caminho perfeito (sem erros)
Alternate path
Caminhos diferentes
Error path
O que acontece nos erros
Onboarding flow
Primeiro uso do app

↓
[Abre app de delivery]
↓
◇ Já tem cadastro?
├── Sim → [Login]
└── Não → [Cadastro] → [Confirma e-mail]
↓
[Busca restaurante]
↓
◇ Por filtro? Por busca? Por mapa?
↓
[Escolhe restaurante] → [Adiciona itens]
↓
◇ Atingiu valor mínimo?
├── Sim → [Finaliza]
└── Não → ⚠️ "Faltam R$10" → [Adiciona mais]
↓
[Escolhe pagamento]
↓
◇ Pagamento aprovado?
├── Sim → ✅ "Pedido confirmado"
└── Não → ❌ "Recusado" → [Tenta outra forma]
↓
[Acompanha entrega] → [Recebe] → 🎉 Sucesso!
✅ Por que Fluxo do Usuário é essencial? Ele revela onde o usuário desiste, onde erra e onde tem atrito. Com esse mapa, a equipe de UX pode redesenhar etapas problemáticas antes mesmo de escrever uma linha de código.
