1 🧠 UX — User Experience (Experiência do Usuário)
🎯 O que é UX?
UX (User Experience) é sobre como a pessoa se sente ao usar um produto. O foco é resolver problemas de forma eficiente e agradável. Não se trata apenas do que aparece na tela, mas de toda a jornada do usuário — desde o primeiro contato até a conclusão da tarefa.
💡 Resumo para memorizar
UX é sobre funcionar bem e fazer sentido. Um produto com boa UX resolve o problema do usuário sem atritos, de forma quase invisível.
🧩Conceitos básicos de UX
♿ Usabilidade
O produto é fácil de usar? O usuário consegue fazer o que quer sem dificuldade?
🌍 Acessibilidade
Pessoas com diferentes habilidades (visuais, motoras, cognitivas) conseguem usar?
📂 Arquitetura da Informação
Como o conteúdo está organizado — menus, categorias, fluxo de navegação.
🔍 Pesquisa com Usuários
Entender necessidades reais antes de projetar — entrevistas, testes, observação.
🛤️ Fluxo do Usuário
O caminho que a pessoa percorre para completar uma tarefa dentro do sistema.
🔄 Consistência
Padrões previsíveis ajudam o usuário a não se perder — ícones, termos, posições.
📢 Feedback
O sistema precisa responder às ações — botão muda ao clicar, loading, confirmações.
2 🎨 UI — User Interface (Interface do Usuário)
🎯 O que é UI?
UI (User Interface) é a parte visual e interativa — aquilo que o usuário vê e toca. Envolve todos os elementos gráficos que compõem a tela: botões, cores, fontes, ícones, espaçamentos e imagens.
💡 Resumo para memorizar
UI é sobre aparência e interação visual. Uma boa interface é bonita, organizada e convida o usuário a interagir.
🧩Conceitos básicos de UI
📐 Layout
Organização dos elementos na tela — grid, colunas, posicionamento.
🔤 Tipografia
Escolha e uso das fontes — hierarquia, legibilidade, personalidade.
🎨 Cores
Estética + comunicação — cada cor evoca emoções e significados diferentes.
👁️ Hierarquia Visual
O que chama mais atenção primeiro — tamanho, cor, posição, contraste.
⬜ Espaçamento (Whitespace)
Evita poluição visual — respiro entre elementos melhora a compreensão.
🧱 Componentes
Botões, ícones, menus, formulários — as "peças" reutilizáveis da interface.
📱 Responsividade
Funciona bem em celular, tablet e desktop — adaptação fluida.
3 🤝 Como UI e UX Trabalham Juntos
🧠
UX — O Cérebro
Pesquisa, estratégia, arquitetura, fluxos, testes. Responde: "Isso funciona? Faz sentido?"
🎨
UI — A Pele
Cores, fontes, ícones, botões, animações. Responde: "Isso é bonito? Convida a usar?"
📱 Exemplo prático — App de Delivery
🧠 UX (Experiência):
• Conseguir pedir comida em poucos passos
• Busca inteligente por restaurante ou prato
• Acompanhar o pedido em tempo real
• Salvar endereços e formas de pagamento
🎨 UI (Interface):
• Botões claros e grandes (fáceis de tocar)
• Cores agradáveis e fotos apetitosas
• Ícones intuitivos para cada categoria
• Feedback visual ao adicionar ao carrinho
⚠️ Cenários de desequilíbrio
Boa UI + UX ruim: App bonito mas confuso. O usuário admira, mas não consegue usar e desiste.
Boa UX + UI ruim: App funcional mas feio e desorganizado. O usuário usa, mas não confia nem recomenda.
4 📋 BRIEFING — O GPS do Projeto
🎯 O que é Briefing?
Briefing é um documento ou reunião que transmite informações essenciais e claras sobre um projeto. Ele serve como guia para que todos os envolvidos (cliente e equipe) estejam alinhados sobre o que precisa ser feito, por quê, para quem, quando e como.
💡 Analogia
Briefing é o GPS do projeto: ele mostra o ponto de partida (situação atual), o destino (objetivo) e o caminho (restrições de prazo, verba e tom), mas deixa o volante com o piloto (a equipe executora) para decidir as manobras criativas.
⭐Principais conceitos e características
🚀 Diretriz Inicial
Ponto de partida de qualquer projeto. Sem briefing, as ações tendem a ser descoordenadas.
🤝 Ferramenta de Alinhamento
Evita ruídos de comunicação. Cliente e equipe com a mesma compreensão.
🎯 Foco no Problema
Explica qual problema de comunicação ou negócio a ação precisa resolver.
📝 Documento Dinâmico
Pode ser ajustado conforme novas informações surgem, com comunicação clara.
📄 Contrato Não Técnico
Acordo de intenções e expectativas. Ajuda a medir o sucesso no final.
❓As 6 perguntas que um briefing deve responder
Conceito
Pergunta que responde
🎯 Objetivo
O que exatamente queremos alcançar? (meta mensurável)
👥 Público-alvo
Para quem essa mensagem é direcionada? (quem são, o que pensam)
💬 Mensagem principal
O que precisa ser dito para convencer esse público?
🎭 Tom e voz
Como devemos falar? (sério, humorado, técnico, emocional)
📦 Entregáveis
Quais são os resultados concretos esperados? (vídeo, site, campanha?)
⏰ Prazo e orçamento
Até quando e com quanto dinheiro?
📂Tipos de Briefing
🎨 Briefing Criativo
Focado em comunicação, design e publicidade. Dá liberdade para criar dentro de limites claros.
💰 Briefing de Vendas
Usado para preparar equipe comercial antes de reunião com cliente potencial.
⚙️ Briefing de Projeto (Técnico)
Mais detalhado, usado em engenharia ou software, com especificações funcionais.
🔄 Briefing Reverso
O executor escreve e envia para o cliente aprovar — garante que entendeu o pedido.
5 📝 Exemplo Prático — Briefing de App de Delivery
Conceito
Resposta
🎯 Objetivo
Aumentar pedidos via app em 30% em 6 meses, reduzindo abandono de carrinho.
👥 Público-alvo
Jovens de 18 a 35 anos, urbanos, que pedem comida pelo menos 2x por semana.
💬 Mensagem principal
"Comida que você ama, entregue onde você estiver, em minutos."
🎭 Tom e voz
Jovem, descontraído, com toque de urgência. Usar gírias leves e emojis com moderação.
📦 Entregáveis
Protótipo navegável (UI), teste de usabilidade com 10 usuários (UX), versão final para desenvolvimento.
⏰ Prazo e orçamento
3 meses para protótipo, 6 meses para lançamento. Orçamento: R$ 150.000.
✅ Resultado: Com esse briefing, a equipe de UX/UI sabe exatamente onde focar — fluxo de pedido rápido (UX) com botões grandes e cores vibrantes (UI) para o público jovem. O sucesso será medido pela redução do abandono de carrinho.
