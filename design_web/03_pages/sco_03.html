1 🖼️ A Tag <img> — Imagens no HTML
🎯 O que é a tag <img>?
A tag <img> é uma tag auto-fechante (não tem tag de fechamento) usada para incorporar imagens em uma página HTML. É um dos elementos mais importantes da web — páginas sem imagens são menos atrativas e têm menor engajamento.
💡 Analogia simples
A tag <img> é como uma moldura digital — ela não contém a imagem dentro de si, mas aponta para o arquivo e o exibe na página. O atributo src é o "endereço" de onde a imagem está.
📝Sintaxe básica
<img
  src="caminho/para/imagem.jpg"
  alt="Texto descritivo da imagem"
  width="400"
  height="300"
  loading="lazy"
>
⚙️Principais atributos da tag <img>
src (source) — Obrigatório
Caminho para o arquivo de imagem. Pode ser local (/img/foto.jpg) ou URL externa.
alt (alternative text) — Essencial
Texto descritivo exibido quando a imagem não carrega. Fundamental para acessibilidade e SEO.
width e height — Dimensões
Define largura e altura em pixels. Ajuda o navegador a reservar espaço antes do carregamento.
loading="lazy" — Performance
Carrega a imagem apenas quando ela está prestes a aparecer na tela. Melhora a velocidade da página.
title — Tooltip opcional
Texto que aparece ao passar o mouse sobre a imagem. Não substitui o atributo alt.
srcset — Imagens responsivas
Define diferentes versões da imagem para diferentes tamanhos de tela (mobile, tablet, desktop).
📂Formatos de imagem para web
JPEG / JPG
Fotos, sem transparência, compressão com perda.
PNG
Com transparência, qualidade alta, arquivo maior.
WebP
Formato moderno, menor que JPEG/PNG, com transparência.
SVG
Vetorial, escala infinita, ideal para ícones e logos.
💡 Dica de performance: Sempre use imagens no tamanho exato em que serão exibidas. Não use uma imagem de 4000px de largura em um espaço de 400px — isso desperdiça banda e deixa o site lento.
2 ♿ Acessibilidade em Imagens — Por que o alt é tão importante?
🎯 O problema: nem todos "veem" imagens
Pessoas com deficiência visual usam leitores de tela (screen readers) que leem o conteúdo em voz alta. Quando encontram uma imagem, o leitor lê o atributo alt. Se ele estiver vazio ou ausente, o usuário não sabe o que há ali.
💡 Analogia simples
O atributo alt é como um amigo descrevendo uma foto para alguém que não pode vê-la. Se você diz "IMG_20240518_143021.jpg", a pessoa não entende nada. Se você diz "Cachorro golden retriever brincando no parque com uma bola vermelha", ela visualiza mentalmente a cena.
📝Como escrever um bom texto alternativo (alt)
✅ Regra 1 — Seja descritivo e específico
Descreva o conteúdo e a função da imagem. Exemplo: alt="Homem segurando um smartphone mostrando o aplicativo de delivery com a tela de pedidos"
✅ Regra 2 — Seja conciso (125 caracteres ou menos)
Leitores de tela geralmente cortam textos muito longos. Vá direto ao ponto.
✅ Regra 3 — Não comece com "Imagem de..." ou "Foto de..."
O leitor de tela já anuncia que é uma imagem. Ir direto à descrição economiza tempo do usuário.
✅ Regra 4 — Imagens decorativas usam alt vazio
Se a imagem é puramente decorativa (não transmite informação), use alt="". O leitor de tela ignorará a imagem.
✅ Regra 5 — Imagens com texto devem ter o texto no alt
Se a imagem contém texto importante (banner, logo), reproduza esse texto literalmente no atributo alt.
✅ Bons exemplos de alt
✅ alt="Cachorro golden retriever brincando com bola vermelha no parque"
✅ alt="Gráfico de barras mostrando crescimento de 25% nas vendas em 2024"
✅ alt="Logo da Empresa XYZ"
✅ alt="Botão de pesquisa com ícone de lupa"
❌ Maus exemplos de alt
❌ alt="foto1.jpg"
❌ alt="" (em imagem informativa)
❌ alt="Imagem de um cachorro brincando no parque em um dia ensolarado de verão" (muito longo)
❌ alt="imagem"
👥Quem se beneficia do atributo alt?
🦯 Pessoas cegas
Usam leitores de tela que vocalizam o texto alternativo.
👁️ Pessoas com baixa visão
Podem usar ampliadores e o alt complementa a compreensão.
🔍 Mecanismos de busca
O Google usa o alt para indexar imagens no Google Imagens.
📶 Conexões lentas
Quando a imagem não carrega, o alt é exibido no lugar.
🧠 Pessoas com deficiência cognitiva
O texto descritivo ajuda a processar o conteúdo visual.
🌐 Todos os usuários
Se a imagem quebrar, o alt mantém o contexto da página.
⚠️ Importante: O atributo alt é obrigatório por lei em muitos países (incluindo Brasil, via Lei de Inclusão) para sites governamentais e de empresas. Além disso, o Google penaliza sites sem acessibilidade.
3 🎬 Tags <video> e <audio> — Mídia no HTML5
🎬 A Tag <video>
Diferente da tag <img>, a tag <video> tem abertura e fechamento. Ela permite conteúdo alternativo entre as tags para navegadores que não suportam vídeo.
<video
  src="video.mp4"
  controls
  width="640"
  poster="capa.jpg"
>
  Seu navegador não suporta vídeo.
</video>
Atributos principais:
controls — Exibe controles (play, pause, volume)
autoplay — Inicia automaticamente (use com moderação)
muted — Começa sem som
loop — Repete ao final
poster — Imagem de capa antes da reprodução
🔊 A Tag <audio>
Funciona de forma muito similar ao vídeo. Ideal para podcasts, músicas e efeitos sonoros.
<audio controls>
  <source src="musica.mp3" type="audio/mpeg">
  Navegador não suporta áudio.
</audio>
Formatos suportados:
MP3 — O mais compatível, suporte universal
OGG — Formato livre e aberto
WAV — Sem compressão, arquivos grandes
Use múltiplos <source> para garantir compatibilidade com diferentes navegadores.
📂 Múltiplos formatos com <source>
Para máxima compatibilidade, ofereça o mesmo vídeo/áudio em diferentes formatos. O navegador usará o primeiro que souber reproduzir.
<video controls>
  <source src="video.webm" type="video/webm">
  <source src="video.mp4" type="video/mp4">
  Seu navegador não suporta vídeos.
</video>
♿ Acessibilidade em Vídeo e Áudio
📝 Legendas (track)
Use a tag <track> para adicionar legendas em formato VTT. Essencial para surdos.
📖 Transcrição
Forneça uma transcrição textual do áudio/vídeo para leitores de tela e indexação.
🔊 Audiodescrição
Para vídeos, uma faixa de áudio extra descrevendo cenas visuais para pessoas cegas.
⚠️ Evite autoplay
Vídeos com som automático desorientam usuários de leitores de tela e causam má experiência.
4 🔗 A Tag <a> — Links e Âncoras
🎯 O que é a tag <a>?
A tag <a> (anchor = âncora) é o elemento que torna a web uma teia interconectada. Ela cria hyperlinks — clicáveis que levam o usuário de uma página a outra, para uma seção da mesma página, para um e-mail ou para um telefone.
💡 Analogia simples
Links são como portas entre salas. Cada porta (link) conecta o cômodo atual (sua página) a outro cômodo (outra página ou site). Sem portas, cada sala seria uma ilha isolada.
📝Sintaxe básica e atributos
<a
  href="https://www.exemplo.com"
  target="_blank"
  rel="noopener noreferrer"
>
  Clique aqui para visitar o site
</a>
⚙️Atributos essenciais da tag <a>
href — Obrigatório
URL de destino. Pode ser absoluta (outro site), relativa (mesmo site), âncora (#secao), e-mail (mailto:), telefone (tel:).
target — Onde abrir
_self (mesma aba, padrão) ou _blank (nova aba).
rel — Relação (segurança)
Para links externos com target _blank, use rel="noopener noreferrer" por segurança.
download — Baixar arquivo
Força o download do link em vez de abri-lo. Pode definir nome do arquivo salvo.
🔗Tipos de links e seus prefixos
🌐 Link para outro site
href="https://google.com"
📄 Link para página interna
href="/sobre.html"
📧 Link para e-mail
href="mailto:email@exemplo.com"
📞 Link para telefone
href="tel:+5511999999999"
⚓ Link para seção da página
href="#contato"
📥 Link para download
href="arquivo.pdf" download
💡 Boas práticas para links: Use textos descritivos (não "clique aqui"), indique visualmente que é um link (cor, sublinhado), e mantenha consistência. Para links externos, avise o usuário com o atributo target="_blank" e rel="noopener".
5 🖼️ Tags Avançadas — <figure> e <picture>
📷 <figure> + <figcaption> — Imagem com legenda semântica
A tag <figure> envolve uma imagem (ou vídeo, gráfico, código) e a associa a uma legenda descritiva usando <figcaption>. O Google valoriza essa associação semântica.
<figure>
  <img src="grafico.jpg" alt="Gráfico de vendas 2024">
  <figcaption>Figura 1: Vendas trimestrais de 2024</figcaption>
</figure>
🖼️ <picture> — Imagens responsivas (Art Direction)
A tag <picture> permite definir diferentes versões da mesma imagem para diferentes tamanhos de tela. O navegador escolhe automaticamente a melhor versão.
<picture>
  <source media="(min-width: 768px)" srcset="grande.jpg">
  <img src="pequena.jpg" alt="Paisagem montanhosa">
</picture>
