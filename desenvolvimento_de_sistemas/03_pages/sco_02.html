Orientação a Objetos com JavaScript
Revisão completa dos quatro pilares da POO aplicados com JavaScript moderno: classes, herança, encapsulamento, abstração e polimorfismo.
Capacidades Trabalhadas

O que é Programação Orientada a Objetos?

A Programação Orientada a Objetos (POO) é um paradigma que organiza o software em torno de objetos — entidades que combinam dados (atributos) e comportamentos (métodos) — em vez de funções e lógica isolada.

Em JavaScript, a POO é implementada por meio de classes (introduzidas no ES6), protótipos e objetos literais. Entender esses conceitos é essencial para criar sistemas organizados, reutilizáveis e fáceis de manter.

Encapsulamento
Proteger e organizar dados internos de um objeto

O encapsulamento consiste em esconder os detalhes internos de um objeto e expor apenas o necessário. Em JavaScript, podemos usar campos privados (prefixo #) para proteger dados que não devem ser acessados diretamente.

É a técnica de esconder os detalhes internos de um objeto e expor apenas o que é necessário. Funciona como uma "cápsula" que protege os dados contra acessos indevidos, garantindo que o estado interno do objeto só mude através de métodos específicos (como getters e setters). 
 
  • Exemplo: Você usa um controle remoto para ligar a TV (interface), mas não precisa saber como os circuitos internos funcionam (detalhes encapsulados).
Sem Encapsulamento (problema)
Dados expostos — má prática 
class ContaBancaria {
  constructor(titular, saldo) {
    this.titular = titular;
    this.saldo = saldo; // Qualquer um pode alterar diretamente
  }
}

const conta = new ContaBancaria('João', 1000);
conta.saldo = -99999; // Problema: saldo negativo sem validação!
console.log(conta.saldo); // -99999
Com Encapsulamento (solução)
Usando campos privados com # 
class ContaBancaria {
  // Campos privados — só acessíveis dentro da classe
  #saldo;
  #titular;

  constructor(titular, saldoInicial) {
    this.#titular = titular;
    this.#saldo = saldoInicial;
  }

  get saldo() {
    return this.#saldo;
  }

  get titular() {
    return this.#titular;
  }

  depositar(valor) {
    if (valor <= 0) throw new Error('Valor inválido para depósito');
    this.#saldo += valor;
    return `Depósito de R$ ${valor} realizado. Saldo: R$ ${this.#saldo}`;
  }

  sacar(valor) {
    if (valor > this.#saldo) throw new Error('Saldo insuficiente');
    this.#saldo -= valor;
    return `Saque de R$ ${valor} realizado. Saldo: R$ ${this.#saldo}`;
  }
}

const conta = new ContaBancaria('João', 1000);
console.log(conta.saldo); // 1000 (getter público)
console.log(conta.depositar(500)); // Depósito de R$ 500. Saldo: R$ 1500
console.log(conta.sacar(200));   // Saque de R$ 200. Saldo: R$ 1300

// conta.#saldo = -99999; // ERRO! Campo privado inacessível
Conceitos do Encapsulamento
Os getters e setters servem para controlar o acesso às propriedades de um objeto, adicionando uma camada de lógica antes de ler ou escrever um dado. Eles funcionam como "porteiros" do encapsulamento.
  • Campos Privados (#): Prefixo # torna a propriedade inacessível fora da classe
  • Getters: Método get para leitura controlada de propriedades. Quando você quer apenas validar ou formatar uma propriedade (ex: formatar moeda, validar e-mail). É mais elegante e "limpo".
  • Setters: Método set para escrita controlada com validação. Quando a ação é complexa, envolve cálculos pesados ou efeitos colaterais (ex: salvarNoBancoDeDados(), processarPagamento()).
  • Métodos públicos: Interface segura para interagir com dados internos
2
Herança
Reutilizar e especializar comportamentos de outras classes

A herança permite que uma classe filha herde propriedades e métodos de uma classe pai, evitando repetição de código e criando hierarquias lógicas. Use a palavra-chave extends para herdar e super para acessar a classe pai.

Permite que uma classe (filha/subclasse) herde características e comportamentos de outra classe (pai/superclasse). Isso evita a repetição de código, pois você pode criar uma hierarquia onde classes mais específicas aproveitam o que já foi definido em classes mais genéricas. 
 
  • Exemplo: Uma classe Carro e uma classe Moto podem herdar de uma classe Veiculo atributos comuns como marca e modelo.
Sistema de Funcionários com Herança 
// Classe pai (genérica)
class Funcionario {
  constructor(nome, cpf, salarioBase) {
    this.nome = nome;
    this.cpf = cpf;
    this.salarioBase = salarioBase;
  }

  calcularSalario() {
    return this.salarioBase;
  }

  exibirInfo() {
    return `Funcionário: ${this.nome} | Salário: R$ ${this.calcularSalario()}`;
  }
}

// Classe filha: herda Funcionario e especializa
class Gerente extends Funcionario {
  constructor(nome, cpf, salarioBase, bonificacao) {
    super(nome, cpf, salarioBase);
    this.bonificacao = bonificacao;
  }

  calcularSalario() {
    return this.salarioBase + this.bonificacao;
  }

  aprovarFerias(funcionario) {
    return `${this.nome} aprovou férias de ${funcionario.nome}`;
  }
}

class Desenvolvedor extends Funcionario {
  constructor(nome, cpf, salarioBase, linguagens) {
    super(nome, cpf, salarioBase);
    this.linguagens = linguagens;
  }

  calcularSalario() {
    const bonus = this.linguagens.length * 200;
    return this.salarioBase + bonus;
  }
}

const ana   = new Gerente('Ana', '123.456.789-00', 6000, 2000);
const pedro = new Desenvolvedor('Pedro', '987.654.321-00', 4500, ['JS', 'Python', 'SQL']);

console.log(ana.exibirInfo());          // Funcionário: Ana   | Salário: R$ 8000
console.log(pedro.exibirInfo());        // Funcionário: Pedro | Salário: R$ 5100
console.log(ana.aprovarFerias(pedro));  // Ana aprovou férias de Pedro
console.log(pedro instanceof Funcionario); // true
extends
Declara que uma classe herda de outra. A filha recebe automaticamente todos os métodos públicos da pai.
super()
Chama o construtor da classe pai. Deve ser chamado antes de usar this na classe filha.
instanceof
Verifica se um objeto é instância de uma classe, inclusive das classes pai na cadeia de herança. ("Instanciar" é o ato de criar um objeto real a partir de uma classe.)
O operador instanceof serve para verificar se um objeto foi criado a partir de uma classe específica (ou de uma classe que ela herda).
Pense nele como um teste de "DNA": você pergunta ao JavaScript se aquele objeto pertence a uma determinada linhagem.
Polimorfismo
Mesmo método, comportamentos diferentes

Polimorfismo significa "muitas formas". É a capacidade de objetos de diferentes classes responderem ao mesmo método de formas distintas. Permite tratar objetos variados de maneira uniforme, tornando o código mais flexível e extensível.

Significa "muitas formas". É a capacidade de objetos de diferentes classes responderem à mesma mensagem (método) de maneiras distintas. Geralmente ocorre quando uma classe filha redefine (sobrescreve) um método que herdou da classe pai. 
 
  • Exemplo: Imagine que Cachorro e Gato herdam o método fazerSom() de Animal. No polimorfismo, quando você chama fazerSom(), o cachorro late e o gato mia. 
Sistema de Pagamentos — Polimorfismo em ação 
class Pagamento {
  constructor(valor) {
    this.valor = valor;
  }

  processar() {
    return `Processando pagamento de R$ ${this.valor}`;
  }
}

class PagamentoCartao extends Pagamento {
  constructor(valor, numeroCartao) {
    super(valor);
    this.numeroCartao = numeroCartao;
  }
  processar() {
    return `Cartão **** ${this.numeroCartao.slice(-4)}: debitado R$ ${this.valor}`;
  }
}

class PagamentoPix extends Pagamento {
  constructor(valor, chave) {
    super(valor);
    this.chave = chave;
  }
  processar() {
    return `Pix enviado para ${this.chave}: R$ ${this.valor} (instântaneo)`;
  }
}

class PagamentoBoleto extends Pagamento {
  constructor(valor) {
    super(valor);
    this.vencimento = new Date(Date.now() + 3 * 86400000);
  }
  processar() {
    return `Boleto de R$ ${this.valor} gerado. Vence em ${this.vencimento.toLocaleDateString()}`;
  }
}

// Polimorfismo: mesma função processa qualquer tipo de pagamento
function realizarCheckout(pagamentos) {
  pagamentos.forEach(p => console.log(p.processar()));
}

const pagamentos = [
  new PagamentoCartao(150, '4111111111111111'),
  new PagamentoPix(89.90, 'joao@email.com'),
  new PagamentoBoleto(200)
];
realizarCheckout(pagamentos);
Por que o Polimorfismo é poderoso?
A função realizarCheckout não precisa saber qual tipo de pagamento está processando. Ela chama processar() e cada objeto sabe como se comportar. Para adicionar um novo meio de pagamento, basta criar uma nova classe — sem alterar a função existente.
4
Abstração
Esconder complexidade, expor apenas o essencial

A abstração consiste em modelar apenas os aspectos relevantes de uma entidade, ocultando detalhes desnecessários. Em JavaScript, implementamos abstração criando interfaces claras onde o usuário da classe sabe o que ela faz, mas não precisa saber como ela faz.

A Abstração é o pilar da POO que foca em extrair apenas as características e comportamentos essenciais de um objeto para o contexto do seu sistema, descartando detalhes irrelevantes e complexos.
 
Imagine que você está modelando um sistema para uma concessionária:
 
  • Abstração de um Carro: No seu código, o objeto Carro terá modelo, ano e preço.
  • O que fica de fora: Você ignora a composição química da tinta ou o número de parafusos no motor, pois essas informações não importam para a venda.
Como funciona na prática?
 
  1. Simplificação: Reduz a complexidade ao esconder a lógica técnica por trás de interfaces simples (como métodos).
  2. Identidade: Define "o que o objeto é" e "o que ele faz" de forma genérica.
  3. Classes Abstratas: Em muitas linguagens, você cria uma "classe modelo" que não pode ser instanciada diretamente, servindo apenas de base para classes mais específicas.
Diferença rápida:
 
  • Abstração: Define o que um objeto faz (visão externa/essencial).
  • Encapsulamento: Esconde como ele faz (proteção interna).
Abstração — Conexão com Banco de Dados 
class Database {
  #conexao;
  #tentativas = 0;
  #maxTentativas = 3;

  constructor(config) {
    this.#conexao = config;
  }

  // Método privado (lógica interna complexa)
  #validarConexao() {
    return this.#conexao.host && this.#conexao.porta;
  }

  #registrarLog(operacao) {
    const timestamp = new Date().toISOString();
    console.log(`[${timestamp}] ${operacao}`);
  }

  // Interface pública simples
  buscar(tabela, condicao = {}) {
    if (!this.#validarConexao()) throw new Error('Conexão inválida');
    this.#registrarLog(`SELECT * FROM ${tabela}`);
    return `Buscando em ${tabela} com filtros: ${JSON.stringify(condicao)}`;
  }

  inserir(tabela, dados) {
    if (!this.#validarConexao()) throw new Error('Conexão inválida');
    this.#registrarLog(`INSERT INTO ${tabela}`);
    return `Registro inserido em ${tabela}`;
  }
}

const db = new Database({ host: 'localhost', porta: 5432 });
console.log(db.buscar('usuarios', { ativo: true }));
console.log(db.inserir('pedidos', { produto: 'Notebook' }));
Benefícios da Abstração
  • Simplicidade: Quem usa a classe não precisa entender a implementação interna
  • Manutenção: A lógica interna pode ser alterada sem impactar quem usa a classe
  • Segurança: Métodos e dados privados ficam protegidos de uso incorreto
  • Legibilidade: Interface clara e intuitiva facilita o uso e colaboração
Abstração vs Encapsulamento — Qual a diferença?

Esses dois pilares são frequentemente confundidos por atuarem juntos. A distinção essencial é:

Encapsulamento
É o mecanismo. Trata de como proteger os dados — usando campos e métodos privados para impedir acesso externo indevido.
Responde: "Quem pode acessar esse dado?"
Abstração
É o conceito. Trata de o que expor — definir uma interface simples que esconda a complexidade interna de como algo funciona.
Responde: "O que o usuário precisa ver?"
Controle Remoto — Ambos os pilares no mesmo código 
class ControleRemoto {

  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  // ENCAPSULAMENTO: #volume é privado.
  // O usuário NÃO pode fazer: controle.#volume = 1000
  // O dado está protegido contra alterações indevidas.
  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  #volume = 10;

  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  // ABSTRAÇÃO: aumentarVolume() é a interface simples.
  // O usuário chama um método intuitivo e não precisa saber
  // como o hardware processa o ganho de decibéis internamente.
  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  aumentarVolume() {
    if (this.#volume < 100) {
      this.#volume++;
      this.#logStatus();
    }
  }

  diminuirVolume() {
    if (this.#volume > 0) {
      this.#volume--;
      this.#logStatus();
    }
  }

  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  // ENCAPSULAMENTO + ABSTRAÇÃO: método privado.
  // É um detalhe interno de funcionamento que o usuário
  // do controle não vê nem precisa chamar diretamente.
  // ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  #logStatus() {
    console.log(`Volume atual: ${this.#volume}%`);
  }
}

// Uso prático — simples, intuitivo, sem expor internals
const meuControle = new ControleRemoto();
meuControle.aumentarVolume(); // Volume atual: 11%
meuControle.aumentarVolume(); // Volume atual: 12%
meuControle.diminuirVolume(); // Volume atual: 11%

// meuControle.#volume = 50; // ERRO! O JS protege o dado privado.
Resumo Visual da Diferença
Encapsulamento no Controle
O campo #volume é privado. Ninguém de fora pode ler ou escrever diretamente. O JS lança um erro se tentar: controle.#volume = 50
Abstração no Controle
O método aumentarVolume() é simples e intuitivo. Quem usa o controle não sabe (nem precisa saber) como os decibéis são calculados internamente.
Analogia do Mundo Real
Encapsulamento = O motor do carro está trancado no capô.
Abstração = Você tem apenas volante, acelerador e freio para dirigir.
Exemplo Final: Todos os Pilares Juntos

Um sistema de e-commerce que aplica os quatro pilares simultaneamente:

Sistema de E-commerce — POO Completa 
// ABSTRAÇÃO + ENCAPSULAMENTO
class Produto {
  #preco;
  #estoque;

  constructor(nome, preco, estoque) {
    this.nome = nome;
    this.#preco = preco;
    this.#estoque = estoque;
  }

  get preco()   { return this.#preco; }
  get estoque() { return this.#estoque; }

  set preco(valor) {
    if (valor < 0) throw new Error('Preço não pode ser negativo');
    this.#preco = valor;
  }

  reduzirEstoque(qtd) {
    if (qtd > this.#estoque) throw new Error('Estoque insuficiente');
    this.#estoque -= qtd;
  }

  calcularFrete() { return 15; } // Sobrescrito nas filhas (polimorfismo)
}

// HERANÇA
class ProdutoPesado extends Produto {
  constructor(nome, preco, estoque, pesoKg) {
    super(nome, preco, estoque);
    this.pesoKg = pesoKg;
  }
  calcularFrete() { return this.pesoKg * 5; } // POLIMORFISMO
}

class ProdutoDigital extends Produto {
  calcularFrete() { return 0; } // Digital não tem frete
}

class Carrinho {
  #itens = [];

  adicionar(produto, quantidade) {
    produto.reduzirEstoque(quantidade);
    this.#itens.push({ produto, quantidade });
  }

  calcularTotal() {
    return this.#itens.reduce((total, item) => {
      const subtotal = item.produto.preco * item.quantidade;
      const frete    = item.produto.calcularFrete(); // Polimorfismo!
      return total + subtotal + frete;
    }, 0);
  }
}

const notebook = new ProdutoPesado('Notebook', 3500, 10, 2.5);
const ebook    = new ProdutoDigital('E-book POO', 49, 999);

const carrinho = new Carrinho();
carrinho.adicionar(notebook, 1);
carrinho.adicionar(ebook, 2);
console.log('Total: R$', carrinho.calcularTotal()); // R$ 3610.50
