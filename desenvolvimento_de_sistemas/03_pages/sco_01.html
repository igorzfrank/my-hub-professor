Funções em JavaScript
Blocos de código reutilizáveis

Funções são blocos de código que executam uma tarefa específica. Elas podem receber parâmetros, processar dados e retornar resultados. São fundamentais para organizar e reutilizar código.

Declaração de Função Tradicional
Função Tradicional 
// Declaração de função
function saudar(nome) {
  return `Olá, ${nome}!`;
}

// Chamando a função
const mensagem = saudar('Maria');
console.log(mensagem); // Olá, Maria!
Arrow Function (Função Seta)
Arrow Function 
// Sintaxe moderna e mais concisa
const calcularArea = (largura, altura) => {
  return largura * altura;
};

// Arrow function de uma linha (return implícito)
const dobrar = (numero) => numero * 2;

console.log(calcularArea(5, 10)); // 50
console.log(dobrar(7)); // 14
Pontos-Chave sobre Funções
  • Funções podem receber parâmetros e retornar valores
  • Arrow functions são mais concisas e não alteram o contexto this
  • Funções são first-class citizens: podem ser atribuídas a variáveis, passadas como argumentos
  • Use const para declarar funções que não serão reatribuídas

Métodos
Funções dentro de objetos

Um método é uma função que pertence a um objeto. Métodos definem o comportamento de um objeto e podem acessar suas propriedades através da palavra-chave this.

Objeto com Métodos
Exemplo de Métodos 
const usuario = {
  nome: 'João',
  idade: 25,
  
  // Método tradicional
  apresentar: function() {
    return `Meu nome é ${this.nome} e tenho ${this.idade} anos.`;
  },
  
  // Sintaxe simplificada (ES6)
  fazerAniversario() {
    this.idade++;
    return `Agora tenho ${this.idade} anos!`;
  }
};

console.log(usuario.apresentar()); 
// Meu nome é João e tenho 25 anos.

console.log(usuario.fazerAniversario()); 
// Agora tenho 26 anos!
Diferença entre Função e Método
Função: Bloco de código independente
Método: Função associada a um objeto que pode acessar suas propriedades via this

Objetos Literais
Estruturas de dados chave-valor

Objetos são coleções de propriedades (dados) e métodos (comportamentos). São a base da programação orientada a objetos em JavaScript e permitem agrupar informações relacionadas.

Criando e Manipulando Objetos
Objetos Literais 
// Criando um objeto literal
const produto = {
  id: 1,
  nome: 'Notebook',
  preco: 3500,
  emEstoque: true,
  
  calcularDesconto(percentual) {
    return this.preco * (1 - percentual / 100);
  }
};

// Acessando propriedades
console.log(produto.nome); // Notebook
console.log(produto['preco']); // 3500

// Modificando propriedades
produto.preco = 3200;

// Adicionando novas propriedades
produto.categoria = 'Eletrônicos';

// Chamando método
console.log(produto.calcularDesconto(10)); // 2880
Destructuring (Desestruturação)
Desestruturação de Objetos 
const pessoa = {
  nome: 'Ana',
  idade: 30,
  cidade: 'São Paulo'
};

// Extraindo propriedades
const { nome, idade } = pessoa;
console.log(nome); // Ana
console.log(idade); // 30

// Renomeando variáveis
const { nome: nomePessoa, cidade: cidadePessoa } = pessoa;
console.log(nomePessoa); // Ana
Métodos Úteis para Objetos
Object.keys(obj) → Retorna array com as chaves
Object.values(obj) → Retorna array com os valores
Object.entries(obj) → Retorna array de pares [chave, valor]
Object.assign() → Copia propriedades de um objeto para outro

Classes e Orientação a Objetos
Criando moldes para objetos

Classes são "moldes" para criar objetos com propriedades e métodos predefinidos. Introduzidas no ES6, tornam a programação orientada a objetos mais clara e organizada em JavaScript.

Criando uma Classe
Definição de Classe 
class Pessoa {
  // Construtor: executado ao criar nova instância
  constructor(nome, idade) {
    this.nome = nome;
    this.idade = idade;
  }
  
  // Métodos da classe
  apresentar() {
    return `Olá, meu nome é ${this.nome} e tenho ${this.idade} anos.`;
  }
  
  fazerAniversario() {
    this.idade++;
    return `Feliz aniversário! Agora tenho ${this.idade} anos.`;
  }
}

// Criando instâncias (objetos) da classe
const pessoa1 = new Pessoa('Carlos', 28);
const pessoa2 = new Pessoa('Marina', 32);

console.log(pessoa1.apresentar()); 
// Olá, meu nome é Carlos e tenho 28 anos.

console.log(pessoa2.fazerAniversario()); 
// Feliz aniversário! Agora tenho 33 anos.
Herança de Classes
Exemplo de Herança 
// Classe pai (base)
class Animal {
  constructor(nome) {
    this.nome = nome;
  }
  
  fazerSom() {
    return `${this.nome} faz algum som`;
  }
}

// Classe filha (herda de Animal)
class Cachorro extends Animal {
  constructor(nome, raca) {
    super(nome); // Chama o construtor da classe pai
    this.raca = raca;
  }
  
  // Sobrescrevendo método
  fazerSom() {
    return `${this.nome} late: Au au!`;
  }
  
  // Método específico da classe Cachorro
  abanarRabo() {
    return `${this.nome} está abanando o rabo!`;
  }
}

const meuCachorro = new Cachorro('Rex', 'Labrador');
console.log(meuCachorro.fazerSom()); // Rex late: Au au!
console.log(meuCachorro.abanarRabo()); // Rex está abanando o rabo!
Constructor
Método especial executado automaticamente ao criar uma nova instância. Inicializa as propriedades do objeto.
Palavra-chave 'this'
Refere-se à instância atual do objeto. Permite acessar propriedades e métodos dentro da classe.
Herança (extends)
Permite que uma classe herde propriedades e métodos de outra, promovendo reutilização de código.
Super
Chama o construtor ou métodos da classe pai. Obrigatório usar antes de acessar 'this' em classes filhas.
Exemplo Prático: Sistema de Produtos

Vamos criar um sistema simples de gerenciamento de produtos que demonstra todos os conceitos revisados:

Sistema Completo 
// Classe base Produto
class Produto {
  constructor(id, nome, preco) {
    this.id = id;
    this.nome = nome;
    this.preco = preco;
  }
  
  calcularDesconto(percentual) {
    return this.preco * (1 - percentual / 100);
  }
  
  exibirInfo() {
    return {
      id: this.id,
      nome: this.nome,
      preco: this.preco
    };
  }
}

// Classe especializada que herda de Produto
class ProdutoDigital extends Produto {
  constructor(id, nome, preco, tamanhoMB) {
    super(id, nome, preco);
    this.tamanhoMB = tamanhoMB;
  }
  
  exibirInfo() {
    const infoBase = super.exibirInfo();
    return { ...infoBase, tamanho: `${this.tamanhoMB}MB` };
  }
}

// Gerenciador de produtos
const gerenciador = {
  produtos: [],
  
  adicionar(produto) {
    this.produtos.push(produto);
  },
  
  listar() {
    return this.produtos.map(p => p.exibirInfo());
  },
  
  buscarPorId(id) {
    return this.produtos.find(p => p.id === id);
  }
};

// Usando o sistema
const produto1 = new Produto(1, 'Livro', 45);
const produto2 = new ProdutoDigital(2, 'E-book', 25, 5.2);

gerenciador.adicionar(produto1);
gerenciador.adicionar(produto2);

console.log(gerenciador.listar());
// [
//   { id: 1, nome: 'Livro', preco: 45 },
//   { id: 2, nome: 'E-book', preco: 25, tamanho: '5.2MB' }
// ]
Princípios da Orientação a Objetos
Encapsulamento
Agrupar dados (propriedades) e comportamentos (métodos) relacionados em uma única unidade (classe/objeto). Protege dados internos.
Herança
Permite que classes filhas herdem propriedades e métodos de classes pai, promovendo reutilização e hierarquia de código.
Polimorfismo
Permite que objetos de diferentes classes sejam tratados de forma uniforme. Métodos podem ser sobrescritos em classes filhas.
Abstração
Esconder complexidade interna e expor apenas funcionalidades essenciais. O usuário não precisa saber "como" funciona, apenas "o que" faz.
