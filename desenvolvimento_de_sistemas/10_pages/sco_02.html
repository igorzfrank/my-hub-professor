Express.js + PostgreSQL
Integrando banco de dados real ao MVC
Exercícios práticos de integração entre Express, PostgreSQL e a arquitetura MVC com ES Modules. Cada exercício parte do zero — você criará o banco, a tabela, os dados iniciais e todos os arquivos da API.
Node.js Express PostgreSQL node-postgres (pg) MVC ES Modules
Configuração obrigatória antes de começar
Terminal — instalação
npm init -y
npm install express pg
# adicionar "type":"module" no package.json
Estrutura de pastas
projeto/
├── src/
│ ├── database/ → db.js
│ ├── models/ → *.model.js
│ ├── controllers/ → *.controller.js
│ └── routes/ → *.routes.js
└── app.js
O arquivo src/database/db.js é o mesmo para ambos os exercícios — crie-o uma vez com new Pool({...}) apontando para o banco do exercício que estiver fazendo e importe-o nos Models.
Exercício 1
API de Livros — Biblioteca Municipal
GET · POST · DELETE
A Biblioteca Municipal de Palhoça precisa de uma API para gerenciar seu acervo de livros. Você deve criar do zero toda a estrutura MVC com banco de dados PostgreSQL, incluindo o banco, a tabela, os dados iniciais e as rotas da API.
Passo 1 Banco de dados e tabela — executar no pgAdmin
SQL — criação
-- 1. Crie o banco
CREATE DATABASE biblioteca;

-- 2. Conecte ao banco e execute o restante

CREATE TABLE livros (
id SERIAL PRIMARY KEY,
titulo TEXT NOT NULL,
autor TEXT NOT NULL,
ano INTEGER NOT NULL,
disponivel BOOLEAN DEFAULT TRUE
);
Passo 2 Popular a tabela com dados iniciais
SQL — dados de exemplo
INSERT INTO livros (titulo, autor, ano, disponivel) VALUES
('Dom Casmurro', 'Machado de Assis', 1899, TRUE),
('O Cortiço', 'Aluísio Azevedo', 1890, TRUE),
('Vidas Secas', 'Graciliano Ramos', 1938, FALSE),
('Grande Sertão: Veredas', 'João Guimarães Rosa', 1956, TRUE),
('A Hora da Estrela', 'Clarice Lispector', 1977, FALSE);
Passo 3 O que você deve construir
db.js — conexão
Crie o arquivo src/database/db.js com um Pool apontando para o banco biblioteca. Exporte o pool como export default.
livro.model.js — queries
Crie as funções buscarTodos, buscarPorId e criar, cada uma fazendo a query SQL correspondente com await pool.query(). Use RETURNING * no INSERT.
livro.controller.js — handlers
Crie as funções listar, buscarPorId e criar. Todas devem ser async com try/catch. Retorne 404 se o livro não existir e 201 ao criar.
livro.routes.js + app.js
Crie o Router com as rotas abaixo e registre-o no app.js com o prefixo /livros. Não esqueça do express.json().
Rotas esperadas
GET /livros → lista todos os livros do banco
GET /livros/:id → retorna um livro pelo id (404 se não existir)
POST /livros → cadastra um novo livro (retorna 201 + livro criado)
Body esperado no POST (JSON)
{
"titulo": "Capitães da Areia",
"autor": "Jorge Amado",
"ano": 1937,
"disponivel": true
}
Como verificar
Use o Postman ou Insomnia. Faça um GET em localhost:3000/livros — deve retornar os 5 livros inseridos no banco. Depois crie um novo livro com POST e confirme que ele aparece com um id gerado automaticamente pelo PostgreSQL.
Exercício 2
API de Produtos — Controle de Estoque
CRUD completo
Uma loja de informática precisa de uma API para controlar seu estoque de produtos. Aqui você vai implementar o CRUD completo — incluindo PUT para atualizar e DELETE para remover — tudo persistido no PostgreSQL.
Passo 1 Banco de dados e tabela — executar no pgAdmin
SQL — criação
-- 1. Crie o banco
CREATE DATABASE estoque;

-- 2. Conecte ao banco e execute o restante

CREATE TABLE produtos (
id SERIAL PRIMARY KEY,
nome TEXT NOT NULL,
categoria TEXT NOT NULL,
preco NUMERIC(10, 2) NOT NULL,
quantidade INTEGER NOT NULL DEFAULT 0
);
Passo 2 Popular a tabela com dados iniciais
SQL — dados de exemplo
INSERT INTO produtos (nome, categoria, preco, quantidade) VALUES
('Teclado Mecânico', 'Periférico', 350.00, 15),
('Mouse Gamer', 'Periférico', 180.00, 22),
('Monitor 24"', 'Monitor', 1200.00, 8),
('SSD 480GB', 'Armazenamento', 280.00, 30),
('Headset USB', 'Periférico', 220.00, 12),
('Placa de Vídeo RTX 3060', 'Hardware', 2800.00, 4);
Passo 3 O que você deve construir
produto.model.js — CRUD completo
Implemente as funções buscarTodos, buscarPorId, criar, atualizar e remover. Use RETURNING * no UPDATE para devolver o produto atualizado. Use rowCount no DELETE para saber se o produto existia.
produto.controller.js — 5 handlers
Crie os 5 handlers correspondentes às funções do Model, todos com async/await e try/catch. O handler de atualizar deve retornar 404 se o id não existir. O de remover deve retornar 404 se rowCount === 0 e 204 em caso de sucesso.
produto.routes.js + app.js
Registre todas as 5 rotas no Router e configure o app.js com o prefixo /produtos.
Rotas esperadas
GET /produtos → lista todos os produtos
GET /produtos/:id → retorna um produto (404 se não existir)
POST /produtos → cadastra um novo produto (retorna 201)
PUT /produtos/:id → atualiza um produto (404 se não existir)
DELETE /produtos/:id → remove um produto (204 em sucesso, 404 se não existir)
Body esperado no POST
{
"nome": "Webcam HD",
"categoria": "Periférico",
"preco": 320.00,
"quantidade": 10
}
Body esperado no PUT
{
"nome": "Mouse Gamer Pro",
"categoria": "Periférico",
"preco": 210.00,
"quantidade": 18
}
Como verificar
Siga esta sequência no Postman ou Insomnia:
1. GET em /produtos — deve listar os 6 produtos.
2. POST com os dados da Webcam — deve retornar status 201 com o produto e o id gerado.
3. PUT em /produtos/2 — deve atualizar o Mouse e retornar o produto atualizado.
4. DELETE em /produtos/2 — deve retornar 204 sem corpo.
5. GET em /produtos/2 — deve retornar 404, confirmando que o produto foi removido do banco.

Exercício 3
API de Usuários — Cadastro, Login e Atualização
AUTH · UPDATE
Neste exercício, você irá desenvolver uma API de autenticação simples com cadastro, login e atualização de usuários utilizando Express + PostgreSQL.

Importante: Este exercício é didático — não utilize bcrypt ou qualquer tipo de criptografia. As senhas serão armazenadas como texto puro.
PASSO 1 Banco de dados e tabela
-- Criar banco
CREATE DATABASE usuarios;

-- Criar tabela
CREATE TABLE usuarios (
  id SERIAL PRIMARY KEY,
  nome TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  senha TEXT NOT NULL
);
PASSO 2 Estrutura MVC a ser criada
usuario.model.js
criarUsuario · buscarPorEmail · buscarPorId · atualizarUsuario
usuario.controller.js
cadastrar · login · atualizar
usuario.routes.js
Definição das rotas e integração com o controller
Regras importantes

• Não utilizar bcrypt ou criptografia
• Senha deve ser comparada diretamente (texto)
• Email deve ser único no banco
• Login deve validar email + senha corretamente
• Retornar 404 se usuário não existir
ROTAS ESPERADAS
POST /usuarios/register → cadastrar usuário
POST /usuarios/login → autenticar usuário
PUT /usuarios/:id → atualizar dados do usuário
Cadastro
{
 "nome": "João",
 "email": "joao@email.com",
 "senha": "123456"
}
Login
{
 "email": "joao@email.com",
 "senha": "123456"
}
Body para atualização (PUT)
{
 "nome": "João Silva",
 "email": "joao.silva@email.com",
 "senha": "novaSenha"
}
Como verificar
1. Faça um POST em /usuarios/register
2. Faça login com os mesmos dados (POST /usuarios/login)
3. Atualize o usuário com PUT /usuarios/:id
4. Tente logar novamente com os dados atualizados
