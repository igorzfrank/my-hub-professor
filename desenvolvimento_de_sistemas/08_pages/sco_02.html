Core Modules do Node.js — fs, os e path
Nesta aula voce aprenderá a usar os tres principais modulos nativos do Node.js: fs para manipular arquivos, os para consultar o sistema operacional e path para montar caminhos de forma segura — tudo com a sintaxe moderna de ES Modules.
Capacidades Trabalhadas
 
Aplicar os modulos nativos do Node.js para interagir com o sistema operacional e o sistema de arquivos.
 
Utilizar a sintaxe ES Modules (import/export) em scripts Node.js.
 
Integrar multiplos modulos nativos para resolver problemas compostos do mundo real.
Por que usar os modulos nativos?
Quando voce instala o Node.js, junto com ele vem uma biblioteca padrao chamada Core Modules — um conjunto de modulos prontos para uso que cobrem as necessidades mais comuns de qualquer aplicacao backend: ler arquivos, verificar a memoria do servidor, montar caminhos de pasta...
A grande vantagem e que voce nao precisa instalar nada. Nao ha npm install, nao ha dependencias externas, nao ha risco de versao incompativel. Os modulos ja estao disponiveis assim que o Node esta instalado na maquina.
Analogia do mundo real: os modulos nativos sao como as ferramentas que ja vem na caixa de ferramentas basica — martelo, chave de fenda, alicate. Voce nao precisa comprar nada para comecar a trabalhar. So depois, em casos especificos, voce vai buscar ferramentas especializadas em lojas externas (o npm).
Configurando o projeto para ES Modules
O Node.js suporta dois sistemas de modulos: o antigo CommonJS (com require) e o moderno ES Modules (com import). Nesta aula usaremos ES Modules, que e o padrao atual do JavaScript. Ha duas formas de habilitar:
Forma 1 — Recomendada
Adicionar "type" no package.json
package.json 
{
  "name": "meu-projeto",
  "version": "1.0.0",
  "type": "module"
}
Com isso, todos os arquivos .js do projeto passam a usar ES Modules automaticamente.
Forma 2 — Por arquivo
Usar a extensao .mjs
terminal 
# Salve o arquivo como .mjs
script.mjs

# Execute normalmente
node script.mjs
Util quando voce quer misturar arquivos CommonJS e ES Modules no mesmo projeto.
Atencao: __dirname e __filename nao existem em ES Modules
No sistema CommonJS existiam duas variaveis magicas: __dirname (pasta do arquivo) e __filename (caminho completo do arquivo). Em ES Modules elas nao estao disponiveis. Use o padrao abaixo sempre que precisar do diretorio atual:
substituto de __dirname em ES Modules — copie sempre que precisar 
import { fileURLToPath } from 'url';
import { dirname } from 'path';

const __filename = fileURLToPath(import.meta.url);
const __dirname  = dirname(__filename);

// Agora __dirname funciona igual ao CommonJS
console.log(__dirname);  // /home/usuario/meu-projeto
console.log(__filename); // /home/usuario/meu-projeto/script.js
fs
Modulo 01
File System — Sistema de Arquivos
O modulo fs (File System) e o modulo responsavel por toda interacao com o sistema de arquivos: criar, ler, alterar, listar e remover arquivos e pastas. E um dos modulos mais usados no desenvolvimento Node.js.
Ele possui tres "sabores": a versao com callbacks (mais antiga), a versao sincrona (bloqueia a execucao) e a versao com Promises — que e a recomendada hoje. Nesta aula usamos sempre a versao com Promises:
importando o modulo fs 
// Importa funcoes especificas da versao com Promises
import { writeFile, readFile, appendFile, readdir, unlink } from 'fs/promises';

// Todas as funcoes retornam Promises — use sempre async/await
1 Criando e escrevendo arquivos — writeFile
A funcao writeFile cria um arquivo com o conteudo fornecido. Se o arquivo ja existir, ele e sobrescrito completamente. Recebe tres argumentos: caminho do arquivo, conteudo e encoding ('utf8' para texto).
escrever-arquivo.js 
import { writeFile } from 'fs/promises';

async function criarArquivo() {
  try {
    const conteudo = 'Linha 1: Ola, Node.js!\nLinha 2: Arquivo criado com sucesso.';

    // Cria (ou sobrescreve) o arquivo com o conteudo
    await writeFile('meu-arquivo.txt', conteudo, 'utf8');

    console.log('Arquivo criado com sucesso!');
  } catch (erro) {
    console.error('Erro ao criar arquivo:', erro.message);
  }
}

criarArquivo();
2 Lendo arquivos — readFile
readFile le o conteudo de um arquivo e retorna uma Promise com o texto. Sempre informe o encoding 'utf8' — sem ele, o Node retorna um Buffer de bytes em vez de texto legivel.
ler-arquivo.js 
import { readFile } from 'fs/promises';

async function lerArquivo() {
  try {
    const conteudo = await readFile('meu-arquivo.txt', 'utf8');
    console.log('Conteudo do arquivo:');
    console.log(conteudo);
  } catch (erro) {
    // error.code === 'ENOENT' significa que o arquivo nao existe
    if (erro.code === 'ENOENT') {
      console.log('Arquivo nao encontrado.');
    } else {
      console.error('Erro ao ler arquivo:', erro.message);
    }
  }
}

lerArquivo();
3 Acrescentando conteudo — appendFile
Diferente do writeFile, o appendFile nao apaga o conteudo existente — ele acrescenta o novo conteudo ao final do arquivo. Muito usado para logs e diarios de execucao.
log-execucao.js — execute multiplas vezes e observe o arquivo crescer 
import { appendFile } from 'fs/promises';

async function registrarLog() {
  const agora = new Date().toLocaleString('pt-BR');
  const linha = `[${agora}] Script executado com sucesso.\n`;

  await appendFile('log.txt', linha, 'utf8');
  console.log('Log registrado!');
}

registrarLog();

// Conteudo de log.txt apos 3 execucoes:
// [10/06/2025, 09:10:01] Script executado com sucesso.
// [10/06/2025, 09:12:34] Script executado com sucesso.
// [10/06/2025, 09:15:00] Script executado com sucesso.
4 Listando pastas — readdir
readdir lista o conteudo de uma pasta. Com a opcao { withFileTypes: true }, em vez de retornar apenas os nomes, ela retorna objetos Dirent que possuem metodos para identificar o tipo de cada item.
listar-pasta.js 
import { readdir } from 'fs/promises';
import { fileURLToPath } from 'url';
import { dirname } from 'path';

const __dirname = dirname(fileURLToPath(import.meta.url));

async function listarPasta() {
  const itens = await readdir(__dirname, { withFileTypes: true });

  for (const item of itens) {
    const tipo = item.isDirectory() ? '[pasta]  ' : '[arquivo]';
    console.log(`${tipo} ${item.name}`);
  }
}

listarPasta();

// Saida esperada:
// [pasta]   node_modules
// [arquivo] listar-pasta.js
// [arquivo] package.json
Principais metodos do objeto Dirent
item.name — nome do item
item.isDirectory() — e uma pasta?
item.isFile() — e um arquivo?
item.isSymbolicLink() — e um link?
os
Modulo 02
Operating System — Sistema Operacional
O modulo os expoe informacoes sobre o sistema operacional onde o Node.js esta rodando: nome da maquina, plataforma, quantidade de CPUs, memoria RAM disponivel, diretorio home do usuario e muito mais. E extremamente util para scripts de monitoramento e automacao.
importando o modulo os 
// O modulo os e exportado como default — import direto, sem chaves
import os from 'os';
1 Principais metodos do modulo os
Metodo O que retorna Exemplo de saida
os.hostname() Nome do computador na rede NOTEBOOK-JOAO
os.platform() Sistema operacional linux / win32 / darwin
os.release() Versao do kernel do SO 5.15.0-91-generic
os.arch() Arquitetura do processador x64 / arm64
os.totalmem() Total de RAM (em bytes) 16641695744
os.freemem() RAM livre agora (em bytes) 9876543210
os.homedir() Diretorio home do usuario /home/joao
os.cpus() Array com info de cada nucleo [ { model, speed, times }, ... ]
2 Exemplo completo — ficha tecnica da maquina
Os metodos totalmem() e freemem() retornam valores em bytes. Para exibir em GB, divida por 1024 ** 3 (que equivale a 1.073.741.824 bytes).
ficha-tecnica.js 
import os from 'os';

const totalGB = (os.totalmem() / 1024 ** 3).toFixed(1);
const livreGB = (os.freemem()  / 1024 ** 3).toFixed(1);

console.log('=====================================');
console.log('      FICHA TECNICA DA MAQUINA');
console.log('=====================================');
console.log(`Computador : ${os.hostname()}`);
console.log(`Sistema    : ${os.platform()}`);
console.log(`Versao     : ${os.release()}`);
console.log(`Arquitetura: ${os.arch()}`);
console.log(`RAM Total  : ${totalGB} GB`);
console.log(`RAM Livre  : ${livreGB} GB`);
console.log(`Home       : ${os.homedir()}`);
console.log(`CPUs       : ${os.cpus().length} nucleos`);
console.log('=====================================');

path
Modulo 03
Path — Caminhos de Arquivo
O modulo path resolve um problema classico: caminhos de arquivo sao escritos de forma diferente em cada sistema operacional. No Windows usa-se barra invertida (\); no Linux e macOS usa-se barra normal (/).
Regra de ouro — nunca concatene caminhos com strings
✗ Errado (quebra no Windows)
'pasta' + '/' + 'arquivo.txt'
✓ Correto (multiplataforma)
path.join('pasta', 'arquivo.txt')
importando o modulo path 
import path from 'path';
1 Montando caminhos — path.join()
path.join() recebe varios segmentos de caminho e os une usando o separador correto do SO atual. E a funcao mais usada do modulo.
path-join.js 
import path from 'path';
import { fileURLToPath } from 'url';
import { dirname } from 'path';

const __dirname = dirname(fileURLToPath(import.meta.url));

// Monta caminhos a partir do diretorio atual do script
const config = path.join(__dirname, 'config', 'settings.json');
const dados  = path.join(__dirname, 'dados');
const log    = path.join(__dirname, 'logs', 'app.log');

console.log('Separador do SO:', path.sep); // / no Linux, \ no Windows
console.log('Config:', config);
console.log('Dados: ', dados);
console.log('Log:   ', log);

// Saida no Linux/Mac:
// /home/joao/projeto/config/settings.json
// /home/joao/projeto/dados
// /home/joao/projeto/logs/app.log
2 Decompondo caminhos — path.parse() e metodos auxiliares
path.parse() decompoe um caminho em suas partes. Ha tambem metodos menores para extrair partes especificas sem precisar do objeto completo.
path-parse.js 
import path from 'path';

const caminho = '/projetos/clinica/src/relatorio.json';

// path.parse — retorna todas as partes de uma vez
const partes = path.parse(caminho);
console.log(partes);
// { root: '/', dir: '/projetos/clinica/src',
//   base: 'relatorio.json', ext: '.json', name: 'relatorio' }

// Metodos individuais — mais diretos
console.log(path.basename(caminho));         // relatorio.json
console.log(path.basename(caminho, '.json')); // relatorio (sem extensao)
console.log(path.extname(caminho));          // .json
console.log(path.dirname(caminho));          // /projetos/clinica/src
Resumo dos metodos do modulo path
path.join(...partes)
Une segmentos com o sep correto
path.parse(caminho)
Decompoe em root, dir, base, name, ext
path.basename(caminho)
Nome do arquivo com extensao
path.extname(caminho)
Extensao do arquivo (.json, .txt...)
path.dirname(caminho)
Pasta que contem o arquivo
path.sep
Separador do SO (/ ou \)
Exemplo Final — Integrando os tres modulos
fs + os + path juntos
Gerador automatico de relatorio de ambiente
Este exemplo combina os tres modulos em um unico script: usa os para coletar dados da maquina, fs para listar arquivos e salvar o resultado, e path para montar os caminhos de forma segura.
relatorio-ambiente.js 
import os                   from 'os';
import path                 from 'path';
import { readdir, writeFile } from 'fs/promises';
import { fileURLToPath }    from 'url';
import { dirname }          from 'path';

const __dirname = dirname(fileURLToPath(import.meta.url));

async function gerarRelatorio() {
  // 1. Coleta dados do sistema (modulo os)
  const totalGB = (os.totalmem() / 1024 ** 3).toFixed(1);
  const livreGB = (os.freemem()  / 1024 ** 3).toFixed(1);

  // 2. Lista arquivos do projeto (modulo fs)
  const itens = await readdir(__dirname);

  // 3. Monta caminhos absolutos (modulo path)
  const caminhos = itens.map(item => path.join(__dirname, item));

  // 4. Monta o conteudo do relatorio
  const agora    = new Date().toLocaleString('pt-BR');
  const conteudo = [
    '=============================================',
    `  RELATORIO DO PROJETO`,
    `  Gerado em: ${agora}`,
    '=============================================',
    '',
    'AMBIENTE',
    '--------',
    `Computador  : ${os.hostname()}`,
    `Sistema     : ${os.platform()}`,
    `Arquitetura : ${os.arch()}`,
    `RAM Total   : ${totalGB} GB`,
    `RAM Livre   : ${livreGB} GB`,
    '',
    'ARQUIVOS DO PROJETO',
    '-------------------',
    ...caminhos,
    '',
    '=============================================',
  ].join('\n');

  // 5. Salva o arquivo (modulo fs + path)
  const saida = path.join(__dirname, 'relatorio-projeto.txt');
  await writeFile(saida, conteudo, 'utf8');
  console.log(`Relatorio gerado em: ${saida}`);
}

gerarRelatorio();
