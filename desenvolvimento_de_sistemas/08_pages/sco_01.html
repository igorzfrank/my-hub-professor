ES Modules — Organizando código com import e export
Nesta aula você aprenderá o que são ES Modules, por que eles existem, como dividir seu código em arquivos reutilizáveis e como dominar as diferentes formas de exportar e importar — o padrão moderno usado em todos os projetos JavaScript profissionais.

Aplicar linguagem de programação por meio de bibliotecas e frameworks na construção de rotinas de software.
 
Integrar sistemas multiplataformas por meio da linguagem de programação.
 
Selecionar linguagem de programação de acordo com os requisitos do projeto.

O problema que os módulos resolvem
Imagine que você está desenvolvendo um sistema com centenas de funções: validações, cálculos, formatações, regras de negócio... Se tudo isso estiver em um único arquivo index.js, você logo terá um arquivo com milhares de linhas, impossível de manter, de ler e de reutilizar.
Além disso, sem uma forma de controlar o que cada arquivo "vê", qualquer variável definida em um lugar poderia ser acidentalmente sobrescrita em outro — um bug silencioso e difícil de rastrear.
Analogia do mundo real: pense num projeto de construção civil. Você não tem um único profissional fazendo tudo. Existe o pedreiro, o eletricista, o encanador — cada um especialista na sua área, trabalhando no seu "módulo" do projeto. O resultado final integra o trabalho de todos. É exatamente assim que os módulos funcionam no JavaScript.
O que são ES Modules?
ES Modules (abreviado: ESM) é o sistema de módulos nativo do JavaScript, introduzido oficialmente na versão ES6 (2015). Com ele, cada arquivo .js se torna um módulo independente, com seu próprio escopo — variáveis e funções só são visíveis para fora se forem explicitamente exportadas, e só ficam disponíveis dentro de outro arquivo se forem importadas.
Antes dos módulos (script comum)
✗ Todas as variáveis são globais
✗ Risco de colisão de nomes
✗ Ordem dos arquivos importa
✗ Difícil de testar isoladamente
Com ES Modules
✓ Escopo isolado por arquivo
✓ Você controla o que é público
✓ Dependências explícitas e claras
✓ Código reutilizável e testável
CommonJS vs ES Modules — qual a diferença?
O Node.js usou por anos outro sistema chamado CommonJS (com require e module.exports). Hoje o padrão moderno é ESM — usado nativamente pelo navegador e pelo Node.js com configuração simples.
CommonJS (legado)
const soma = require('./math');
module.exports = { soma };
ES Modules (moderno)
import { soma } from './math.js';
export { soma };
Configurando o projeto para usar ESM
Para usar ES Modules com Node.js, você precisa informar ao runtime que seus arquivos são módulos. Há duas formas:
Opção 1 — package.json
Recomendada para projetos inteiros
{
  "name": "meu-projeto",
  "type": "module"
}
Todos os .js do projeto passam a ser tratados como módulos.
Opção 2 — extensão .mjs
Para arquivos isolados
# renomeie o arquivo
calculadora.mjs

# execute normalmente
node calculadora.mjs
Útil quando apenas alguns arquivos precisam ser módulos.
⚠️ Atenção — caminho com extensão obrigatória
Em ES Modules com Node.js, o caminho no import deve incluir a extensão do arquivo. import { soma } from './math' causará erro — o correto é import { soma } from './math.js'.
Export Nomeado — export { nome }
O export nomeado permite que um arquivo exporte múltiplos elementos — funções, variáveis, constantes ou classes — cada um com seu próprio nome. Na importação, você usa { chaves } para dizer exatamente o que quer trazer. Se o nome não existir no módulo, o JavaScript gera um erro imediatamente.
Como funciona o export nomeado
calculadora.js (módulo)
export function somar(a,b) {
  return a + b;
}
export function subtrair(a,b) {
  return a - b;
}
export const PI = 3.14159;
→
main.js (usa o módulo)
import { somar, PI }
  from './calculadora.js';

console.log(somar(2, 3)); // 5
console.log(PI); // 3.14159
Três formas equivalentes de escrever export nomeado
Forma 1 — export inline (na declaração)
export function somar(a, b) { return a + b; }
export const PI = 3.14159;
Forma 2 — export em bloco no final do arquivo
function somar(a, b) { return a + b; }
const PI = 3.14159;

export { somar, PI }; // lista de exports no final
Forma 3 — export com alias (renomear ao exportar)
function somar(a, b) { return a + b; }

export { somar as add }; // exporta com nome "add"
Importação seletiva — importe apenas o que precisar
Uma das grandes vantagens do export nomeado é que você pode importar somente o que vai usar, deixando o resto de fora. Isso melhora a organização e facilita ferramentas de otimização (como o tree-shaking em projetos com bundlers).
// o módulo exporta: somar, subtrair, multiplicar, dividir, PI
import { somar, PI } from './calculadora.js';
// subtrair, multiplicar e dividir NÃO foram importados
Export Default — export default
O export default é usado quando um arquivo tem um único elemento principal para oferecer — a "entrega principal" do módulo. Cada arquivo pode ter apenas um export default. Ao importar, você pode escolher qualquer nome, sem precisar de chaves.
Como funciona o export default
saudacao.js (módulo)
export default function saudacao(nome) {
  return `Olá, ${nome}!`;
}
→
main.js (usa o módulo)
// sem chaves, nome livre
import boasVindas from './saudacao.js';

console.log(boasVindas('Ana'));
// Olá, Ana!
Formas de escrever export default
Default de uma função nomeada
export default function calcularIMC(peso, altura) {
  return peso / (altura * altura);
}
Default de uma classe
export default class Produto {
  constructor(nome, preco) {
    this.nome = nome;
    this.preco = preco;
  }
}
Definir e exportar separadamente
const config = { porta: 3000, debug: true };
export default config; // exporta a constante como default
💡 Atenção — apenas um default por arquivo
Cada arquivo pode ter no máximo um export default. Tentar exportar dois defaults no mesmo arquivo causará um erro de sintaxe. Para exportar múltiplas coisas, use exports nomeados.
Combinando default e nomeado no mesmo arquivo
Na prática, é comum um módulo ter um elemento principal (default) e algumas funções auxiliares (nomeadas). Você pode combinar os dois estilos no mesmo arquivo — e importar tudo de uma vez em uma única linha.
usuario.js — módulo com os dois tipos
// export nomeados
export const ROLES = {
  ADMIN: 'admin',
  USER: 'user'
};

export function validarEmail(email) {
  return email.includes('@');
}

// export default — elemento principal
export default function criarUsuario(nome, email) {
  return { nome, email, role: ROLES.USER };
}
main.js — importando os dois juntos
// default primeiro, nomeados entre { }
import criarUsuario, { ROLES, validarEmail }
  from './usuario.js';

const email = 'ana@email.com';

if (validarEmail(email)) {
  const u = criarUsuario('Ana', email);
  console.log(u);
}
Regra de sintaxe — a ordem importa
Na linha de import, o default sempre vem primeiro, sem chaves — depois, separado por vírgula, vêm os nomeados entre chaves: import default, { nomeado1, nomeado2 } from '...'
Import Namespace — import * as
Quando um módulo exporta muitas funções e você quer importar todas de uma vez, use import * as NomeDoGrupo. Isso cria um objeto com todos os exports do módulo como propriedades. Você acessa cada função com NomeDoGrupo.funcao().
Exemplo completo — import * as
import * as Geo from './geometria.js';

console.log(Geo.areaCirculo(5));    // usa areaCirculo
console.log(Geo.areaRetangulo(4, 6)); // usa areaRetangulo
console.log(Geo.perimetro(3, 4));  // usa perimetro
Quando usar import * as?
Use quando um módulo tem muitas funções relacionadas e você precisa de todas elas, ou quando quiser deixar claro de onde cada função veio (ex: Geo.areaCirculo é mais legível do que só areaCirculo em arquivos com muitas importações).
Alias — renomeando com as
A palavra-chave as permite renomear um export ao importá-lo. Isso é especialmente útil quando dois módulos exportam funções com o mesmo nome — sem alias, haveria conflito.
Resolvendo conflito de nomes com alias
// ambos os módulos exportam uma função chamada "calcular"
import { calcular as calcImposto } from './imposto.js';
import { calcular as calcDesconto } from './desconto.js';

console.log(calcImposto(1000)); // chama calcular de imposto.js
console.log(calcDesconto(1000)); // chama calcular de desconto.js
Resumo — Quando usar cada tipo de export
Situação Export Nomeado Export Default
Exportar múltiplas funções do mesmo arquivo ✓ ✗
Exportar a "entrega principal" de um módulo ✓ ✓
Importar com nome livre (sem chaves) ✗ ✓
Importar apenas parte do módulo ✓ ✗
Exportar uma classe como componente principal ✓ ✓
Limite por arquivo Ilimitado Apenas 1
Exemplo completo — Sistema de Cadastro de Alunos
A seguir, um projeto pequeno mas completo, com a estrutura de arquivos organizada por responsabilidade. Cada módulo faz uma coisa só — e o arquivo principal integra tudo.
projeto/
├── index.js ← arquivo principal
├── package.json ← "type": "module"
├── Aluno.js ← classe (export default)
├── validadores.js ← funções (export nomeado)
└── formatadores.js ← funções (export nomeado)
Aluno.js export default class
export default class Aluno {
  constructor(nome, matricula, nota) {
    this.nome = nome;
    this.matricula = matricula;
    this.nota = nota;
  }
  situacao() {
    return this.nota >= 6 ? 'Aprovado' : 'Reprovado';
  }
}
validadores.js export nomeado
export function validarNome(nome) {
  return nome.length >= 3;
}

export function validarNota(nota) {
  return nota >= 0 && nota <= 10;
}
index.js arquivo principal — combina os módulos
import Aluno from './Aluno.js';
import { validarNome, validarNota } from './validadores.js';

const nome = 'Ana Silva';
const nota = 8.5;

if (validarNome(nome) && validarNota(nota)) {
  const aluno = new Aluno(nome, '2024001', nota);
  console.log(`${aluno.nome}: ${aluno.situacao()}`);
  // Ana Silva: Aprovado
}
Erros comuns e como resolver
Erro 1 — esquecer a extensão .js
// ✗ errado
import { soma } from './math';
// ✓ correto
import { soma } from './math.js';
Erro 2 — usar { } para importar o default
// ✗ errado
import { calcularIMC } from './imc.js';
// ✓ correto
import calcularIMC from './imc.js';
Erro 3 — importar nome que não foi exportado
// módulo só exporta "somar"
import { dividir } from './math.js';
// SyntaxError!
// ✓ importe só o que existe
import { somar } from './math.js';
