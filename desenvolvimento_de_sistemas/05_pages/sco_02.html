Conceito API
📌 O que é uma API?

APIs são como conjuntos de regras que permitem que diferentes programas de computador se comuniquem entre si. Elas definem como os componentes de software devem interagir.

Imagine que cada aplicativo ou site é como um clube com suas próprias regras internas. A API é como a porta de entrada que permite que esses clubes compartilhem informações e trabalhem juntos.

Conceito de Estado (State)
📌 O que é Estado?
No React, estado (state) representa dados que podem mudar ao longo do tempo dentro de um componente.
Sempre que o estado muda, o React re-renderiza o componente e atualiza automaticamente a interface.
Exemplos de dados que usam estado
🔢 Contadores 📝 Campos de formulário 🛒 Carrinho de compras ✅ Lista de tarefas 🌐 Dados de API
Exemplo conceitual
Estado inicial: contador = 0
Usuário clica no botão
Estado atualizado: contador = 1
React atualiza a interface
Fluxo do Estado
Estado muda
    ↓
React detecta mudança
    ↓
Componente re-renderiza
    ↓
Interface atualiza

Hook useState
📌 O que é um Hook?
Hooks são funções especiais do React que permitem usar recursos como estado, efeitos e contexto em componentes funcionais.
📌 O que faz o useState?
O useState permite criar e controlar estado dentro de um componente funcional.
Sintaxe
const [estado, setEstado] = useState(valorInicial);
Elemento
Função
estado
valor atual
setEstado
função que atualiza o estado
valorInicial
valor inicial do estado
Exemplo de contador
import { useState } from "react";

function Contador() {

  const [contador, setContador] = useState(0);

  return (
    <div>
      <h2>{contador}</h2>

      <button onClick={() => setContador(contador + 1)}>
        Incrementar
      </button>
    </div>
  );

}
3
Atualização de Estado
Quando usamos a função de atualização (setState), o React:
• Atualiza o valor do estado
• Re-renderiza o componente
• Atualiza a interface
Atualização simples
setContador(contador + 1)
Atualização baseada no valor anterior
✅ Forma recomendada:
setContador(prev => prev + 1)
💡 Essa abordagem evita problemas quando várias atualizações acontecem rapidamente.

5
Ciclo de Vida com useEffect
Antes dos Hooks, o React usava componentes de classe com métodos de ciclo de vida.
Principais fases:
🟢 montagem 🟡 atualização 🔴 desmontagem
No React moderno
O useEffect substitui os métodos antigos.
Executar ao montar o componente
useEffect(() => {
  console.log("Componente carregado");
}, []);
Executa apenas uma vez, quando o componente é montado.
Executar quando o estado mudar
useEffect(() => {
  console.log("Estado mudou");
}, [contador]);
Executa quando contador muda.
Executar sempre que renderizar
useEffect(() => {
  console.log("Renderizou");
});
Executa em todas as renderizações.

Boas Práticas com Hooks
1️⃣ Usar Hooks somente no topo do componente
❌ Errado
if(condicao){
  useState()
}
✔ Correto
Hooks sempre no nível superior do componente.
2️⃣ Não usar Hooks fora de componentes
Hooks só funcionam em:
componentes React custom hooks
3️⃣ Evitar múltiplos efeitos desnecessários
Organizar os efeitos para evitar execuções repetidas e desnecessárias.
