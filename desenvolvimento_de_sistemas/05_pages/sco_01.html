📌 O que é um componente?
Um componente é um bloco reutilizável de interface. Ele funciona como uma função JavaScript que retorna JSX.
function Saudacao() {
  return <h1>Olá React</h1>;
}
🧠 Por que usar componentes?
Sem componentes 😵‍💫
<header></header>
<section></section>
<footer></footer>
Com componentes ✨
<Header />
<Section />
<Footer />
✔ Código mais organizado ✔ Reutilização ✔ Manutenção mais fácil

├── Header
├── Menu
├── Conteúdo

│    ├── Card

2
Props (Properties)
📌 Definição
Props são dados enviados de um componente pai para um componente filho.
🔁 Fluxo das props
Componente Pai → envia props → Componente Filho
Exemplo (props)
function Aluno(props) {
  return (
    <div>
      <h2>{props.nome}</h2>
      <p>{props.curso}</p>
    </div>
  );
}
Uso
<Aluno nome="Ana" curso="Dev Sistemas" />
Resultado
Ana
Dev Sistemas
Forma moderna (desestruturação)
function Aluno({ nome, curso }) {
  return (
    <div>
      <h2>{nome}</h2>
      <p>{curso}</p>
    </div>
  );
}
✔ Mais limpo ✔ Mais usado em projetos reais
3
Composição de Componentes
Composição significa combinar vários componentes para criar interfaces maiores (tipo LEGO 🧱✨).
function Card({ nome, curso }) {
  return (
    <div>
      <Avatar />
      <Info nome={nome} curso={curso} />
      <Botao />
    </div>
  );
}
Estrutura
Card
├── Avatar
├── Info
└── Botao
4
Renderização Condicional
Renderização condicional permite mostrar elementos dependendo de uma condição.
Exemplo simples
function Status({ logado }) {
  if (logado) {
    return <p>Usuário logado</p>;
  }

  return <p>Faça login</p>;
}
Usando operador ternário
function Status({ logado }) {
  return (
    <p>{logado ? "Usuário logado" : "Usuário não logado"}</p>
  );
}
5
Renderização de Listas + Key
React permite renderizar listas de dados usando map().
Exemplo (dados)
const alunos = [
  { nome: "Ana", idade: 17 },
  { nome: "Lucas", idade: 18 },
  { nome: "Lia", idade: 16 }
];
Renderizando
function Lista() {
  return (
    <div>
      {alunos.map((aluno, index) => (
        <p key={index}>{aluno.nome} - {aluno.idade}</p>
      ))}
    </div>
  );
}
🔑 O que é Key?
A key ajuda o React a identificar quais elementos mudaram. Sem key: ⚠ performance pior • ⚠ possíveis erros de renderização.
{alunos.map((aluno) => (
  <Aluno key={aluno.id} nome={aluno.nome} />
))}
6
Manipulação de Eventos
Eventos permitem reagir às ações do usuário (cliques, digitação etc). 🎮
Evento de clique
function Botao() {
  function clicar() {
    alert("Você clicou!");
  }

  return <button onClick={clicar}>Clique</button>;
}
Evento com parâmetro
<button onClick={() => alert("Olá")}>
  Clique
</button>
⚛️
Principais Tipos de Componentes no React
No React, praticamente tudo na interface é um componente — e cada tipo tem um papel diferente no projeto 🧩
🎯 Foco: organização real de projeto
💡 Dica: reutilizar sempre
1
Componentes Funcionais (Functional Components)
📌 O que são
São funções JavaScript que retornam JSX. Hoje são o padrão moderno do React.
function Header() {
  return <h1>Bem-vindo ao sistema</h1>;
}

<Header />
✔ Simples ✔ Mais usados ✔ Podem usar Hooks ✔ Fáceis de manter
2
Componentes de Classe (Class Components)
📌 O que são
Forma antiga de criar componentes usando classes. Hoje são pouco usados em projetos novos.
import React, { Component } from "react";

class Header extends Component {
  render() {
    return <h1>Bem-vindo</h1>;
  }
}
🕰️ Ciclo de vida tradicional this.state ↔ Substituídos por Hooks
3
Componentes de Apresentação (Presentational / UI)
📌 Função
Servem para exibir informações. Normalmente recebem dados via props e têm pouca lógica.
function ProdutoCard({ nome, preco }) {
  return (
    <div>
      <h3>{nome}</h3>
      <p>R$ {preco}</p>
    </div>
  );
}
✔ Recebem dados via props ✔ Foco na interface ✔ Sem estado complexo
4
Componentes Container (Smart Components)
📌 Função
Responsáveis por lógica e estado. Normalmente passam dados para componentes de UI.
function Produtos() {
  const produtos = [
    { nome: "Mouse", preco: 50 },
    { nome: "Teclado", preco: 120 }
  ];

  return (
    <div>
      {produtos.map((p, i) => (
        <ProdutoCard key={i} nome={p.nome} preco={p.preco} />
      ))}
    </div>
  );
}
🧠 Controlam estado 🌐 Buscam dados (API) 🧰 Gerenciam lógica
5
Componentes de Layout
📌 Função
Servem para organizar a estrutura da interface: Header, Sidebar, Footer, Navbar, Layout geral.
function Layout({ children }) {
  return (
    <div>
      <Header />
      <main>{children}</main>
      <Footer />
    </div>
  );
}
✔ Definem estrutura ✔ Usam children
6
Componentes de Página (Pages)
Representam páginas completas da aplicação. Muito usados com React Router (Home, Login, Dashboard, Perfil).
function Home() {
  return (
    <div>
      <h1>Página Inicial</h1>
      <ListaProdutos />
    </div>
  );
}
7
Componentes Controlados (Forms)
Controlam inputs usando estado (value + onChange). 📝
import { useState } from "react";

function Formulario() {
  const [nome, setNome] = useState("");

  return (
    <input
      value={nome}
      onChange={(e) => setNome(e.target.value)}
    />
  );
}
8
Componentes Reutilizáveis (Reusable Components)
São criados para serem usados em vários lugares do projeto (Button, Card, Modal, Input, Avatar...). ♻️
function Button({ texto }) {
  return <button>{texto}</button>;
}

<Button texto="Salvar" />
<Button texto="Cancelar" />
🎓 Pergunta comum
“Todo componente precisa ser diferente?” Não. O ideal é reutilizar o máximo possível.
<Card titulo="Produto 1" />
<Card titulo="Produto 2" />
<Card titulo="Produto 3" />
