Express.js — Parâmetros de Rota, Query String e Body
Nesta aula você vai entender as três formas de enviar dados para uma API Express: parâmetros de rota, query string e corpo da requisição. Cada uma tem um propósito específico — e usar a forma certa faz toda a diferença na qualidade da sua API.
Capacidades Trabalhadas
 
Aplicar linguagem de programação por meio de apis, bibliotecas, frameworks na construção de rotinas de software.
 
Selecionar ferramentas de gerenciamento na aplicação da metodologia.
Por que existem três formas de enviar dados?
Quando um cliente faz uma requisição para uma API, ele frequentemente precisa informar o que quer ou com quais dados. O Express oferece três maneiras de receber essas informações — cada uma adequada para um tipo de situação diferente.
Analogia: pense em uma pizzaria. Você pode pedir uma pizza específica pelo número da mesa (params), pode filtrar o cardápio pedindo "só as sem glúten" (query), ou pode preencher um formulário inteiro com seu pedido personalizado (body).
Tipo Onde fica na URL / requisição Usado para Acessado via
Route Params /produtos/42 Identificar um recurso específico req.params
Query String /produtos?cat=eletronico Filtrar, buscar, paginar req.query
Body { "nome": "Teclado" } no corpo Criar ou atualizar dados req.body
Tipo 1
Parâmetros de Rota — req.params
Os parâmetros de rota são partes variáveis da própria URL. Eles são declarados com o símbolo : seguido do nome na definição da rota. Quando a requisição chega, o Express preenche automaticamente o objeto req.params com os valores reais.
Anatomia da URL com parâmetro de rota
http://localhost:3000/produtos/42
 
/produtos/ — parte fixa da rota
 
42 — valor do parâmetro :id
Exemplo básico — buscando um produto pelo id
app.get('/produtos/:id', (req, res) => {
  const { id } = req.params;              // id = "42" (sempre string)
  const idNumero = Number(id);          // converte para número
  res.json({ mensagem: `Produto de id ${idNumero}` });
});
⚠️ Atenção — params sempre chegam como string
O valor de req.params.id é sempre do tipo string, mesmo que o cliente tenha enviado um número. Ao comparar com dados que são números, converta com Number() ou parseInt().
Múltiplos parâmetros na mesma rota
Você pode ter mais de um parâmetro na mesma rota. Cada um será uma chave separada dentro de req.params.
Requisição: GET /categorias/eletronicos/produtos/5
app.get('/categorias/:categoria/produtos/:id', (req, res) => {
  const { categoria, id } = req.params;
  // categoria = "eletronicos" | id = "5"
  res.json({ categoria, id });
});
✅ Quando usar Route Params
→ Para identificar um recurso único e específico: /usuarios/7, /posts/meu-titulo
→ Quando o dado faz parte da estrutura da URL, não é opcional
→ Nos métodos GET por id, PUT e DELETE
Tipo 2
Query String — req.query
A query string é um conjunto de pares chave=valor adicionados ao final da URL após o símbolo ?. Para múltiplos parâmetros, separe-os com &. Diferente dos route params, a query string é opcional — a rota funciona mesmo sem ela.
Anatomia da URL com query string
http://localhost:3000/produtos?categoria=eletronico&pagina=2
 
/produtos — rota base
 
? e & — separadores da query string
 
categoria=eletronico — par chave/valor
 
pagina=2 — segundo parâmetro
Exemplo — filtrando produtos por categoria e página
// GET /produtos?categoria=eletronico&pagina=2
app.get('/produtos', (req, res) => {
  const { categoria, pagina } = req.query;

  // categoria = "eletronico" | pagina = "2"
  // se não enviado, o valor é undefined

  res.json({
    filtro: categoria ?? 'todos',
    pagina: Number(pagina) || 1,
  });
});
Boa prática — sempre defina um valor padrão
Como a query string é opcional, o valor pode chegar como undefined. Use o operador ?? (nullish coalescing) ou desestruturação com valor padrão para evitar bugs.
// opção 1 — desestruturação com padrão
const { categoria = 'todos', pagina = '1' } = req.query;

// opção 2 — operador ??
const categoria = req.query.categoria ?? 'todos';
const pagina = req.query.pagina ?? '1';
✅ Quando usar Query String
→ Para filtrar uma listagem: /produtos?categoria=eletronico
→ Para paginar resultados: /produtos?pagina=2&limite=10
→ Para buscar por texto: /produtos?busca=teclado
→ Sempre em requisições GET — nunca para enviar dados sensíveis (a query fica visível na URL)
Tipo 3
Corpo da Requisição — req.body
O body é o corpo da requisição HTTP — é onde ficam os dados estruturados enviados pelo cliente para criar ou atualizar um recurso. O body não aparece na URL; ele é transmitido de forma separada, o que o torna adequado para dados maiores e sensíveis.
🔧 Pré-requisito obrigatório — ativar o parser de JSON
O Express não lê o body automaticamente. Você precisa ativar o middleware express.json() — sem ele, req.body sempre será undefined.
const app = express();
app.use(express.json());   // ← necessário para req.body funcionar
Exemplo — criando um novo usuário com POST
// Body enviado pelo cliente: { "nome": "Ana", "email": "ana@email.com" }

app.post('/usuarios', (req, res) => {
  const { nome, email } = req.body;

  // validação básica
  if (!nome || !email) {
    return res.status(400).json({ erro: 'nome e email são obrigatórios' });
  }

  const novoUsuario = { id: 1, nome, email };
  res.status(201).json(novoUsuario);
});
Combinando body + params — atualizando um recurso
O método PUT usa o :id do params para saber qual recurso atualizar, e o req.body para receber os novos dados.
PUT /usuarios/7 — body: { "nome": "Ana Souza" }
app.put('/usuarios/:id', (req, res) => {
  const { id } = req.params;   // id = "7" — quem atualizar
  const { nome } = req.body;    // nome = "Ana Souza" — o que atualizar
  res.json({ id: Number(id), nome });
});
✅ Quando usar Body
→ Para criar recursos novos — método POST
→ Para atualizar recursos existentes — método PUT ou PATCH
→ Quando os dados são estruturados e complexos (objetos com muitos campos)
→ Quando os dados não devem aparecer na URL (senhas, informações pessoais)
Comparação — qual usar em cada situação?
Essa é uma das dúvidas mais comuns de quem está aprendendo APIs. A regra geral é simples:
req.params
Quando você quer acessar um item específico pelo seu identificador.
GET /produtos/5
DELETE /usuarios/3
PUT /posts/12
req.query
Quando você quer filtrar ou refinar uma listagem de forma opcional.
GET /produtos?cat=tec
GET /posts?pagina=2
GET /usuarios?busca=ana
req.body
Quando você vai criar ou atualizar um recurso com dados estruturados.
POST /produtos
PUT /usuarios/3
PATCH /posts/12
Característica params query body
Visível na URL? ✓ Sim ✓ Sim ✗ Não
Obrigatório? ✓ Sim ✗ Opcional ✓ Sim
Usado em qual método? GET, PUT, DELETE GET POST, PUT, PATCH
Tipo do valor recebido string string qualquer (JSON)
Adequado para dados sensíveis? ✗ Não ✗ Não ✓ Sim
⭐ Exemplo Completo
API de Livros — usando os três tipos de parâmetro
Um CRUD completo onde cada rota demonstra qual tipo de parâmetro usar e por quê.
 
 
 
app.js
import express from 'express';
const app = express();
app.use(express.json());

let livros = [
  { id: 1, titulo: 'O Alquimista', genero: 'romance', preco: 39.9 },
  { id: 2, titulo: 'Duna', genero: 'ficcao', preco: 59.9 },
  { id: 3, titulo: 'Dom Casmurro', genero: 'romance', preco: 29.9 },
];
let proximoId = 4;

// ── req.query ─────────────────────────────────────────────────────
// GET /livros?genero=romance&precoMax=50
app.get('/livros', (req, res) => {
  const { genero, precoMax } = req.query;   // filtros opcionais
  let resultado = livros;
  if (genero) resultado = resultado.filter(l => l.genero === genero);
  if (precoMax) resultado = resultado.filter(l => l.preco <= Number(precoMax));
  res.json(resultado);
});

// ── req.params ────────────────────────────────────────────────────
// GET /livros/2
app.get('/livros/:id', (req, res) => {
  const livro = livros.find(l => l.id === Number(req.params.id));
  if (!livro) return res.status(404).json({ erro: 'Livro não encontrado' });
  res.json(livro);
});

// ── req.body ──────────────────────────────────────────────────────
// POST /livros — body: { "titulo": "...", "genero": "...", "preco": ... }
app.post('/livros', (req, res) => {
  const { titulo, genero, preco } = req.body;
  if (!titulo || !genero || !preco) {
    return res.status(400).json({ erro: 'titulo, genero e preco obrigatórios' });
  }
  const novo = { id: proximoId++, titulo, genero, preco };
  livros.push(novo);
  res.status(201).json(novo);
});

// ── params + body ─────────────────────────────────────────────────
// PUT /livros/2 — body: { "preco": 49.9 }
app.put('/livros/:id', (req, res) => {
  const idx = livros.findIndex(l => l.id === Number(req.params.id));
  if (idx === -1) return res.status(404).json({ erro: 'Livro não encontrado' });
  livros[idx] = { ...livros[idx], ...req.body };      // merge dos dados
  res.json(livros[idx]);
});

// ── params ────────────────────────────────────────────────────────
// DELETE /livros/1
app.delete('/livros/:id', (req, res) => {
  const antes = livros.length;
  livros = livros.filter(l => l.id !== Number(req.params.id));
  if (livros.length === antes) return res.status(404).json({ erro: 'Livro não encontrado' });
  res.status(204).send();
});

app.listen(3000, () => console.log('API rodando em http://localhost:3000'));
