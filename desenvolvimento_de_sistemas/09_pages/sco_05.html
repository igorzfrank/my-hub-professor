Express.js — Organizando Rotas

O problema: tudo no mesmo arquivo
Nas aulas anteriores, criamos uma API funcional com um único arquivo app.js. Isso funciona bem para projetos pequenos, mas conforme a aplicação cresce, esse arquivo vira um caos: dezenas de rotas misturadas, lógica de negócio embutida, e qualquer alteração pode quebrar tudo.
Exemplo do problema — app.js monolítico
import express from 'express';
const app = express();
app.use(express.json());

// --- Rotas de Usuários ---
app.get('/usuarios', (req, res) => { /* ... */ });
app.get('/usuarios/:id', (req, res) => { /* ... */ });
app.post('/usuarios', (req, res) => { /* ... */ });
app.put('/usuarios/:id', (req, res) => { /* ... */ });
app.delete('/usuarios/:id', (req, res) => { /* ... */ });

// --- Rotas de Produtos ---
app.get('/produtos', (req, res) => { /* ... */ });
app.get('/produtos/:id', (req, res) => { /* ... */ });
app.post('/produtos', (req, res) => { /* ... */ });
// ... e mais 30 rotas de pedidos, categorias, pagamentos...
A solução é o Express Router. Ele permite criar um "mini-app" de rotas para cada recurso da sua API e registrá-los no app principal com um prefixo. O resultado é um projeto organizado, onde cada arquivo cuida de exatamente uma responsabilidade.
Conceito 1
O que é o express.Router()?
O express.Router() é uma função que cria um objeto roteador isolado — pense nele como um app Express em miniatura, dedicado a um conjunto de rotas relacionadas. Você define as rotas nele, exporta e registra no app principal com um prefixo.
Como o Router funciona

Cada Router é um arquivo separado — o app.js só conhece o prefixo.
Sintaxe básica
src/routes/usuarios.routes.js
import { Router } from 'express';

// 1. Cria o roteador
const router = Router();

// 2. Define as rotas (sem o prefixo /usuarios)
router.get('/', (req, res) => {
  res.json({ mensagem: 'lista de usuários' });
});

router.get('/:id', (req, res) => {
  res.json({ id: req.params.id });
});

// 3. Exporta o roteador
export default router;
app.js
import express from 'express';
import usuariosRouter from './src/routes/usuarios.routes.js';

const app = express();
app.use(express.json());

// 4. Registra o router com o prefixo
app.use('/usuarios', usuariosRouter);

app.listen(3000);
Como funciona o prefixo? Quando você registra app.use('/usuarios', usuariosRouter), o Express pega o prefixo /usuarios e concatena com os caminhos definidos no router. Então / vira GET /usuarios e /:id vira GET /usuarios/:id.
Conceito 2
Estrutura de Pastas com Router + MVC
Ao usar o Router, a estrutura de pastas do projeto evolui naturalmente. A convenção mais adotada separa o código em quatro camadas principais, que mapeiam diretamente o padrão MVC:
Estrutura de pastas
projeto/
 ├── src/
 │   ├── models/
 │   │   └── produto.model.js
 │   ├── controllers/
 │   │   └── produto.controller.js
 │   └── routes/
 │       └── produto.routes.js
 ├── app.js
 └── package.json
models/
Armazena os dados (array em memória ou conexão com banco). É o M do MVC — responsável pelos dados.
controllers/
Contém a lógica de cada operação. É o C do MVC — recebe a requisição, processa e responde.
routes/
Define quais URLs existem e quais controllers atendem cada uma. É o roteador — faz a ligação.

Conceito 3
Criando um Router — Passo a Passo
Vamos construir juntos um Router completo, passando por cada arquivo da arquitetura.
Passo 1 Configure o projeto
Crie a pasta do projeto, inicialize o npm e instale o Express. Não esqueça de adicionar "type": "module" no package.json para usar ES Modules.
mkdir minha-api && cd minha-api
npm init -y
npm install express
package.json — adicionar a linha "type"
{
  "name": "minha-api",
  "version": "1.0.0",
  "type": "module",
  "main": "app.js"
}
Passo 2 Crie o Model src/models/produto.model.js
O Model é responsável pelos dados. Por enquanto usamos um array em memória, mas futuramente aqui entraria a conexão com o banco de dados. O Model só expõe os dados — nunca toca no req ou no res.
src/models/produto.model.js
// Array que simula o banco de dados
let produtos = [
  { id: 1, nome: 'Teclado Mecânico', preco: 350 },
  { id: 2, nome: 'Mouse Gamer', preco: 180 },
  { id: 3, nome: 'Monitor 24"', preco: 1200 },
];

// Exporta os dados para uso nos controllers
export default produtos;
Passo 3 Crie o Controller src/controllers/produto.controller.js
O Controller contém as funções que respondem às requisições. Cada função recebe (req, res), consulta ou modifica o Model e devolve uma resposta JSON.
src/controllers/produto.controller.js
import produtos from '../models/produto.model.js';

// Listar todos
export const listar = (req, res) => {
  res.json(produtos);
};

// Buscar por ID
export const buscarPorId = (req, res) => {
  const produto = produtos.find(p => p.id === Number(req.params.id));
  if (!produto) return res.status(404).json({ erro: 'Produto não encontrado' });
  res.json(produto);
};

// Criar
export const criar = (req, res) => {
  const { nome, preco } = req.body;
  if (!nome || !preco) return res.status(400).json({ erro: 'nome e preco são obrigatórios' });
  const novoProduto = { id: Date.now(), nome, preco };
  produtos.push(novoProduto);
  res.status(201).json(novoProduto);
};

// Atualizar
export const atualizar = (req, res) => {
  const index = produtos.findIndex(p => p.id === Number(req.params.id));
  if (index === -1) return res.status(404).json({ erro: 'Produto não encontrado' });
  produtos[index] = { ...produtos[index], ...req.body };
  res.json(produtos[index]);
};

// Remover
export const remover = (req, res) => {
  const index = produtos.findIndex(p => p.id === Number(req.params.id));
  if (index === -1) return res.status(404).json({ erro: 'Produto não encontrado' });
  produtos.splice(index, 1);
  res.status(204).send();
};
Passo 4 Crie o Router src/routes/produto.routes.js
O arquivo de rotas é o mais simples dos três: ele apenas conecta URLs a funções do Controller. Perceba que o Controller não fica aqui — apenas é importado e referenciado.
src/routes/produto.routes.js
import { Router } from 'express';
import * as ProdutoController from '../controllers/produto.controller.js';

const router = Router();

// GET /produtos
router.get('/', ProdutoController.listar);

// GET /produtos/:id
router.get('/:id', ProdutoController.buscarPorId);

// POST /produtos
router.post('/', ProdutoController.criar);

// PUT /produtos/:id
router.put('/:id', ProdutoController.atualizar);

// DELETE /produtos/:id
router.delete('/:id', ProdutoController.remover);

export default router;
Passo 5 Registre no app.js app.js
O app.js fica extremamente enxuto: só configura o servidor e registra os routers com seus respectivos prefixos. É fácil ver todas as rotas da API de uma só vez.
app.js
import express from 'express';
import produtoRoutes from './src/routes/produto.routes.js';

const app = express();
app.use(express.json());

// Registro dos routers — aqui fica o mapa da API
app.use('/produtos', produtoRoutes);

// Para adicionar mais recursos, basta importar e registrar:
// app.use('/usuarios', usuarioRoutes);
// app.use('/pedidos', pedidoRoutes);

app.listen(3000, () => {
  console.log('Servidor rodando em http://localhost:3000');
});
Rotas geradas pelo código acima
Método URL completa Controller chamado O que faz
GET /produtos listar() Retorna todos os produtos
GET /produtos/:id buscarPorId() Retorna um produto pelo ID
POST /produtos criar() Cria um novo produto
PUT /produtos/:id atualizar() Atualiza um produto pelo ID
DELETE /produtos/:id remover() Remove um produto pelo ID
Conceito 4
Adicionando múltiplos recursos
A grande vantagem do Router se revela quando a API cresce. Para adicionar um novo recurso — como usuarios — o processo é sempre o mesmo: criar o trio Model + Controller + Router e registrar no app.js com uma única linha.
app.js — com dois recursos
import express from 'express';
import produtoRoutes from './src/routes/produto.routes.js';
import usuarioRoutes from './src/routes/usuario.routes.js';

const app = express();
app.use(express.json());

app.use('/produtos', produtoRoutes);
app.use('/usuarios', usuarioRoutes);

app.listen(3000);
Convenção de nomenclatura: use o padrão recurso.routes.js, recurso.controller.js e recurso.model.js para tornar o projeto previsível e fácil de navegar por qualquer desenvolvedor.
 
Resumo: Responsabilidade de cada arquivo
Arquivo Responsabilidade Conhece NÃO conhece

controller.js Lógica de cada operação req, res, model URLs, rotas
routes.js Mapear URLs a controllers URLs, controller Dados, lógica
app.js Configurar e iniciar o servidor Routers, prefixos Controllers, dados

2 / 4 src/controllers/livro.controller.js Controller — lógica
import livros from '../models/livro.model.js';

// ── CREATE ───────────────────────────────────────────
export const criar = (req, res) => {
  const { titulo, autor, ano } = req.body;

  if (!titulo || !autor || !ano) {
    return res.status(400).json({ erro: 'titulo, autor e ano são obrigatórios' });
  }

  const novoLivro = { id: Date.now(), titulo, autor, ano: Number(ano) };
  livros.push(novoLivro);
  res.status(201).json(novoLivro);
};

// ── READ (todos) ─────────────────────────────────────
export const listar = (req, res) => {
  res.json(livros);
};

// ── READ (um) ─────────────────────────────────────────
export const buscarPorId = (req, res) => {
  const livro = livros.find(l => l.id === Number(req.params.id));
  if (!livro) {
    return res.status(404).json({ erro: 'Livro não encontrado' });
  }
  res.json(livro);
};

// ── UPDATE ───────────────────────────────────────────
export const atualizar = (req, res) => {
  const index = livros.findIndex(l => l.id === Number(req.params.id));
  if (index === -1) {
    return res.status(404).json({ erro: 'Livro não encontrado' });
  }
  livros[index] = { ...livros[index], ...req.body };
  res.json(livros[index]);
};

// ── DELETE ───────────────────────────────────────────
export const remover = (req, res) => {
  const index = livros.findIndex(l => l.id === Number(req.params.id));
  if (index === -1) {
    return res.status(404).json({ erro: 'Livro não encontrado' });
  }
  livros.splice(index, 1);
  res.status(204).send();
};

Pontos de Atenção
Prefixo no app.js, não no Router
As rotas no arquivo routes.js usam '/' e '/:id' — sem o nome do recurso. O prefixo /livros é definido apenas no app.use().
Extensão .js obrigatória nos imports
Com ES Modules, o Node.js exige a extensão .js nos imports locais. Escreva '../models/livro.model.js', não apenas '../models/livro.model'.
app.use(express.json()) antes dos routers
O middleware express.json() deve ser registrado antes de qualquer router — caso contrário, o req.body chegará indefinido nas requisições POST e PUT.
IDs são strings no req.params
Tudo que vem via URL é uma string. Se o seu array usa números como ID, sempre converta: Number(req.params.id). Sem isso, o find() com === nunca vai casar.
