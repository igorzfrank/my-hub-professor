Exercícios progressivos de construção de APIs com Express utilizando a arquitetura MVC (Model · Controller). Todos os exercícios utilizam arrays em memória como fonte de dados — nenhum banco de dados é necessário.
Node.js Express MVC REST ES Modules

Todos os exercícios usam ES Modules e a estrutura de pastas MVC abaixo. Inicialize o projeto e instale o Express antes de começar.

Nível 1
Primeiros passos com MVC
Model · Controller · GET
Ex 01 API de Frutas GET · Model · Controller
Crie uma API simples de frutas aplicando a separação MVC. O Model deve exportar um array com pelo menos 5 frutas (cada fruta com id, nome e cor). Crie um Controller com duas funções e registre as rotas diretamente no app.js.
Rotas esperadas
GET  /frutas        → retorna todas as frutas
GET  /frutas/:id   → retorna uma fruta pelo id
Ex 02 API de Alunos — com POST GET · POST · req.body
Crie uma API de alunos separada em MVC. O Model deve inicializar um array com 3 alunos (cada um com id, nome e nota). O Controller deve ter uma função criar que lê os dados do req.body, gera um novo id automaticamente e retorna o aluno criado com status 201.
Rotas esperadas
GET   /alunos    → retorna todos os alunos
POST  /alunos    → cria novo aluno (body: nome, nota)
Ex 03 API de Produtos — filtro por query GET · req.query · filtro
Crie uma API de produtos separada em MVC. O Model deve ter ao menos 6 produtos com id, nome, categoria e preco. A função listar do Controller deve verificar se req.query.categoria foi enviado e, se sim, filtrar o array antes de responder.
Rotas esperadas
GET  /produtos                 → todos os produtos
GET  /produtos?categoria=x  → filtrado por categoria
GET  /produtos/:id           → produto pelo id

CRUD Completo em MVC
GET · POST · PUT · DELETE · validação
Ex 04 CRUD de Contatos CRUD completo · validação básica
Implemente um CRUD completo de contatos em MVC. Cada contato tem id, nome, telefone e email. As funções de criação e atualização no Controller devem validar se nome e telefone foram enviados, retornando status 400 com mensagem de erro quando ausentes.
Rotas esperadas
GET     /contatos        → lista todos
GET     /contatos/:id    → busca pelo id
POST   /contatos        → cria (valida campos)
PUT    /contatos/:id    → atualiza (valida campos)
DELETE /contatos/:id    → remove pelo id
API REST em MVC com três entidades relacionadas — sem banco de dados.
Construa uma API de agendamentos para uma clínica fictícia. O sistema gerencia pacientes, médicos e consultas usando apenas arrays em memória e arquitetura MVC. Todas as rotas são registradas diretamente no app.js.
Estrutura obrigatória do projeto src/
 ├── models/
 │   ├── paciente.model.js   ← array + exportação
 │   ├── medico.model.js
 │   └── consulta.model.js
 └── controllers/
     ├── paciente.controller.js
     ├── medico.controller.js
     └── consulta.controller.js ← importa os 3 models
📦 Models (campos de cada entidade)
Paciente
id
nome
telefone
Médico
id
nome
especialidade
Consulta
id
pacienteId
medicoId
data · status
🛣️ Rotas a implementar
Pacientes GET   /pacientes    → lista todos
POST  /pacientes    → cadastra (nome obrigatório)
Médicos GET   /medicos                          → lista todos
GET   /medicos?especialidade=cardiologia  → filtra por especialidade
Consultas GET   /consultas  → lista com nome do paciente e do médico embutidos
POST  /consultas  → agenda (valida se paciente e médico existem)
DELETE /consultas/:id  → cancela consulta
⚠️ Regras obrigatórias
1. Ao criar uma consulta, verificar se o pacienteId e o medicoId existem nos respectivos arrays — retornar 404 com mensagem se não existirem.
2. O status inicial de toda consulta criada deve ser sempre "agendada" — definido automaticamente pelo Controller, não pelo cliente.
3. O GET /consultas deve retornar o nome do paciente e do médico embutidos em cada consulta — não apenas os ids.
