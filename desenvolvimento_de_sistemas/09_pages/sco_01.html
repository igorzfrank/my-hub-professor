Antes de tudo — o que é uma API?
API significa Application Programming Interface — uma interface que permite que dois sistemas diferentes se comuniquem. No contexto web, uma API recebe requisições de um cliente (um app mobile, um site, outro sistema) e devolve dados — geralmente em formato JSON.
Analogia: pense num restaurante. O cliente é você na mesa. O garçom é a API — ele recebe o seu pedido, leva para a cozinha (o servidor) e traz de volta o que você pediu. Você nunca entra na cozinha diretamente.

O que é o Express e por que usá-lo?
O Node.js já vem com um módulo nativo chamado http que permite criar servidores. Mas usá-lo diretamente exige muito código. Veja o quanto é necessário apenas para responder uma rota com Node puro — e como o Express simplifica isso:
❌ Node.js puro — verboso
import http from 'node:http';

http.createServer((req, res) => {
  if (req.url === '/ola' &&
      req.method === 'GET') {
    res.writeHead(200, {
      'Content-Type': 'application/json'
    });
    res.end(JSON.stringify(
      { mensagem: 'Olá!' }
    ));
  }
}).listen(3000);
✅ Com Express — simples e direto
import express from 'express';

const app = express();
app.use(express.json());

app.get('/ola', (req, res) => {
  res.json({ mensagem: 'Olá!' });
});

app.listen(3000);

Instalação e configuração do projeto
Para usar o Express, você precisa de um projeto Node.js inicializado e da dependência instalada. Siga os três passos abaixo dentro da pasta do projeto:
1
Inicializar o projeto e instalar o Express
npm init -y
npm install express
2
Adicionar "type": "module" no package.json
Isso permite usar import no lugar de require.
{
  "name": "minha-api",
  "type": "module",
  "dependencies": {
    "express": "^4.18.2"
  }
}
3
Criar o arquivo principal e rodar o servidor
Crie um arquivo app.js na raiz do projeto. Para iniciar o servidor, execute:
node app.js
Anatomia do servidor Express
Todo servidor Express tem a mesma estrutura base. Veja cada linha comentada e o que ela faz:
app.js — estrutura base do servidor
import express from 'express';                      // 1. importa o Express
const app = express();                               // 2. cria a aplicação
app.use(express.json());                            // 3. habilita leitura de JSON

app.get('/', (req, res) => {                         // 4. define uma rota
  res.json({ mensagem: 'Servidor funcionando!' });   // 5. envia resposta JSON
});

app.listen(3000, () => {                               // 6. inicia o servidor
  console.log('Servidor rodando em http://localhost:3000');
});
express()
Cria a instância da aplicação. O objeto app é o ponto central do Express — tudo passa por ele.
express.json()
Configura o Express para ler automaticamente o corpo das requisições em formato JSON. Sem isso, req.body ficaria undefined.
app.get(rota, fn)
Define uma rota que responde ao método HTTP GET. O primeiro argumento é o caminho da URL, o segundo é a função que trata a requisição.
req, res
req (request) representa a requisição que chegou — contém dados enviados pelo cliente. res (response) representa a resposta que será enviada de volta.
app.listen(porta)
Inicia o servidor na porta especificada. Qualquer requisição enviada para localhost:3000 será recebida.
Métodos HTTP — o vocabulário das APIs
Uma API REST se comunica usando os métodos do protocolo HTTP. Cada método tem um significado diferente — eles indicam o que o cliente quer fazer com os dados:
Método O que faz Analogia No Express
GET Buscar dados Pedir o cardápio do restaurante app.get()
POST Criar novo dado Fazer um pedido no restaurante app.post()
PUT Atualizar dado existente Trocar um item do pedido app.put()
DELETE Remover um dado Cancelar o pedido app.delete()
💡 Juntos formam o CRUD
Esses quatro métodos mapeiam diretamente para as quatro operações fundamentais de qualquer sistema: Create (POST) · Read (GET) · Update (PUT) · Delete (DELETE).
Como passar dados para as rotas
Além de definir o método, as rotas podem receber dados de três formas diferentes. Cada uma tem o seu uso correto:
req.params Parâmetro de Rota /rota/:id
Parte variável da própria URL, indicada com :. Usado para identificar um recurso específico, como o id de um produto.
Requisição: GET /produtos/42
app.get('/produtos/:id', (req, res) => {
  const { id } = req.params;  // id = "42"
  res.json({ mensagem: `Produto ${id}` });
});
req.query Query String /rota?chave=valor
Parâmetros opcionais passados após o ? na URL. Muito usados para filtros e buscas sem alterar a estrutura da rota.
Requisição: GET /produtos?categoria=eletronico
app.get('/produtos', (req, res) => {
  const { categoria } = req.query;  // categoria = "eletronico"
  res.json({ categoria });
});
req.body Corpo da Requisição POST / PUT
Dados enviados no corpo da requisição em formato JSON. Usado para criar ou atualizar recursos. Exige que express.json() esteja ativo no servidor.
Requisição: POST /produtos — body: {"nome": "Teclado", "preco": 250}
app.post('/produtos', (req, res) => {
  const { nome, preco } = req.body;
  // nome = "Teclado", preco = 250
  res.status(201).json({ nome, preco });
});
⭐ Exemplo Completo
API de Produtos — CRUD básico
Todas as 4 operações (GET, POST, PUT, DELETE) em um único arquivo, usando um array em memória como banco de dados temporário.
 
 
 
app.js
import express from 'express';

const app = express();
app.use(express.json());

// array simulando um banco de dados
let produtos = [
  { id: 1, nome: 'Teclado', preco: 250 },
  { id: 2, nome: 'Mouse', preco: 89.9 },
];
let proximoId = 3;

// GET /produtos — retorna todos
app.get('/produtos', (req, res) => {
  res.json(produtos);
});

// GET /produtos/:id — retorna um produto pelo id
app.get('/produtos/:id', (req, res) => {
  const produto = produtos.find(p => p.id === Number(req.params.id));
  if (!produto) return res.status(404).json({ erro: 'Produto não encontrado' });
  res.json(produto);
});

// POST /produtos — cria novo produto
app.post('/produtos', (req, res) => {
  const { nome, preco } = req.body;
  if (!nome || !preco) return res.status(400).json({ erro: 'nome e preco obrigatórios' });
  const novo = { id: proximoId++, nome, preco };
  produtos.push(novo);
  res.status(201).json(novo);
});

// PUT /produtos/:id — atualiza um produto
app.put('/produtos/:id', (req, res) => {
  const idx = produtos.findIndex(p => p.id === Number(req.params.id));
  if (idx === -1) return res.status(404).json({ erro: 'Produto não encontrado' });
  produtos[idx] = { ...produtos[idx], ...req.body };
  res.json(produtos[idx]);
});

// DELETE /produtos/:id — remove um produto
app.delete('/produtos/:id', (req, res) => {
  const antes = produtos.length;
  produtos = produtos.filter(p => p.id !== Number(req.params.id));
  if (produtos.length === antes) return res.status(404).json({ erro: 'Produto não encontrado' });
  res.status(204).send();
});

app.listen(3000, () => console.log('API rodando em http://localhost:3000'));
