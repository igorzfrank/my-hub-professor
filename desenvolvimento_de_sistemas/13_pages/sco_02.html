src/models/ produto.model.js
Define a estrutura da tabela
Use sequelize.define('Produto', { ... }, { tableName: 'produtos', timestamps: true }) com os campos abaixo. O Sequelize adicionará id, createdAt e updatedAt automaticamente.
nome:     DataTypes.STRING, allowNull: false
preco:    DataTypes.DECIMAL(10,2), allowNull: false
estoque: DataTypes.INTEGER, defaultValue: 0

3 GET /produtos → deve listar todos. GET /produtos/999 → deve retornar 404.

🎯 Objetivo do exercício
Construir uma API com autenticação completa: usuários se cadastram e fazem login, recebendo um token JWT. Rotas protegidas verificam o token via middleware. Certas rotas exigem que o usuário tenha um role específico (ex: admin) — quem não tiver a permissão recebe 403 Forbidden.

Criar os arquivos do projeto
src/models/ usuario.model.js
⭐ Hook beforeCreate para hash automático
Defina os campos com sequelize.define('Usuario', { ... }):
nome:  DataTypes.STRING, allowNull: false
email:  DataTypes.STRING, allowNull: false, unique: true
senha:  DataTypes.STRING, allowNull: false
role:   DataTypes.ENUM('user', 'admin'), defaultValue: 'user'
Hook beforeCreate: Adicione após o define: Usuario.beforeCreate(async (u) => { u.senha = await bcrypt.hash(u.senha, 10) }). Assim o hash acontece automaticamente ao chamar Usuario.create().
src/middlewares/ auth.middleware.js
⭐ Peça central do exercício
Função 1 — autenticar(req, res, next)
1. Leia o header: req.headers.authorization (formato: Bearer <token>).
2. Se não existir → 401 { erro: 'Token não fornecido' }.
3. Extraia o token: const token = authorization.split(' ')[1].
4. Verifique com jwt.verify(token, process.env.JWT_SECRET) — se inválido → 401 { erro: 'Token inválido' }.
5. Se válido → salve em req.usuario = decoded e chame next().
Função 2 — exigirRole(...roles) → retorna um middleware
Esta função recebe um ou mais roles e retorna um middleware. Exemplo de uso na rota: router.delete('/:id', autenticar, exigirRole('admin'), remover).

Lógica interna: verifique se roles.includes(req.usuario.role). Se não incluir → 403 { erro: 'Acesso negado' }. Se incluir → next().

src/controllers/ auth.controller.js
Cadastro e Login
Função O que deve fazer Status
cadastrar Chama Usuario.create({ nome, email, senha, role }). O Hook faz o hash. Retorne os dados sem a senha. 201
login Busca com findOne({ where: { email } }). Usa bcrypt.compare(senha, usuario.senha). Se ok: jwt.sign({ id, nome, role }, JWT_SECRET, { expiresIn }). Retorna { token }. Se falhar: 401. 200
src/controllers/ usuario.controller.js
Gerenciamento — rotas protegidas
Função O que deve fazer Role Exigida
listar Usuario.findAll({ attributes: { exclude: ['senha'] } }) user / admin
buscarPorId findByPk(id, { attributes: { exclude: ['senha'] } }) user / admin
atualizar Busca por findByPk e chama usuario.update({ nome }) user / admin
