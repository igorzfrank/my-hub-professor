Módulo 1
O que é um ORM e por que usar o Sequelize?
1.1 — O problema de escrever SQL manual
Nas aulas anteriores, toda comunicação com o banco de dados era feita escrevendo strings SQL dentro do código JavaScript. Isso funciona, mas tem desvantagens importantes à medida que o projeto cresce:
SQL manual — desvantagens
  • Queries longas e difíceis de ler
  • Erros de digitação no SQL só aparecem em execução
  • Criar tabelas exige rodar SQL no pgAdmin manualmente
  • Trocar de banco (ex: MySQL) exige reescrever as queries
Com ORM — vantagens
  • Tabelas criadas automaticamente a partir de modelos JS
  • Operações em JavaScript puro — sem strings SQL
  • Validações e tipos declarados no próprio modelo
  • Troca de banco com mínima alteração no código
1.2 — O que é um ORM?
ORM significa Object-Relational Mapper — mapeador objeto-relacional. Ele cria uma camada de abstração entre o código e o banco de dados: você define seus dados como classes JavaScript (chamadas de modelos), e o ORM converte automaticamente as operações nessas classes em comandos SQL.
Analogia: o ORM é como um tradutor simultâneo. Você fala JavaScript — ele traduz para SQL e devolve os resultados já convertidos em objetos JS. Você nunca precisa aprender a "língua" do banco — o tradutor faz isso por você.
1.3 — SQL manual vs. Sequelize
SQL manual (pool.query)
const result = await pool.query(
  'INSERT INTO produtos (nome, preco)'
  ' VALUES ($1, $2) RETURNING *',
  [nome, preco]
);
return result.rows[0];
Com Sequelize
const produto = await Produto.create({
  nome,
  preco
});
return produto;

Instalação e configuração do Sequelize
2.1 — Instalação e estrutura de pastas
O Sequelize precisa de dois pacotes: o próprio ORM (sequelize) e o driver do banco de dados (pg e pg-hstore para PostgreSQL). A estrutura de pastas permanece igual ao que já conhecemos — apenas o conteúdo dos modelos muda.

2.3 — Configurando a conexão: db.js
Em vez de criar um Pool do pg, agora criamos uma instância do Sequelize passando as mesmas informações de conexão. Essa instância é exportada e usada em todos os modelos.
src/database/db.js
import { Sequelize } from 'sequelize';

const sequelize = new Sequelize(
  process.env.DB_NAME,
  process.env.DB_USER,
  process.env.DB_PASSWORD,
  {
    host: process.env.DB_HOST,
    port: process.env.DB_PORT,
    dialect: 'postgres', // diz ao Sequelize qual banco estamos usando
    logging: false, // desativa o log de SQL no terminal
  }
);

export default sequelize;
2.4 — O app.js — sincronizando o banco
Uma diferença crucial: em vez de criar tabelas no pgAdmin, o Sequelize pode criá-las automaticamente com sequelize.sync(). O alter: true atualiza a tabela se o modelo mudou, sem destruir os dados.
app.js
import 'dotenv/config'; // ← sempre primeira linha
import express from 'express';
import sequelize from './src/database/db.js';
import produtoRoutes from './src/routes/produto.routes.js';

const app = express();
app.use(express.json());
app.use('/produtos', produtoRoutes);

const PORT = process.env.PORT || 3000;

// Sincroniza os modelos com o banco antes de iniciar o servidor
sequelize.sync({ alter: true }).then(() => {
  app.listen(PORT, () => {
    console.log(`Servidor rodando na porta ${PORT}`);
  });
});
Sobre o sync()
O alter: true é ideal para desenvolvimento — atualiza colunas sem apagar dados. Em produção, usa-se migrations no lugar do sync. Nunca use force: true em produção — ele apaga e recria todas as tabelas.
Módulo 3
Definindo Modelos — a tabela em JavaScript
3.1 — O que é um Modelo no Sequelize?
Um modelo é uma classe JavaScript que representa uma tabela do banco. Cada propriedade da classe vira uma coluna. O Sequelize cria automaticamente as colunas id (chave primária auto-incremental), createdAt e updatedAt em toda tabela.
src/models/produto.model.js
import { DataTypes } from 'sequelize';
import sequelize from '../database/db.js';

const Produto = sequelize.define('Produto', {

  // Cada chave aqui vira uma coluna na tabela
  nome: {
    type: DataTypes.STRING, // TEXT no banco
    allowNull: false, // NOT NULL
  },
  preco: {
    type: DataTypes.DECIMAL(10, 2), // NUMERIC(10,2)
    allowNull: false,
  },
  quantidade: {
    type: DataTypes.INTEGER,
    defaultValue: 0, // DEFAULT 0
  },

}, {
  tableName: 'produtos', // nome da tabela no banco
  timestamps: true, // cria createdAt e updatedAt automaticamente
});

export default Produto;
3.2 — Tipos de dados mais usados
Sequelize (DataTypes) Equivalente SQL Quando usar
DataTypes.STRING VARCHAR(255) Texto curto: nome, email, título
DataTypes.TEXT TEXT Texto longo: descrição, conteúdo
DataTypes.INTEGER INTEGER Números inteiros: quantidade, idade
DataTypes.DECIMAL(10,2) NUMERIC(10,2) Valores monetários: preço, salário
DataTypes.BOOLEAN BOOLEAN Verdadeiro/falso: ativo, disponivel
DataTypes.DATE TIMESTAMP Datas: data_nascimento, vencimento
3.3 — Opções de coluna mais usadas
Por coluna
allowNull: false    // NOT NULL
unique: true        // UNIQUE
defaultValue: 0    // DEFAULT 0
validate: { isEmail: true }
No segundo objeto (tabela)
tableName: 'produtos'
timestamps: true
paranoid: true     // soft delete
underscored: true // snake_case
Módulo 4
Operações CRUD com o Sequelize
4.1 — Os métodos do Sequelize
Cada operação do CRUD vira um método chamado diretamente no modelo. Compare com o que fazíamos com SQL:
CREATE — inserir
const produto = await Produto.create({
  nome: 'Teclado',
  preco: 350.00,
  quantidade: 10
});
// retorna o objeto criado com id
READ — buscar todos
const produtos = await Produto.findAll();

// com ordenação:
const produtos = await Produto.findAll({
  order: [['nome', 'ASC']]
});
READ — buscar por ID
const produto = await Produto.findByPk(id);

// findByPk = find By Primary Key
// retorna null se não encontrar
 
UPDATE — atualizar
const produto = await Produto.findByPk(id);
if (!produto) return null;

await produto.update({ nome, preco });
return produto;
DELETE — remover
const produto = await Produto.findByPk(id);
if (!produto) return false;
await produto.destroy();
return true;  // em vez de verificar rowCount, verificamos se o objeto existia
4.2 — findAll com filtros (WHERE)
Para filtrar resultados, usamos o where dentro do findAll. O Sequelize importa operadores especiais chamados Op para comparações como "maior que", "menor que" e "contém":
Exemplos de filtro com Op
import { Op } from 'sequelize';

// Produtos com preço maior que 100
Produto.findAll({ where: { preco: { [Op.gt]: 100 } } });

// Produtos cujo nome contenha "teclado" (case insensitive)
Produto.findAll({ where: { nome: { [Op.iLike]: '%teclado%' } } });

// Buscar por email exato (equals — padrão, sem Op)
Usuario.findOne({ where: { email } });
Módulo 5
O Controller com Sequelize
O Controller muda pouco em relação ao que já conhecemos — o try/catch e o async/await permanecem. A diferença é que agora chamamos métodos do modelo Sequelize em vez de funções do Model manual. Note que não há mais um arquivo model.js separado — o modelo Sequelize já é o Model.
src/controllers/produto.controller.js
import Produto from '../models/produto.model.js';

// ── Listar todos ──────────────────────────────────
export async function listar(req, res) {
  try {
    const produtos = await Produto.findAll({ order: [['nome', 'ASC']] });
    res.json(produtos);
  } catch { res.status(500).json({ erro: 'Erro interno' }); }
}

// ── Buscar por ID ──────────────────────────────────
export async function buscarPorId(req, res) {
  try {
    const produto = await Produto.findByPk(req.params.id);
    if (!produto) return res.status(404).json({ erro: 'Produto não encontrado' });
    res.json(produto);
  } catch { res.status(500).json({ erro: 'Erro interno' }); }
}

// ── Criar ───────────────────────────────────────────
export async function criar(req, res) {
  try {
    const { nome, preco, quantidade } = req.body;
    const produto = await Produto.create({ nome, preco, quantidade });
    res.status(201).json(produto);
  } catch { res.status(500).json({ erro: 'Erro interno' }); }
}

// ── Atualizar ───────────────────────────────────────
export async function atualizar(req, res) {
  try {
    const produto = await Produto.findByPk(req.params.id);
    if (!produto) return res.status(404).json({ erro: 'Produto não encontrado' });
    const { nome, preco, quantidade } = req.body;
    await produto.update({ nome, preco, quantidade });
    res.json(produto);
  } catch { res.status(500).json({ erro: 'Erro interno' }); }
}

// ── Remover ─────────────────────────────────────────
export async function remover(req, res) {
  try {
    const produto = await Produto.findByPk(req.params.id);
    if (!produto) return res.status(404).json({ erro: 'Produto não encontrado' });
    await produto.destroy();
    res.status(204).send();
  } catch { res.status(500).json({ erro: 'Erro interno' }); }
}

O que mudou em relação ao SQL manual
Aspecto SQL manual (pg) Sequelize (ORM)
Criar tabela Rodar SQL no pgAdmin sequelize.sync() cria automaticamente
Conexão new Pool({ host, port... }) new Sequelize(db, user, pass, { dialect })
Inserir pool.query('INSERT INTO ... VALUES ($1) RETURNING *') Modelo.create({ ... })
Buscar por ID pool.query('SELECT * WHERE id = $1') → rows[0] Modelo.findByPk(id)
Atualizar pool.query('UPDATE ... RETURNING *') instancia.update({ ... })
Deletar pool.query('DELETE ...') → rowCount instancia.destroy()
