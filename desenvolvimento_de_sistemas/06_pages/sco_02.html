1 🌐 O Que é uma API?
📌 Conceito
API = Application Programming Interface
É uma forma de um sistema conversar com outro — trocar dados de forma padronizada pela internet.
📸Instagram busca posts da API
🛵iFood busca restaurantes da API
🎵Spotify busca músicas da API
🍽️
Analogia: API = Garçom
1Você pede (faz a requisição)
2Garçom leva ao cozinheiro (servidor)
3Ele traz o dado (resposta JSON)
2 🧩 A PokéAPI — o que vamos usar
🔗 API pública gratuita
https://pokeapi.co/
Teste no navegador:
https://pokeapi.co/api/v2/pokemon/pikachu
📦 Resposta JSON que você receberá:
{

  "height": 4,

}

Nome do Pokémon
height
Altura (em décimos de metro)
weight
Peso (em décimos de kg)
sprites.front_default
URL da imagem do Pokémon

5 💻 Primeiro Exemplo: App.jsx
 
 
 
App.jsx
import { useEffect, useState } from "react";

export default function App() {
  const [pokemon, setPokemon] = useState(null);

  useEffect(() => {
    fetch("https://pokeapi.co/api/v2/pokemon/pikachu")
      .then((res) => res.json())
      .then((data) => setPokemon(data));
  }, []);

  return (
    <div>
      <h1>Pokédex</h1>
      {pokemon && (
        <div>
          <p>Nome: {pokemon.name}</p>
          <p>Peso: {pokemon.weight}</p>
        </div>
      )}
    </div>
  );
}
🧠Explicação linha por linha
useState
Guarda os dados do Pokémon. Começa como null (vazio) e é preenchido após a resposta da API.
useEffect
Executa quando o componente carrega. O [] no final significa: execute apenas uma vez.
fetch()
Faz a requisição HTTP para a API. O .then(res => res.json()) converte a resposta para objeto JavaScript.
setPokemon
Salva os dados no estado → React detecta a mudança → atualiza a tela automaticamente.
pokemon &&
Renderização condicional: só exibe o card quando o pokemon não for mais null (dados carregados).
6 ⏳ Loading — Boas Práticas
Enquanto os dados não chegam, exiba uma mensagem ao usuário.
if (!pokemon) {
  return <p>Carregando...</p>;
}
💡Isso evita erros ao tentar acessar pokemon.name antes dos dados chegarem
7 🧯 Tratamento de Erro
Use .catch() para tratar falhas na requisição.
fetch("...url...")
  .then((res) => res.json())
  .then((data) => setPokemon(data))
  .catch(() =>
    console.log("Erro ao buscar")
  );
💡Em produção: salve o erro no estado e mostre mensagem ao usuário
8 🎨 Componente Separado — Boa Prática
Separar a exibição em um componente próprio deixa o código mais organizado, reutilizável e legível. O App gerencia dados; o PokemonCard apenas exibe.
PokemonCard.jsx
export default function PokemonCard({ pokemon }) {
  return (
    <div style={{ border: "1px solid #ccc" }}>
      <img src={pokemon.sprites.front_default} />
      <h2>{pokemon.name}</h2>
      <p>{pokemon.weight}</p>
    </div>
  );
}
App.jsx (atualizado)
import PokemonCard from "./PokemonCard";

// ... useState e useEffect ...

if (!pokemon) return <p>Carregando...</p>;

return (
  <div>
    <h1>Pokédex</h1>
    <PokemonCard pokemon={pokemon} />
  </div>
);
9 🔍 Buscando Pokémon Dinâmico
Em vez de fixar um Pokémon no código, use um estado para o nome e deixe o usuário escolher qual buscar.
Estado para o nome:
const [nome, setNome] =
  useState("pikachu");
URL dinâmica com template string:
fetch(
  `...pokemon/${nome}`
)
💡Template string ` ` com ${variavel} insere o valor dinamicamente na URL
