1-5 🔀 if/else BÁSICO
1
Maior de idade
Crie uma variável let idade = 17. Use if/else para verificar se a pessoa é maior de idade (>= 18). Exiba "Maior de idade" ou "Menor de idade". Teste com diferentes valores.
2
Número positivo ou negativo
Crie uma variável let numero = -5. Use if/else para verificar se o número é positivo (> 0), negativo (< 0) ou zero (=== 0). Exiba a classificação correta.
3
Par ou ímpar
Crie uma variável let valor = 7. Use if/else com o operador % (módulo) para verificar se o número é par ou ímpar. Exiba: "O número X é par" ou "O número X é ímpar". Teste com outros valores.
4
Aprovado ou reprovado
Crie let nota = 6.5. Use if/else para verificar se a nota é maior ou igual a 7. Se for, exiba "Aprovado". Caso contrário, exiba "Reprovado". Teste com nota 7.0 e 4.5.
5
Vogal ou consoante
Crie let letra = 'e'. Use if/else com o operador || (OU) para verificar se a letra é uma vogal (a, e, i, o, u). Exiba "Vogal" ou "Consoante". Dica: compare com cada vogal usando ||.
6-10 🔗 else if E TERNÁRIO
6
Classificação de idade
Crie let idade = 15. Use if/else if/else para classificar: "Criança" (0-12), "Adolescente" (13-17), "Adulto" (18-64), "Idoso" (65+). Teste com pelo menos 4 idades diferentes.
7
Classificação de notas escolares
Crie let nota = 8.5. Use if/else if/else: nota >= 9 → "Excelente", nota >= 7 → "Bom", nota >= 5 → "Regular", abaixo → "Insuficiente". Cuidado com a ordem das condições!
8
Cálculo de desconto
Crie let valorCompra = 250. Use if/else if/else: compras acima de 500 → 20% de desconto, acima de 200 → 10%, acima de 100 → 5%, abaixo → sem desconto. Calcule e exiba o valor final.
9
Operador ternário simples
Crie let logado = true. Use o operador ternário para armazenar em uma variável mensagem o texto "Bem-vindo!" (se true) ou "Faça login" (se false). Exiba a mensagem no console.
10
Ternário com número
Crie let numero = 42. Use o operador ternário para verificar se o número é par ou ímpar (use % === 0) e armazene o resultado em uma variável. Exiba: "O número X é [par/ímpar]".
11-15 🔀 switch/case
11
Dias da semana com switch
Crie let dia = 3. Use switch para exibir o nome do dia (1 = "Domingo", 2 = "Segunda"... 7 = "Sábado"). Inclua default para valores inválidos. Teste com dia = 1, 4, 7 e 10.
12
Meses do ano com switch
Crie let mes = 6. Use switch para exibir o nome do mês (1 = "Janeiro", 2 = "Fevereiro"... 12 = "Dezembro"). Teste com vários valores incluindo 13 para testar o default.
13
Status de pedido com switch
Crie let status = 'entregue'. Use switch com strings: 'pendente' → "Pagamento não confirmado", 'enviado' → "A caminho", 'entregue' → "Finalizado". Inclua default. Não esqueça do break!
14
Cargos de funcionários
Crie let cargo = 'gerente'. Use switch para exibir o salário base: 'estagiario' → R$ 1500, 'analista' → R$ 4000, 'gerente' → R$ 8000, 'diretor' → R$ 15000. Use default para "Cargo não encontrado".
15
Mini projeto: Calculadora com switch
Crie let num1 = 15, num2 = 5, operador = '/'. Use switch no operador para: '+' → soma, '-' → subtração, '*' → multiplicação, '/' → divisão (verifique se num2 !== 0 antes). Exiba o resultado ou "Divisão por zero não permitida" ou "Operador inválido". Teste com todos os operadores.
💡 DICAS PARA RESOLUÇÃO
✅ O que você deve praticar
🔀 Usar if/else para dois caminhos
🔗 Encadear else if para múltiplas condições
❓ Usar operador ternário (? :) em uma linha
🔀 Usar switch/case para muitos valores
🛑 Usar break em cada case
📋 Incluir default para casos não previstos
📐 Ordenar condições da mais restritiva para a mais ampla
⚠️ Erros comuns para evitar
❌ Esquecer as chaves { } no if/else
❌ Usar = (atribuição) em vez de === (comparação)
❌ Colocar condições na ordem errada no else if
❌ Esquecer o break no switch (cai para o próximo case)
❌ Usar ternário para lógicas muito complexas
❌ Esquecer o default no switch para casos inesperados
❌ Divisão por zero sem verificação prévia
💡 Dica de ouro: Condicionais são o coração da lógica de programação. Praticar if/else, else if, ternário e switch vai te preparar para qualquer desafio de tomada de decisão em código. Varie os valores de entrada para testar todos os caminhos possíveis! 🚀
