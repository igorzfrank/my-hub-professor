1 🔀 if E else — O Coração das Decisões
🧠 O programa decide o caminho
Até agora nossos programas executavam todas as linhas, do início ao fim. Com condicionais, o código toma decisões baseadas em condições que resultam em true ou false. É assim que um site sabe se você está logado, se um campo foi preenchido ou se um desconto deve ser aplicado.
📐Fluxograma de decisão
INÍCIO
idade >= 18?
✅ true
"Maior de idade"
❌ false
"Menor de idade"
// Estrutura básica do if/else
let idade = 18;

if (idade >= 18) {
  console.log('Maior de idade');
} else {
  console.log('Menor de idade');
}
🧠 Como o if funciona
O if avalia a condição entre parênteses. Se for true, executa o bloco de código. Se for false, pula para o else (quando existe) ou simplesmente segue adiante. O else é opcional — você pode usar if sozinho quando só precisa agir em caso positivo.
2 🔗 else if E OPERADOR TERNÁRIO
1else if — Múltiplas condições
Quando você tem mais de dois caminhos possíveis, encadeie else if para testar condições adicionais. O primeiro bloco cuja condição for true é executado — os demais são ignorados.
let nota = 7.5;

if (nota >= 9) {
  console.log('Excelente!');
} else if (nota >= 7) {
  console.log('Aprovado');
} else if (nota >= 5) {
  console.log('Recuperação');
} else {
  console.log('Reprovado');
}
⚠️ Atenção à ordem: As condições são testadas de cima para baixo. Se você colocar nota >= 5 antes de nota >= 9, o programa nunca chegará a exibir "Excelente". Coloque as condições mais restritivas primeiro.
2Operador Ternário — if em uma linha
Para decisões simples, o operador ternário é uma forma compacta de escrever if/else. Sintaxe: condição ? valorSeTrue : valorSeFalse.
let idade = 20;
let status = idade >= 18 ? 'Maior' : 'Menor';

// Equivale a:
// if (idade >= 18) { status = 'Maior' }
// else { status = 'Menor' }
💡 Quando usar ternário: Ideal para decisões simples e diretas, como atribuir um valor com base em uma condição. Evite ternários aninhados (um dentro do outro) — eles tornam o código difícil de ler.
3 🔀 switch — Quando São Muitos Casos
🧠 Quando usar switch em vez de if?
O switch é ideal quando você tem uma única variável sendo comparada com vários valores diferentes. Ele compara o valor com cada case e executa o bloco correspondente. Mais limpo que vários else if encadeados para a mesma variável.
📝 Sintaxe do switch
let dia = 3;

switch (dia) {
  case 1:
    console.log('Segunda');
    break;
  case 2:
    console.log('Terça');
    break;
  default:
    console.log('Outro');
}
🧩 Elementos do switch
switch(variável) — A variável a ser comparada
case valor: — Um valor possível
break; — Interrompe e sai do switch
default: — Caso nenhum case corresponda (opcional, como o else)

⚠️ Sem break, o código "cai" para o próximo case!
// Exemplo real: classificando um pedido
let status = 'enviado';

switch (status) {
  case 'pendente':
    console.log('Aguardando pagamento');
    break;
  case 'enviado':
    console.log('Pedido a caminho');
    break;
  case 'entregue':
    console.log('Pedido finalizado');
    break;
  default:
    console.log('Status desconhecido');
}
4 ✏️ EXERCÍCIOS PRÁTICOS
📝 Pratique no console ou em um arquivo .js
Exercício 1 — Par ou ímpar
Crie uma variável numero. Use if/else com o operador % para verificar se o número é par ou ímpar. Exiba: "O número X é par" ou "O número X é ímpar".
Exercício 2 — Classificação de idade
Usando if/else if/else, classifique uma idade: "Criança" (0-12), "Adolescente" (13-17), "Adulto" (18-64), "Idoso" (65+). Teste com diferentes valores.
Exercício 3 — Operador ternário
Crie uma variável logado = true. Use o operador ternário para exibir "Bem-vindo de volta!" se true, ou "Faça login" se false.
Exercício 4 — Switch de dias
Crie uma variável diaSemana de 1 a 7. Use switch para exibir o nome do dia correspondente (1 = "Domingo", 2 = "Segunda"...). Inclua um default para valores inválidos.
💡 Desafio extra: Crie uma mini calculadora: variáveis num1, num2 e operador (+, -, *, /). Use switch no operador para realizar a conta correspondente e exibir o resultado. Se o operador não for válido, exiba "Operador inválido". Teste com diferentes operadores e também com divisão por zero (dica: verifique com if antes de dividir).
