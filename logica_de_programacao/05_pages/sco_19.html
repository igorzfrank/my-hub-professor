DOM Eventos classList Prática
1-5 🔍 SELECIONANDO E LENDO ELEMENTOS
1
Selecionando por tag
Crie um HTML com um <h1>, dois <p> e um <button>. Use querySelector para selecionar o h1 e exiba seu texto no console com textContent.
2
Selecionando por classe
No mesmo HTML, adicione uma classe "destaque" a um dos parágrafos. Use querySelector('.destaque') para selecioná-lo e exiba seu texto no console.
3
Selecionando por ID
Adicione um id="botaoPrincipal" ao botão. Use querySelector('#botaoPrincipal') para selecioná-lo e exiba no console o texto do botão com textContent.
4
Selecionando múltiplos elementos
Use querySelectorAll('p') para selecionar todos os parágrafos. Percorra a lista com forEach e exiba o texto de cada um no console.
5
Lendo e exibindo no console
Crie um HTML com <h2> contendo o texto "Bem-vindo". Selecione o elemento e exiba no console: o texto (textContent), o conteúdo HTML (innerHTML) e o estilo de cor (style.color).
6-10 ✏️ MODIFICANDO CONTEÚDO E ESTILO
6
Alterando texto com textContent
Crie um HTML com um <h1> e um botão "Alterar". Ao clicar no botão, use textContent para mudar o texto do h1 para "Texto modificado!".
7
Inserindo HTML com innerHTML
Crie uma <div> vazia e um botão. Ao clicar, use innerHTML para inserir dentro da div: <strong>Texto em negrito</strong> e <em>itálico</em>.
8
Mudando estilos com style
Crie um parágrafo e três botões: "Vermelho", "Azul" e "Verde". Ao clicar em cada botão, use style.color para mudar a cor do texto do parágrafo para a cor correspondente.
9
Alterando múltiplos estilos
Crie um parágrafo e um botão "Destacar". Ao clicar, use style para aplicar: cor de fundo amarela, borda preta de 2px e padding de 10px no parágrafo.
10
Alternando com toggle
Crie um parágrafo e um botão "Mostrar/Esconder". No CSS, crie a classe .escondido { display: none; }. Use classList.toggle('escondido') para alternar a visibilidade.
11-15 🏗️ CRIANDO E REMOVENDO ELEMENTOS
11
Criando um elemento simples
Crie uma <ul> vazia e um botão. Ao clicar, use createElement('li') para criar um item de lista com o texto "Novo item" e adicione à lista com appendChild.
12
Removendo um elemento
Crie um parágrafo com um texto qualquer e um botão "Remover". Ao clicar, use .remove() para remover o parágrafo da página.
13
Contador com eventos
Crie um <span> exibindo "0" e dois botões: "+" e "-". Ao clicar, aumente ou diminua o número exibido. Dica: use uma variável let contador = 0 e atualize o textContent do span.
14
classList com múltiplas classes
Crie uma <div> e três botões: "Ativar", "Desativar" e "Verificar". Ao clicar em Ativar: use classList.add('ativo'). Desativar: classList.remove('ativo'). Verificar: exiba no console o resultado de classList.contains('ativo').
15
Mini projeto: Lista de tarefas completa
Crie um HTML com: um <input>, um botão "Adicionar" e uma <ul> vazia. Ao clicar no botão: (1) pegue o valor do input com .value, (2) crie um <li> com esse texto, (3) adicione à lista, (4) limpe o input. Depois faça cada <li> ter um evento de clique que remove o item clicado com .remove(). Dica: adicione o evento no momento da criação do elemento.
💡 DICAS PARA RESOLUÇÃO
✅ O que você deve praticar
🔍 Selecionar com querySelector e querySelectorAll
📖 Ler conteúdo com textContent e innerHTML
✏️ Alterar conteúdo e estilo com style
🖱️ Usar addEventListener para responder a cliques
🎨 Manipular classes com classList.add/remove/toggle/contains
🏗️ Criar elementos com createElement e appendChild
🗑️ Remover elementos com .remove()
⚠️ Erros comuns para evitar
❌ Esquecer o . ou # no querySelector
❌ Tentar usar forEach em querySelector (só funciona com querySelectorAll)
❌ Esquecer de colocar o script antes do fechamento do </body>
❌ Usar textContent esperando que renderize HTML (use innerHTML)
❌ Achar que classList.add substitui todas as classes (ele só adiciona)
❌ Tentar manipular elementos que ainda não foram carregados
❌ Esquecer de limpar o input após adicionar o item
💡 Dica de ouro: A manipulação do DOM é o que torna suas páginas interativas. Saber selecionar elementos, responder a eventos e modificar a página dinamicamente é o que diferencia um site estático de uma aplicação web real. Cada exercício aqui te aproxima de construir projetos completos! 🚀
