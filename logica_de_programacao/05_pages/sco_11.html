Console Variáveis let/const Prática
1-5 💻 CONSOLE E PRIMEIROS COMANDOS
1
Primeiro contato com o console
Abra o console do navegador (F12) e digite: console.log("Estou aprendendo JavaScript!"). Depois, execute os comandos: console.log(100), console.log(10 + 5) e console.log("Resultado:", 10 * 3).
2
Explorando o console.log
Use console.log para exibir: seu nome completo, sua idade, sua cidade e uma frase que junte todas essas informações em uma única linha, separadas por vírgula.
3
Criando seu primeiro arquivo
Crie um arquivo index.html com a estrutura básica (html, head, body). Dentro da tag <script>, coloque um console.log("Meu primeiro script!"). Abra o arquivo no navegador e verifique o console.
4
Arquivo externo .js
Separe o código JavaScript do HTML: crie um arquivo script.js com o console.log e vincule ao HTML usando <script src="script.js"></script>. Teste para ver se funciona igual.
5
Calculadora no console
Usando apenas console.log, calcule e exiba o resultado de: 25 + 15, 100 - 45, 12 * 8, 144 / 12, e o resto da divisão de 17 por 5 (use o operador %).
6-10 📦 VARIÁVEIS COM let E const
6
Declarando variáveis com let
Crie 3 variáveis usando let: uma para armazenar o nome de um filme, outra para o ano de lançamento e outra para a nota (número decimal). Exiba todas no console.
7
Declarando constantes com const
Crie 3 constantes usando const: uma para o nome do seu país, outra para o valor de PI (3.14159) e outra para o número de dias da semana (7). Exiba todas no console.
8
Diferença entre let e const
Crie uma variável let pontuacao = 100 e altere seu valor para 150. Crie uma constante const nomeJogo = "Xadrez" e tente alterá-la. O que acontece? Anote o erro que aparece.
9
Criando uma ficha pessoal
Crie variáveis para: nome (let), idade (let), anoNascimento (const), cidade (let) e profissao (const). Atribua valores e exiba tudo no console com frases descritivas.
10
Variáveis com template literals
Usando as variáveis do exercício 9, crie uma única frase usando template literals (crases e ${}): "Meu nome é [nome], tenho [idade] anos, nasci em [anoNascimento], moro em [cidade] e sou [profissao]."
11-15 📝 COMENTÁRIOS E BOAS PRÁTICAS
11
Comentário de uma linha
Crie um script com 5 linhas de código. Acima de cada linha, escreva um comentário // explicando o que aquela linha faz. Exemplo: // Exibe uma mensagem no console seguido do console.log.
12
Comentário de múltiplas linhas
No topo do seu script, adicione um bloco de comentário /* */ com: seu nome, data, e uma breve descrição do que o script faz. Depois crie as variáveis e comandos normalmente.
13
Organizando o código
Crie um script com 3 seções separadas por comentários: // === DADOS PESSOAIS ===, // === DADOS DO CURSO === e // === EXIBIÇÃO ===. Em cada seção, crie variáveis relacionadas ao tema.
14
Depurando com console.log
Crie 5 variáveis diferentes (nome, idade, altura, casado, temFilhos). Use console.log para exibir o valor de cada uma ANTES de atribuir os valores. Depois atribua valores e exiba novamente. Observe a diferença (undefined vs valores).
15
Mini projeto: Carta de apresentação
Crie um script completo com: (1) cabeçalho de comentário com seu nome e data, (2) variáveis let para nome, idade, hobby, (3) constantes const para cidade natal e profissão dos sonhos, (4) console.log com uma carta de apresentação completa usando template literals. Seja criativo!
💡 DICAS PARA RESOLUÇÃO
✅ O que você deve praticar
🖥️ Abrir o console (F12) e testar comandos
📄 Criar arquivos HTML com tag <script>
📁 Separar código em arquivos .js externos
📦 Declarar variáveis com let
🔒 Declarar constantes com const
💬 Usar comentários // e /* */
📝 Exibir dados com console.log()
🔤 Concatenar com template literals (`${}`)
⚠️ Erros comuns para evitar
❌ Esquecer de abrir/fechar aspas em strings
❌ Usar var em vez de let/const
❌ Tentar alterar o valor de uma const
❌ Esquecer o <script> no HTML
❌ Caminho errado no src do arquivo .js
❌ Console fechado e achar que o código não rodou
❌ Usar crase (`) achando que é aspas simples (')
💡 Dica de ouro: Faça os exercícios na ordem! Cada um prepara o terreno para o próximo. Se travar em algum, volte na aula teórica e revise o conteúdo antes de continuar. O console do navegador é seu laboratório — use e abuse dele para testar! 🚀
