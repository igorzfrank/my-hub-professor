Estrutura CASE no Portugol
ESCOLHA...CASO — Decisões Múltiplas
Aprenda a substituir longas cadeias de SE...SENÃO SE por uma estrutura limpa e organizada
escolha caso pare caso contrario múltipla escolha
1 🧠 O que é a Estrutura CASE?
🎯 Definição
A estrutura ESCOLHA...CASO (também chamada de CASE) é uma estrutura de decisão que permite testar uma variável contra múltiplos valores fixos e executar um bloco de código diferente para cada valor. É uma alternativa muito mais limpa e organizada do que usar vários SE...SENÃO SE encadeados.
💡 Analogia simples
Imagine um painel de elevador com botões numerados. Cada botão (caso) leva a um andar diferente (ação). O elevador testa qual botão foi pressionado e executa a ação correspondente. É muito mais eficiente do que perguntar: "É o andar 1? Não. É o andar 2? Não. É o andar 3? Sim!"
2 📝 Sintaxe da Estrutura ESCOLHA...CASO
escolha (variavel) {
  caso valor1:
    // bloco de código
    pare
  caso valor2:
    // outro bloco
    pare
  caso contrario:
    // valor não listado
    pare
}
🧠 Regras importantes
escolha (variavel) { } → inicia a estrutura com a variável a ser testada entre parênteses.
caso valor: → define um valor específico a ser comparado. Os dois-pontos são obrigatórios.
pare → encerra o bloco de cada caso. Obrigatório no Portugol.
caso contrario: → captura qualquer valor não listado (equivalente ao SENÃO). Opcional.
⚠️ Atenção: Diferente do pseudocódigo (que usa FIM_ESCOLHA), no Portugol a estrutura é delimitada por chaves { } e cada caso é encerrado com a palavra pare. Não esquecer o pare é um erro comum!
3 🎯 Quando Usar CASE vs. SE...SENÃO SE?
✅ Use ESCOLHA...CASO quando:
✅ A variável tem valores exatos e conhecidos
✅ São muitas opções (3 ou mais)
✅ Os valores são do tipo inteiro ou cadeia
✅ Você quer código mais limpo e legível
✅ Cada valor executa uma ação diferente
✅ Exemplos: menu de opções, dias da semana, categorias
❌ Use SE...SENÃO SE quando:
❌ A condição envolve intervalos (ex: nota > 7)
❌ A condição usa operadores lógicos (E, OU)
❌ São apenas duas opções (melhor SE...SENÃO)
❌ A condição envolve comparações (>, <, >=, <=)
❌ Você precisa testar variáveis diferentes
❌ Exemplos: faixa etária, média de notas, validações
💡 Regra prática: Se você está comparando uma mesma variável com vários valores fixos, use ESCOLHA...CASO. Se precisa de intervalos ou operadores lógicos, use SE...SENÃO SE.
4 🔄 Tradução Lado a Lado — Pseudocódigo → Portugol
📅 Exemplo — Dia da Semana (Pseudocódigo → Portugol)
📝 Pseudocódigo

ESCOLHA dia
  CASO 1: ESCREVER "Domingo"
  CASO 2: ESCREVER "Segunda"
  CASO 3: ESCREVER "Terça"
  CASO 4: ESCREVER "Quarta"
  CASO 5: ESCREVER "Quinta"
  CASO 6: ESCREVER "Sexta"
  CASO 7: ESCREVER "Sábado"
  SENÃO: ESCREVER "Inválido"
FIM_ESCOLHA
💻 Portugol

escolha (dia) {
  caso 1:
    escreva("Domingo")
    pare
  caso 2:
    escreva("Segunda")
    pare
  caso 3:
    escreva("Terça")
    pare
  caso contrario:
    escreva("Inválido")
    pare
}
🔍 Principais diferenças: Parênteses na variável, chaves { } delimitando o bloco, dois-pontos após cada caso, pare obrigatório ao final de cada bloco, e caso contrario em vez de SENÃO.
5 👁️ Comparativo Visual — Mesmo Problema, Duas Abordagens
Veja como o mesmo problema (classificar um número de 1 a 3) fica com cada abordagem:
❌ Com SE...SENÃO SE (confuso)
se (opcao == 1) {
  escreva("Opção 1")
} senao se (opcao == 2) {
  escreva("Opção 2")
} senao se (opcao == 3) {
  escreva("Opção 3")
} senao {
  escreva("Inválido")
}
✅ Com ESCOLHA...CASO (limpo)
escolha (opcao) {
  caso 1:
    escreva("Opção 1")
    pare
  caso 2:
    escreva("Opção 2")
    pare
  caso 3:
    escreva("Opção 3")
    pare
  caso contrario:
    escreva("Inválido")
    pare
}
✅ Vantagem do CASE: Código mais organizado, não repete o nome da variável em cada condição, e fica visualmente claro que todas as opções são mutuamente exclusivas.
6 🧪 Exemplos Práticos Completos
ACardápio de Restaurante
Leia um número de 1 a 5 e exiba o prato correspondente com preço.
programa {
  funcao inicio() {
    inteiro opcao
    escreva("Menu: 1-Hambúrguer 2-Pizza 3-Salada 4-Lasanha 5-Sopa\n")
    escreva("Escolha: ")
    leia(opcao)

    escolha (opcao) {
      caso 1:
        escreva("Hambúrguer — R$ 25,00")
        pare
      caso 2:
        escreva("Pizza — R$ 35,00")
        pare
      caso 3:
        escreva("Salada — R$ 18,00")
        pare
      caso 4:
        escreva("Lasanha — R$ 28,00")
        pare
      caso 5:
        escreva("Sopa — R$ 15,00")
        pare
      caso contrario:
        escreva("Opção inválida")
        pare
    }
  }
}
BCategoria de Habilitação (DETRAN)
Leia uma letra (A a E) e exiba qual veículo a pessoa pode dirigir. O CASE funciona com cadeia também!
programa {
  funcao inicio() {
    cadeia categoria
    escreva("Categoria (A/B/C/D/E): ")
    leia(categoria)

    escolha (categoria) {
      caso "A": escreva("Moto") pare
      caso "B": escreva("Carro") pare
      caso "C": escreva("Caminhão") pare
      caso "D": escreva("Ônibus") pare
      caso "E": escreva("Carreta") pare
      caso contrario: escreva("Inválida") pare
    }
  }
}
CClassificação de Conceito Escolar
Leia um conceito (A, B, C, D) e exiba a descrição correspondente. Use validação antes do CASE.
programa {
  funcao inicio() {
    cadeia conceito
    escreva("Conceito (A/B/C/D): ")
    leia(conceito)

    // Validação antes do CASE
    se (conceito == "A" ou conceito == "B" ou conceito == "C" ou conceito == "D") {
      escolha (conceito) {
        caso "A": escreva("Excelente") pare
        caso "B": escreva("Bom") pare
        caso "C": escreva("Regular") pare
        caso "D": escreva("Insuficiente") pare
      }
    } senao {
      escreva("Conceito inválido")
    }
  }
}
7 ⚠️ Erros Comuns e Checklist
❌ Erros comuns
❌ Esquecer o pare ao final de cada caso
❌ Esquecer os dois-pontos após cada caso
❌ Usar CASE para intervalos (ex: caso > 7)
❌ Usar operadores lógicos dentro do caso
❌ Colocar variável sem parênteses no escolha
❌ Usar CASE com apenas duas opções
❌ Achar que CASE funciona com real
❌ Esquecer de abrir as chaves { }
✅ Checklist — Seu CASE está correto?
✅ A variável está entre parênteses?
✅ Cada caso tem dois-pontos no final?
✅ Cada bloco de caso termina com pare?
✅ As chaves { } estão corretamente fechadas?
✅ Os valores são fixos e exatos?
✅ A variável é do tipo inteiro ou cadeia?
✅ Incluiu o caso contrario para entradas inválidas?
✅ O código ficou mais legível que SE...SENÃO SE?
