E OU NÃO ESCOLHA CASO
1 Nível 1 — Operadores Lógicos Básicos
Ex 01 Cadastro de doador de sangue Operador E
INÍCIO
  LER idade, peso
  SE (idade >= 18) E (idade <= 65) E (peso >= 50) ENTÃO
    ESCREVER "Pode doar sangue"
  SENÃO
    ESCREVER "Não pode doar sangue"
  FIM_SE
FIM
Explicação: O intervalo de idade usa duas comparações unidas por E: (idade >= 18) E (idade <= 65). Isso é combinado com a verificação do peso também via E. Os parênteses isolam cada parte da condição composta.
Ex 02 Desconto no cinema Operador OU
INÍCIO
  LER categoria
  SE (categoria = "estudante") OU (categoria = "idoso") ENTÃO
    ESCREVER "Meia-entrada — R$ 15,00"
  SENÃO
    ESCREVER "Inteira — R$ 30,00"
  FIM_SE
FIM
Explicação: Apenas duas comparações unidas por OU. Se categoria for "estudante" OU "idoso", entra no primeiro bloco. O SENÃO captura qualquer outro valor. Se trocasse OU por E, ninguém receberia desconto (ninguém é "estudante" E "idoso" ao mesmo tempo).
Ex 03 Painel de acesso restrito Operador NÃO
INÍCIO
  LER administrador
  SE (NÃO administrador) ENTÃO
    ESCREVER "Painel disponível"
  SENÃO
    ESCREVER "Painel bloqueado para administradores"
  FIM_SE
FIM
Explicação: Administrador é uma variável lógica (VERDADEIRO/FALSO). O operador NÃO inverte o valor: se for FALSO, a condição vira VERDADEIRO e o painel é liberado. Sem NÃO, seria: SE administrador = FALSO ENTÃO.
2 Nível 2 — Combinações de Operadores
Ex 04 Classificação de IMC E + OU aninhados
INÍCIO
  LER peso, altura
  imc ← peso / (altura * altura)
  SE (imc < 18.5) OU (imc > 30.0) ENTÃO
    ESCREVER "Risco — encaminhar ao médico"
  SENÃO
    ESCREVER "Peso normal"
  FIM_SE
FIM
Explicação: IMC = peso / (altura × altura). A condição de risco captura os extremos: abaixo de 18,5 OU acima de 30,0. O SENÃO captura a faixa intermediária (normal). Parênteses em cada comparação mantêm a clareza.
Ex 05 Validação de senha com limite REPITA + SE + OU
INÍCIO
  tentativas ← 0
  senhaCorreta ← "1234"
  REPITA
    LER senha
    SE (senha = senhaCorreta) ENTÃO
      ESCREVER "Acesso liberado"
      INTERROMPER
    SENÃO
      ESCREVER "Senha incorreta"
      tentativas ← tentativas + 1
    FIM_SE
  ATÉ que tentativas = 3
  SE (tentativas = 3) ENTÃO
    ESCREVER "Conta bloqueada"
  FIM_SE
FIM
Explicação: O laço REPITA ATÉ executa enquanto tentativas for menor que 3. INTERROMPER encerra o laço imediatamente se a senha for correta. O SE após o laço só exibe "bloqueada" se tentativas = 3. O contador incrementa apenas no SENÃO (senha errada).
Ex 06 Evento ao ar livre E + NÃO combinados
INÍCIO
  LER chovendo, temperatura
  SE (NÃO chovendo) E (temperatura > 20) ENTÃO
    ESCREVER "Evento confirmado!"
  SENÃO
    ESCREVER "Evento cancelado"
  FIM_SE
FIM
Explicação: NÃO chovendo inverte o valor da variável lógica (se for FALSO, fica VERDADEIRO). O E une isso à condição da temperatura. Ambas precisam ser verdadeiras para o evento ocorrer. Os parênteses agrupam cada parte.
3 Nível 3 — Estrutura CASE (ESCOLHA...CASO)
Ex 07 Cardápio de restaurante ESCOLHA...CASO · menu
INÍCIO
  LER opcao
  ESCOLHA opcao
    CASO 1: ESCREVER "Hambúrguer — R$ 25,00"
    CASO 2: ESCREVER "Pizza — R$ 35,00"
    CASO 3: ESCREVER "Salada — R$ 18,00"
    CASO 4: ESCREVER "Lasanha — R$ 28,00"
    CASO 5: ESCREVER "Sopa — R$ 15,00"
    SENÃO
      ESCREVER "Opção inválida"
  FIM_ESCOLHA
FIM
Explicação: 5 CASOS, um para cada prato. O SENÃO captura qualquer valor diferente de 1 a 5. Compare mentalmente com um SE...SENÃO SE para ver a economia de código.
Ex 08 Categoria de habilitação ESCOLHA...CASO · letras
INÍCIO
  LER categoria
  ESCOLHA categoria
    CASO "A": ESCREVER "Moto"
    CASO "B": ESCREVER "Carro"
    CASO "C": ESCREVER "Caminhão"
    CASO "D": ESCREVER "Ônibus"
    CASO "E": ESCREVER "Carreta"
    SENÃO
      ESCREVER "Categoria inexistente"
  FIM_ESCOLHA
FIM
Explicação: CASE com strings — cada CASO usa aspas nos valores. O SENÃO captura qualquer caractere diferente de A, B, C, D, E. Com SE...SENÃO SE também funcionaria, mas o CASE é mais enxuto.
Ex 09 Classificação de conceito escolar CASE + operador lógico
INÍCIO
  LER conceito
  SE (conceito = "A") OU (conceito = "B") OU (conceito = "C") OU (conceito = "D") ENTÃO
    ESCOLHA conceito
      CASO "A": ESCREVER "Excelente"
      CASO "B": ESCREVER "Bom"
      CASO "C": ESCREVER "Regular"
      CASO "D": ESCREVER "Insuficiente"
    FIM_ESCOLHA
  SENÃO
    ESCREVER "Conceito inválido"
  FIM_SE
FIM
Explicação: O SE externo usa OU para validar se o conceito está entre A e D. Se estiver, o CASE mapeia para a descrição. Se não estiver, o SENÃO do SE exibe "Conceito inválido". A validação ocorre antes do CASE — se falhar, o CASE nem é executado.
