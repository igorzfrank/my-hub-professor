E OU NÃO ESCOLHA CASO
📌 Como estruturar seu pseudocódigo
① Operador E — Todas as condições precisam ser verdadeiras
Use quando o algoritmo só segue se todas as exigências forem cumpridas. Ex: SE (idade >= 18) E (possuiDoc = "sim") ENTÃO.
② Operador OU — Pelo menos uma condição verdadeira
Use quando basta uma exigência ser atendida. Ex: SE (cat = "estudante") OU (cat = "idoso") ENTÃO.
③ Operador NÃO — Inverte o valor lógico
Use para verificar o oposto. Ex: SE (NÃO bloqueado) ENTÃO.
④ Estrutura CASE — Múltiplas opções fixas
Use quando uma variável assume valores exatos. Ex: ESCOLHA dia... CASO 1: "Domingo"... FIM_ESCOLHA. O SENÃO captura não listados.
1 Nível 1 — Operadores Lógicos Básicos Uso individual de cada operador
Ex 01 Cadastro de doador de sangue Operador E
Um hemocentro lê a idade e o peso do candidato. Para aprovar, ele precisa ter 18 a 65 anos (inclusive) E pesar 50 kg ou mais. Caso contrário, informe que não pode doar. Construa o pseudocódigo.
Pense em: qual operador une as duas exigências? Como escrever o intervalo de idade (duas comparações unidas por qual operador)? Use parênteses para agrupar as comparações de idade.
Ex 02 Desconto no cinema Operador OU
O sistema lê a categoria do cliente. Se for "estudante" OU "idoso", exiba "Meia-entrada — R$ 15,00". Senão, exiba "Inteira — R$ 30,00".
Pense em: quantas comparações? Com OU, basta uma ser verdadeira. O que aconteceria se você usasse E por engano neste caso?
Ex 03 Painel de acesso restrito Operador NÃO
O sistema lê a variável lógica administrador (VERDADEIRO/FALSO). Se NÃO for administrador, exiba "Painel disponível". Senão, exiba "Painel bloqueado para administradores".
Pense em: como escrever "NÃO administrador" sem usar comparação com texto? Reescreva sem usar NÃO — como ficaria a lógica invertida?
2 Nível 2 — Combinações de Operadores E + OU + NÃO juntos
🧠 Como combinar vários operadores?
Use parênteses para agrupar e definir a ordem de avaliação. Ex: SE ( (a > 0) E (b > 0) ) OU (c > 0) ENTÃO — os parênteses mais internos são avaliados primeiro.
Ex 04 Classificação de IMC E + OU aninhados
Leia peso e altura, calcule o IMC (peso / altura²). Se o IMC for abaixo de 18,5 OU acima de 30,0, exiba "Risco — encaminhar ao médico". Senão, "Peso normal".
Pense em: as condições de risco são "fora da faixa" — abaixo de 18,5 OU acima de 30,0. Como escrever essa linha do SE? O SENÃO resolve a normalidade.
Ex 05 Validação de senha com limite REPITA + SE + OU
Caixa eletrônico: senha predefinida "1234", no máximo 3 tentativas. A cada erro: "Senha incorreta". Se acertar: "Acesso liberado" e encerra. Se errar as 3: "Conta bloqueada". Use REPITA ATÉ com contador.
Pense em: o laço termina quando acertar OU estourar tentativas. Como combinar essas duas saídas? O contador incrementa em qual bloco?
Ex 06 Evento ao ar livre E + NÃO combinados
O sistema lê: chovendo (lógico) e temperatura (inteiro). O evento só ocorre se NÃO estiver chovendo E temperatura for maior que 20 graus. Senão, cancelado.
Pense em: como escrever "NÃO está chovendo" com variável lógica? Como essa condição se liga à temperatura usando E? Use parênteses.
3 Nível 3 — Estrutura CASE (ESCOLHA...CASO) Múltiplas opções fixas
🧠 Quando usar ESCOLHA...CASO?
Sempre que uma variável assume valores exatos e conhecidos. O CASE substitui longas cadeias de SE...SENÃO SE. Não use CASE para intervalos (ex: "notas entre 7 e 10") — nesse caso, use SE com operadores lógicos.
Ex 07 Cardápio de restaurante ESCOLHA...CASO · menu
Totem de autoatendimento: cliente digita 1 a 5. Exiba: 1 — Hambúrguer (R$ 25), 2 — Pizza (R$ 35), 3 — Salada (R$ 18), 4 — Lasanha (R$ 28), 5 — Sopa (R$ 15). Se diferente, "Opção inválida". Use ESCOLHA...CASO.
Pense em: quantos CASO? O que colocar no SENÃO para capturar inválidos? Conte mentalmente quantas linhas você economiza comparado ao SE...SENÃO SE.
Ex 08 Categoria de habilitação ESCOLHA...CASO · letras
DETRAN lê a categoria (caractere). Mapeie: A — "Moto", B — "Carro", C — "Caminhão", D — "Ônibus", E — "Carreta". Para qualquer outra letra: "Categoria inexistente".
Pense em: CASE funciona com texto — cada CASO com caractere entre aspas. O SENÃO captura letras diferentes de A a E.
Ex 09 Classificação de conceito escolar CASE + operador lógico
Professor digita o conceito (texto). Tabela: A — "Excelente", B — "Bom", C — "Regular", D — "Insuficiente". Regra extra: antes do CASE, valide se o conceito está entre A e D — se não estiver, exiba "Conceito inválido" e encerre. Se estiver, use o CASE para exibir a descrição.
Pense em: este exercício combina SE com operadores lógicos (validar) + ESCOLHA...CASO (mapear). O SE vem antes do CASE? Como escrever a validação usando OU?
 
Critérios de avaliação do pseudocódigo
✅
Delimitação correta — todo pseudocódigo começa com INÍCIO e termina com FIM.
✅
Identação — blocos dentro de SE, REPITA e ESCOLHA estão recuados à direita.
✅
Operadores lógicos corretos — E, OU e NÃO usados conforme a lógica do problema.
✅
Parênteses na condição — condições compostas agrupadas com parênteses para evitar ambiguidade.
✅
CASE bem estruturado — ESCOLHA alinhado, cada CASO com valor certo, FIM_ESCOLHA fechando o bloco.
✅
SENÃO no CASE — usado corretamente (e quando necessário) para capturar valores não listados.
