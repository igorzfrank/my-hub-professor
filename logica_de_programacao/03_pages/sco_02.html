1 💻 Visual Studio Code — Instalação e Apresentação
🎯 O que é o VS Code?
Visual Studio Code (VS Code) é um editor de código-fonte gratuito e de código aberto desenvolvido pela Microsoft. É atualmente a ferramenta mais popular entre desenvolvedores web por ser leve, rápida, multiplataforma (Windows, Mac, Linux) e altamente personalizável através de extensões.
💡 Analogia simples
O VS Code está para o programador assim como o bisturi está para o cirurgião — é a ferramenta principal de trabalho, precisa ser precisa, confiável e estar sempre bem configurada.
📥Como instalar o VS Code
Passo 1 — Download
Acesse code.visualstudio.com e clique no botão de download para seu sistema operacional (Windows, Mac ou Linux).
Passo 2 — Instalação
Windows: Execute o instalador .exe, aceite os termos e marque as opções "Adicionar ao menu de contexto" e "Adicionar ao PATH".
Mac: Arraste o ícone para a pasta Aplicativos.
Linux: Use o gerenciador de pacotes (snap ou apt).
Passo 3 — Primeira execução
Abra o VS Code. Na tela de boas-vindas, você pode escolher um tema de cores (escuro ou claro) e começar a explorar a interface.
🧩Extensões essenciais para desenvolvimento web
🌐 Live Server
Atualiza o navegador automaticamente ao salvar arquivos HTML/CSS/JS.
🏷️ Auto Rename Tag
Renomeia automaticamente a tag de fechamento ao editar a de abertura.
✨ Prettier
Formatador de código automático (indentação, espaçamento, quebras de linha).
📝 HTML CSS Support
Autocompletar inteligente para classes e ids do CSS no HTML.
🎨 Material Icon Theme
Ícones coloridos para identificar tipos de arquivos no explorador.
🌈 Bracket Pair Colorizer
Colore pares de colchetes e chaves para facilitar a visualização.

2 🏗️ Introdução aos Conceitos de HTML
🎯 O que é HTML?
HTML (HyperText Markup Language) é a linguagem de marcação que estrutura todo o conteúdo da web. Não é uma linguagem de programação — é uma linguagem de marcação que usa tags (etiquetas) para definir o que cada parte do conteúdo representa: títulos, parágrafos, imagens, links, listas e muito mais.
💡 Analogia simples
HTML é o esqueleto de uma página web. Se um site fosse um corpo humano, o HTML seria os ossos (estrutura), o CSS seria a pele e roupas (aparência), e o JavaScript seria os músculos (movimento e interação).
🏗️Estrutura básica de um documento HTML5
<!-- Estrutura mínima de uma página HTML5 -->
<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <title>Minha Primeira Página</title>
</head>
<body>
  <h1>Olá, Mundo!</h1>
  <p>Esta é minha primeira página web.</p>
</body>
</html>
🏷️Tags HTML fundamentais
📐 Tags de Estrutura
<html> <head> <body> — base de toda página.
📝 Tags de Texto
<h1> a <h6> <p> <strong> <em>
🔗 Tags de Link e Mídia
<a> <img> <video> <audio>
📋 Tags de Lista
<ul> <ol> <li>
📦 Tags de Agrupamento
<div> <span> <header> <footer>
📝 Tags de Formulário
<form> <input> <button> <textarea>
🏷️Conceito de Tags: Abertura e Fechamento
A maioria das tags HTML trabalha em pares: uma tag de abertura e uma de fechamento. O conteúdo fica entre elas.

Exemplo: <p>Este é um parágrafo.</p>

Algumas tags são auto-fechantes (não têm par): <img src="foto.jpg"> e <br>.
💡 Dica Emmet: No VS Code, digite ! e pressione Tab para gerar automaticamente toda a estrutura básica de um documento HTML5.
3 📏 Padronização de Código — Boas Práticas
🎯 Por que padronizar o código?
Padronização de código é o conjunto de regras e convenções que tornam o código consistente, legível e de fácil manutenção. Um código padronizado é mais fácil de entender por outros desenvolvedores (e por você mesmo no futuro).
⭐Princípios fundamentais da padronização
↩️ Indentação consistente
Use espaços ou tabs (escolha um e mantenha). O padrão web é 2 ou 4 espaços por nível de indentação. Elementos filhos são recuados em relação aos pais.
🔤 Nomenclatura clara
Use nomes descritivos para classes e IDs. Prefira kebab-case (ex: meu-botao-principal) para classes CSS. Evite nomes genéricos como "div1" ou "caixa".
📝 Comentários úteis
Comente o porquê das decisões, não o quê o código faz. Código óbvio não precisa de comentário. Use <!-- comentário --> no HTML.
📐 Organização de arquivos
Separe HTML, CSS e JavaScript em arquivos distintos. Use pastas como /css, /js, /img para organizar o projeto.
✅ Fechamento de tags
Sempre feche as tags que abriu. Tags não fechadas podem quebrar o layout de formas imprevisíveis. O VS Code ajuda com destaque colorido.
🔡 Minúsculas para tags e atributos
Embora HTML não seja case-sensitive, o padrão da indústria é usar letras minúsculas para todas as tags e atributos: <div class="container">.
🛠️Ferramentas que ajudam na padronização
✨ Prettier
Formatador automático. Instale a extensão e configure para formatar ao salvar (Ctrl+S).
✅ HTMLHint
Extensão que analisa seu HTML e aponta erros de sintaxe e más práticas.
🌐 W3C Validator
Site oficial (validator.w3.org) que verifica se seu HTML segue os padrões web.
📐 EditorConfig
Define regras de formatação (tabs, charset) compartilhadas com toda a equipe.
✅ Exemplo de código padronizado
<header class="cabecalho">
  <h1>Título Principal</h1>
  <nav>
    <a href="/home">Home</a>
  </nav>
</header>
❌ Exemplo de código NÃO padronizado
<HEADER>
<H1>Titulo</H1>
<nav><a href="home">Home</a>
</HEADER>
