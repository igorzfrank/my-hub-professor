1 🧠 O que são Atributos ARIA?
🎯 Definição
ARIA (Accessible Rich Internet Applications) é um conjunto de atributos especiais que podem ser adicionados a elementos HTML para tornar aplicações web mais acessíveis para pessoas que utilizam tecnologias assistivas, como leitores de tela. Eles preenchem a lacuna quando o HTML semântico não é suficiente para descrever componentes complexos e interativos.
💡 Analogia simples
Pense no HTML semântico como as placas de rua padrão (Placa de "PARE", "PROIBIDO ESTACIONAR"). Os atributos ARIA são como placas adicionais com instruções detalhadas em Braille – eles explicam o propósito e o estado atual do elemento para quem não pode ver a interface visual.
🎭 Exemplo do dia a dia
Se você construir um botão com uma <div>, o leitor de tela não saberá que aquilo é um botão. O ARIA sussurra para o leitor: "psst, isso é um botão que pode ser clicado!"
2 ⚙️ Principais Atributos ARIA de Widget
1aria-label — Rótulo textual para elementos sem texto visível
Fornece um nome acessível quando não há texto visível no elemento. Ideal para botões com apenas ícones.
<button aria-label="Fechar janela de diálogo">X</button>
2aria-labelledby — Aponta para outro elemento que contém o rótulo
Conecta um elemento ao seu rótulo definido em outro lugar da página. Útil quando o texto já existe no HTML.
<h2 id="titulo">Configurações</h2>
<div role="dialog" aria-labelledby="titulo">...</div>
3aria-describedby — Descrição adicional para o elemento
Aponta para um elemento que contém uma descrição mais detalhada. Muito usado em campos de formulário com mensagens de erro.
<input aria-describedby="dica-erro">
<span id="dica-erro">O e-mail deve conter @</span>
4Atributos de Estado — Comunicam o estado atual do elemento
aria-checked
Checkbox/Radio: true/false
aria-expanded
Menu/Collapse: true/false
aria-disabled
Elemento desabilitado
aria-pressed
Botão toggle: true/false
aria-selected
Opção selecionada
aria-required
Campo obrigatório
5aria-hidden — Oculta elementos do leitor de tela
Quando true, o elemento e seus filhos ficam invisíveis para tecnologias assistivas. Use para elementos puramente decorativos.
<span aria-hidden="true">⭐</span>
3 ✅❌ Boas Práticas e Erros Comuns com ARIA
✅ O que fazer
✅ Priorize HTML semântico — Use <button> em vez de <div role="button">
✅ Use ARIA para complementar, não substituir o HTML nativo
✅ Teste com leitores de tela reais — NVDA (Windows), VoiceOver (Mac)
✅ Mantenha estados sincronizados — Se aria-expanded muda, o JS deve refletir
✅ Use aria-hidden="true" para elementos decorativos sem significado
❌ O que evitar
❌ Não use ARIA quando HTML nativo resolve — role="button" em <div> é pior
❌ Não repita informações — aria-label sobrepondo texto visível confunde
❌ Não use role="alert" para mensagens que não são urgentes
❌ Não esqueça o gerenciamento de foco — ARIA não move foco automaticamente
❌ Não use aria-hidden em elementos focáveis — usuário "cai" em invisível
🧠 Regra de Ouro do ARIA
"No ARIA is better than bad ARIA."
Nenhum ARIA é melhor que ARIA mal feito.
✅ Exemplo Correto
<!-- HTML nativo: o leitor já sabe que é um botão -->
<button aria-expanded="false">Menu</button>
❌ Exemplo Incorreto
<!-- ARIA desnecessário: já existe tag nativa -->
<div role="button" tabindex="0">Menu</div>
4 🖼️ O Atributo ALT — Texto Alternativo para Imagens
🎯 O que é o atributo alt?
O atributo alt (alternative text) é um atributo obrigatório da tag <img> que fornece uma descrição textual do conteúdo e função de uma imagem. É lido por leitores de tela e exibido quando a imagem não carrega.
💡 Analogia simples
Imagine que você está descrevendo uma foto para um amigo por telefone. O texto alt é exatamente essa descrição verbal — ela permite que quem não pode ver a imagem entenda o que ela representa e qual é seu propósito na página.
⭐Por que o alt é essencial?
🦯 Acessibilidade
Leitores de tela vocalizam o alt. Sem ele, usuários cegos perdem todo o conteúdo visual.
🔍 SEO
Google usa o alt para indexar imagens no Google Imagens. Imagens sem alt perdem tráfego.
📶 Conexões lentas
Se a imagem falhar ao carregar, o texto alt é exibido, mantendo o contexto da página.
📋 Conformidade legal
O atributo alt é obrigatório por normas WCAG e leis de acessibilidade em muitos países.
📝Como escrever um bom texto alternativo
✅ Seja descritivo e específico
Descreva o conteúdo e a função. Ex: alt="Cachorro golden retriever brincando com bola vermelha no parque"
✅ Seja conciso (até 125 caracteres)
Leitores de tela cortam textos longos. Vá direto ao ponto.
✅ Não comece com "Imagem de..."
O leitor de tela já anuncia que é uma imagem. Economize tempo do usuário.
✅ Imagens decorativas: alt vazio
Use alt="" para que o leitor ignore a imagem. Não omita o atributo.
✅ Bons exemplos de alt
✅ alt="Gráfico de barras: vendas cresceram 25% em 2024"
✅ alt="Logo da Empresa XYZ"
✅ alt="Botão de pesquisa com ícone de lupa"
✅ alt="" (imagem decorativa)
❌ Maus exemplos de alt
❌ alt="foto1.jpg"
❌ alt="imagem"
❌ alt="Imagem de um lindo pôr do sol..." (muito longo)
❌ Atributo alt ausente (inválido!)
5 🧪 Exemplos Práticos — ARIA e ALT Juntos
🔘 Botão com ícone (sem texto visível)
<!-- Correto: usa aria-label para nome acessível -->
<button aria-label="Adicionar ao carrinho">
  <img src="carrinho.svg" alt="">
</button>
📂 Menu expansível com aria-expanded
<button aria-expanded="false" aria-controls="submenu">Produtos</button>
<ul id="submenu" aria-hidden="true">
  <li>Eletrônicos</li>
</ul>
🖼️ Galeria de imagens com figure e alt descritivo
<figure>
  <img
    src="por-do-sol.jpg"
    alt="Pôr do sol alaranjado sobre montanhas com silhueta de pássaros voando"
  >
  <figcaption>Vista do mirante — Outono 2024</figcaption>
</figure>
🛍️ Card de produto com ARIA e ALT combinados
<article aria-labelledby="produto-nome">
  <img src="tenis.jpg" alt="Tênis esportivo azul com detalhes brancos">
  <h3 id="produto-nome">Tênis Runner Pro</h3>
  <button aria-label="Comprar Tênis Runner Pro por R$ 299">Comprar</button>
</article>
6 ✅ Checklist de Acessibilidade — ARIA + ALT
✅ Checklist ARIA
✅ HTML semântico é a primeira opção (use <button>, não div role="button")
✅ Todo elemento interativo tem um nome acessível (aria-label ou aria-labelledby)
✅ Estados são comunicados (aria-expanded, aria-checked, aria-pressed)
✅ Elementos decorativos têm aria-hidden="true"
✅ Mensagens dinâmicas usam role="alert" ou aria-live
✅ Testei com pelo menos um leitor de tela real (NVDA ou VoiceOver)
✅ Checklist ALT
✅ Toda tag <img> tem atributo alt (mesmo que vazio)
✅ Imagens informativas têm alt descritivo e conciso
✅ Imagens decorativas têm alt="" (não omita o atributo)
✅ O texto alt não começa com "Imagem de..." ou "Foto de..."
✅ Imagens com texto embutido reproduzem o texto no alt
✅ Logos têm alt com o nome da empresa (ex: alt="Logo Nome da Empresa")
